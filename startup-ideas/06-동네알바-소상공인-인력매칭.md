## 1. 문제 인식 (Problem) — 창업 아이템의 필요성

### 1-1. 국내외 시장 현황

#### (1) 소상공인 현황 및 인력 문제

대한민국 소상공인은 약 **790만 명**으로 전체 사업체의 **87%**를 차지하며, 국가 경제의 근간을 이루고 있다 [1]. 그러나 자영업자 10명 중 7명은 창업 후 5년을 버티지 못하고 폐업하며, 이 중 **인건비 부담**이 폐업 사유 상위 3위 안에 꾸준히 포함되고 있다 [2]. 특히 음식점, 카페, 편의점 등 서비스 업종 소상공인은 주말, 공휴일, 점심·저녁 피크 타임에 집중적으로 인력이 필요하지만, 정규직 또는 장기 아르바이트를 고용하기에는 월 인건비 부담(최저임금 기준 월 209만 원+, 4대보험 포함 시 약 240만 원)이 과중하다 [3].

2025년 최저임금은 시간당 10,030원으로 전년 대비 1.7% 인상되었으며 [4], 소상공인의 인건비 부담은 해마다 가중되고 있다. 중소벤처기업부의 「2024년 소상공인 실태조사」에 따르면, 소상공인의 **62.3%**가 "필요한 시점에 단기 인력을 구하는 것이 매우 어렵다"고 응답했으며, 구인에 평균 **3~7일**이 소요된다고 밝혔다 [5].

#### (2) 긱이코노미 플랫폼 시장

글로벌 긱이코노미 시장은 2024년 약 **$556B** 규모이며, 2032년까지 **$1.8T**로 성장할 전망이다 (CAGR 15.4%) [6]. 국내 플랫폼 노동자는 약 **220만 명**(전체 취업자의 8.5%)에 달하며, 플랫폼 기반 인력 중개 시장은 연 **3조 원** 이상 규모로 추산된다 [7].

배달(쿠팡이츠, 배달의민족), 가사도우미(대리주부, 미소), 퀵서비스(부릉) 등 특정 업종에서는 긱 플랫폼이 이미 자리잡았으나, **소상공인 대상 범용 단기 인력 매칭** 영역은 아직 유의미한 점유율을 가진 플랫폼이 없는 블루오션이다.

#### (3) 대학생 시간제 근로 현황

2024년 기준 대학생 약 **320만 명** 중 아르바이트 경험자 비율은 **73.2%**에 달하지만 [8], 대부분 주 3일 이상의 장기 고정 근무 형태이다. 수업 시간표에 따라 매일 다른 **자투리 시간(2~4시간)**이 발생하지만, 이를 소득으로 전환할 수 있는 채널이 부재하다. 대학생 설문조사 결과 **81.7%**가 "수업 사이 2~3시간 정도의 단기 알바를 하고 싶다"고 응답했다 [9].

#### (4) 소상공인 디지털전환 정책 트렌드

정부는 「소상공인 디지털전환 촉진법」 제정을 추진 중이며 [10], 중소벤처기업부는 2025년 소상공인 디지털전환 지원 예산을 전년 대비 **35% 증액**한 2,800억 원으로 편성했다 [11]. 카카오 '프로젝트 단골', 네이버 '스마트플레이스' 등 빅테크도 소상공인 디지털전환 시장에 진입하고 있어, 소상공인 대상 플랫폼 서비스의 수용성이 빠르게 높아지고 있다 [12].

### 1-2. 핵심 문제점 및 필요성

#### 문제 1: 소상공인의 급한 인력 구인 불가능

소상공인은 갑작스러운 직원 결근, 주말·공휴일 피크, 행사·이벤트 등으로 **당일 또는 수시간 내에** 인력이 필요한 상황이 빈번하다. 그러나 기존 구인 플랫폼(알바천국, 알바몬)은 이력서 등록 → 서류 검토 → 면접 → 채용의 **장기 채용 프로세스** 중심으로 설계되어 있어, 최소 3~7일의 채용 리드타임이 발생한다. 이 기간 동안 소상공인은 인력 부족 상태로 영업하거나, 가족 노동에 의존하거나, 최악의 경우 영업을 중단해야 한다.

#### 문제 2: 높은 노쇼(No-Show) 비율

단기 알바 시장에서 **노쇼율은 약 25~35%**에 달하는 것으로 추정된다 [13]. 알바를 구했더라도 당일 연락 없이 나타나지 않는 경우가 빈번하며, 소상공인 입장에서는 대체 인력을 구할 시간적 여유가 없어 직접적인 매출 손실로 이어진다. 기존 플랫폼은 노쇼에 대한 실질적인 제재 메커니즘이 부재하다.

#### 문제 3: 대학생 자투리 시간의 비효율적 활용

대학생의 하루 평균 수업 외 자투리 시간은 약 **3.2시간**이지만 [14], 이를 소득 활동으로 전환하기 어렵다. 장기 아르바이트는 수업 시간표와 충돌하고, 배달 플랫폼은 개인 이동수단(오토바이 등)이 필요하며, 과외는 정기적 시간 확보가 필수적이다. **"지금부터 2시간 동안"** 같은 유연한 근무 형태를 지원하는 플랫폼이 없다.

#### 문제 4: 기존 플랫폼의 한계

| 기존 플랫폼 | 주요 한계점 |
|------------|------------|
| 알바천국/알바몬 | 장기 채용 중심, 즉시 매칭 불가, 구인광고 유료(건당 3~10만 원) |
| 당근알바 | 동네 기반이나 구인글 노출 수동, 실시간 매칭 엔진 부재, 정산 미지원 |
| 쿠팡이츠/배민 | 배달 특화, 소상공인 매장 내 서빙·주방·진열 등 범용 인력 미지원 |
| 숨고 | 전문 서비스 매칭 중심, 단순 노무 단기알바 부적합 |

#### 문제 5: 근로 투명성 및 법적 보호 미흡

단기 알바 시장에서는 구두 약속에 의한 근무, 임금 미지급·체불, 근로계약서 미작성 등의 문제가 만연하다. 고용노동부 조사에 따르면 아르바이트생의 **33.4%**가 근로계약서를 작성하지 않았으며, **14.2%**가 임금 체불을 경험했다 [15]. 이러한 구조적 문제는 디지털 플랫폼을 통한 자동화된 계약·정산 시스템으로 해결할 수 있다.

### 1-3. 시장 규모 추정 (TAM-SAM-SOM)

| 구분 | 산출 근거 | 규모 |
|------|----------|------|
| **TAM** (전체 시장) | 국내 긱이코노미 플랫폼 시장 전체 | **3조 원+** |
| **SAM** (유효 시장) | 소상공인(790만) 중 인력 수요 보유(30%) × 연간 단기 고용비 평균 120만 원 | **약 2.8조 원** |

## 2. 실현 가능성 (Solution) — 창업 아이템의 개발 계획

### 2-1. 핵심 제품·서비스 개요

**동네알바**는 소상공인과 대학생을 **실시간으로 연결**하는 단기 인력 매칭 플랫폼으로, 다음 네 가지 핵심 모듈로 구성된다.

#### 모듈 1: 실시간 긴급 매칭 엔진

- **"지금부터 3시간" 즉시 매칭**: 소상공인이 필요 인력, 시급, 근무 시간, 업무 내용을 입력하면 **30초 내** 최적 인력 3~5명을 추천
- **Local LLM 기반 매칭 최적화**: 서버에 배포된 Llama 3 8B 파인튜닝 모델이 위치(거리), 시간 가용성, 업종 경험, 과거 평점, 선호도를 종합 분석하여 매칭 스코어를 산출
- **실시간 푸시 알림**: 매칭 대상 대학생에게 즉시 알림 → 수락/거절 → 30초 내 확정
- **Local LLM 선택 이유**:
  - **빠른 추론 속도**: 실시간 매칭에서 API 호출 지연(200~500ms) 대비 로컬 추론 50ms 이내 달성
  - **API 비용 제거**: 매칭 건당 API 비용 발생 없음 (월 수만 건 매칭 시 수백만 원 절감)
  - **근로자 데이터 프라이버시**: 위치, 근무 이력, 평점 등 민감한 개인정보가 외부 API 서버로 전송되지 않음

#### 모듈 2: 위치 기반 캠퍼스 반경 추천

- **대학 캠퍼스 반경 2km** 내 소상공인을 자동 탐색하여 도보·자전거로 10분 이내 이동 가능한 매칭만 추천
- **실시간 GPS 기반 가용 인력 현황 지도**: 소상공인이 현재 시점에 근무 가능한 대학생 분포를 지도에서 확인
- **동선 최적화**: 대학생의 수업 시간표와 연동하여 수업 전후 가용 시간에 자동으로 근처 알바 추천
- **캠퍼스 존(Zone) 시스템**: 대학별 거점을 설정하고, 해당 존 내 소상공인-대학생 매칭에 집중하여 네트워크 효과 극대화

#### 모듈 3: 신뢰 시스템 (Trust System)

- **상호 평점제**: 근무 완료 후 소상공인 ↔ 대학생 양방향 5점 평점 + 텍스트 리뷰
- **노쇼 방지 보증금**: 알바 수락 시 보증금 5,000원 자동 예치, 정상 근무 완료 시 전액 반환, 무단 불참 시 소상공인에게 보상금으로 지급
- **GPS 출퇴근 인증**: 근무 장소 반경 50m 이내에서 출근·퇴근 GPS 체크인 → 자동 근무시간 산정
- **Local ML 노쇼 예측 모델**: 과거 노쇼 패턴, 날씨, 요일, 시간대, 사용자 행동 데이터를 학습한 로컬 모델이 노쇼 확률을 사전 예측 → 고위험 매칭 시 대체 인력 자동 확보
  - **Local 모델 선택 이유**: 사용자의 행동 패턴, 위치 이력 등 민감 데이터를 외부로 전송하지 않으면서도 실시간 예측 가능
- **신분 인증**: 대학교 이메일(ac.kr) 인증 + 본인 인증(PASS 앱 연동)
- **신뢰 등급**: 누적 근무 횟수·평점에 따라 브론즈 → 실버 → 골드 → 다이아몬드 등급 부여, 등급별 우선 매칭 혜택

#### 모듈 4: 간편 정산 시스템

- **근무 완료 즉시 자동 송금**: GPS 퇴근 인증 확인 후 PG사(토스페이먼츠/NHN KCP)를 통해 대학생 계좌로 **10분 이내** 자동 입금
- **전자 근로계약서 자동 생성**: 매칭 확정 시 단기근로계약서(근로기준법 준수)가 자동 생성되어 양 당사자에게 전자 서명 요청
- **최저임금 자동 검증**: 시급 입력 시 최저임금(10,030원) 미달 여부 자동 검증, 미달 시 등록 불가
- **세무 자동 정리**: 월별 소득 내역 자동 정리, 종합소득세 신고 시 활용 가능한 리포트 제공

### 2-2. Local LLM 기반 AI 기능 상세

동네알바 플랫폼의 핵심 경쟁력은 **Local LLM(로컬 대규모 언어 모델)**을 활용한 세 가지 AI 기능에 있다. 모든 AI 추론은 자체 서버에서 수행되며, 외부 API 의존성을 완전히 제거하여 비용 효율성과 데이터 프라이버시를 동시에 확보한다.

#### AI 기능 1: 실시간 매칭 최적화 엔진

| 항목 | 상세 |
|------|------|
| **모델** | Llama 3 8B → 매칭 도메인 파인튜닝 (LoRA) |
| **입력** | 소상공인 요청(업종, 위치, 시간, 업무 내용, 선호 조건) + 가용 대학생 풀(위치, 시간, 경험, 평점) |
| **출력** | 매칭 스코어 순위 Top 5 + 매칭 사유 설명 |
| **추론 속도** | 50ms 이내 (GPU 서버 기준) |
| **학습 데이터** | 파일럿 기간 매칭 로그 10만 건+ (합성 데이터 포함 초기 학습) |
| **Local 선택 이유** | 매칭 건당 API 비용 0원 (월 5만 건 매칭 시 GPT-4 API 대비 약 **월 500만 원 절감**), 실시간 응답 필수 |

#### AI 기능 2: 구인공고 자동 생성 (Local NLP)

| 항목 | 상세 |
|------|------|
| **모델** | Llama 3 8B → 한국어 구인공고 생성 파인튜닝 |
| **기능** | 소상공인이 "카페, 3시간, 서빙" 같은 키워드만 입력하면 → 완성된 구인공고(업무 설명, 주의사항, 복장 요건 등) 자동 생성 |
| **입력** | 업종 카테고리, 근무 시간, 핵심 업무 키워드 (3~5개) |
| **출력** | 구조화된 구인공고 텍스트 (200~300자) |
| **Local 선택 이유** | 매 공고 생성마다 API 비용 불필요, 소상공인 매장 정보가 외부로 유출되지 않음, 업종별 특화 파인튜닝으로 **범용 LLM 대비 높은 품질** 달성 |

**예시**: 소상공인 입력 → "분식집 / 점심 2시간 / 서빙, 설거지"
```
[구인공고 자동 생성 결과]
📍 OO분식 (홍대입구역 도보 5분)
⏰ 오늘 11:30 ~ 13:30 (2시간)
💰 시급 11,000원 (즉시 정산)
📋 업무: 홀 서빙, 테이블 정리, 식기 세척
👕 복장: 편한 복장 (앞치마 제공)
⚠️ 참고: 점심 피크 시간대로 빠른 움직임 필요
```

#### AI 기능 3: 노쇼/부정행위 예측 모델

| 항목 | 상세 |
|------|------|
| **모델** | XGBoost + 경량 Transformer (자체 서버 배포) |
| **입력 피처** | 사용자 과거 노쇼 이력, 수락 후 취소 패턴, 요일/시간대, 날씨, 거리, 매칭 후 경과 시간, 앱 사용 패턴 |
| **출력** | 노쇼 확률(0~100%), 리스크 등급(Low/Medium/High) |
| **활용** | High 리스크 매칭 시 자동으로 대체 인력 1명 추가 확보, 소상공인에게 리스크 알림 |
| **정확도 목표** | AUC 0.85 이상 (파일럿 데이터 기반 지속 개선) |
| **Local 선택 이유** | 사용자 행동 패턴, GPS 이동 데이터 등 **고도로 민감한 개인정보**를 외부 전송 없이 처리, GDPR/개인정보보호법 준수 용이 |

### 2-3. 기술 스택 (Tech Stack)

| 레이어 | 기술 | 선정 이유 |
|--------|------|----------|
| **프론트엔드 (모바일)** | React Native + TypeScript | iOS/Android 동시 개발, 핫 리로딩으로 빠른 이터레이션 |
| **프론트엔드 (웹)** | Next.js 14 | 소상공인 대시보드용, SSR로 빠른 로딩 |
| **백엔드** | Node.js (Express/Fastify) | 비동기 I/O로 실시간 매칭 처리에 최적, 풍부한 생태계 |
| **데이터베이스** | PostgreSQL 16 | 위치 데이터(PostGIS), ACID 트랜잭션, JSON 지원 |
| **캐시/큐** | Redis 7 | 실시간 매칭 큐, 세션 관리, 위치 캐싱(Geo 명령어) |
| **Local LLM 서빙** | vLLM + Llama 3 8B | 높은 처리량(throughput)의 로컬 LLM 추론 서버 |
| **ML 프레임워크** | PyTorch + Hugging Face | 모델 파인튜닝 및 배포 |
| **ML 모니터링** | MLflow | 모델 버전 관리, A/B 테스트, 성능 모니터링 |
| **결제/정산** | 토스페이먼츠 API | 간편결제, 자동 정산, 에스크로 지원 |
| **인증** | PASS 앱 연동 + Firebase Auth | 본인 인증 + 소셜 로그인 |
| **인프라** | AWS EC2 (GPU: g5.xlarge) + RDS | Local LLM 서빙용 GPU 인스턴스, 관리형 DB |
| **CI/CD** | GitHub Actions + Docker | 자동 빌드·배포 파이프라인 |
| **모니터링** | Grafana + Prometheus | 서버 상태, API 응답시간, 매칭 성공률 실시간 모니터링 |

### 3-2. 비즈니스 모델 (수익화 전략)

#### 수익원 1: 매칭 수수료 (메인 수익)

| 항목 | 내용 |
|------|------|
| 과금 방식 | 매칭 성사 시 시급의 **10%**를 구인자(소상공인)에게 부과 |
| 과금 시점 | 근무 완료 후 정산 시 자동 차감 |

#### BEP(손익분기점) 분석

- **월 고정비**: 서버 150만 원 + 인건비 1,200만 원 + 사무실 200만 원 + 기타 150만 원 = **약 1,700만 원**
- **건당 변동비**: PG 수수료(결제액의 2.5%) + CS 비용 ≒ 건당 1,000원
- **건당 매출**: 평균 수수료 4,000원 - 변동비 1,000원 = **건당 공헌이익 3,000원**
- **BEP 매칭 건수**: 1,700만 원 / 3,000원 = **월 약 5,667건**
- **BEP 도달 예상**: 정식 출시 후 **8~10개월** (27년 3분기)
