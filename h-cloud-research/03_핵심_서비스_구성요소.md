# 제3장: 핵심 서비스 구성요소

---

## 3.1 개요

H-Cloud 플랫폼은 모두 오픈소스를 기반으로 자체 개발된 다수의 서비스를 제공하고 있다[2][3]. 이들 서비스는 IaaS(Infrastructure as a Service) 및 PaaS(Platform as a Service) 계층에 걸쳐 구성되어 있으며, 클라우드 환경에서 서비스 개발 및 런칭을 위한 총 9가지 이상의 자체 개발 솔루션이 포함된다[1].

각 서비스명에는 'H'가 접두어로 붙어 H-Cloud 브랜드 정체성을 유지하고 있다는 특징이 있다.

---

## 3.2 컴퓨트 서비스

### 3.2.1 HCS (H-Cloud Compute Service)

HCS는 H-Cloud의 핵심 컴퓨트 서비스로, 가상 서버 인스턴스를 제공한다[2][3]. AWS의 EC2, GCP의 Compute Engine에 대응하는 서비스로 이해할 수 있다. 사용자는 HCS를 통해 필요한 사양의 가상 서버를 신속하게 프로비저닝하고 운영할 수 있다.

HCS의 주요 특징은 다음과 같다:

- **온디맨드 프로비저닝**: 필요 시 즉시 가상 서버를 생성하고 운영할 수 있다.
- **다양한 인스턴스 타입**: 워크로드 특성에 따라 적합한 사양의 인스턴스를 선택할 수 있다.
- **자동 스케일링**: 트래픽 변동에 따라 인스턴스 수를 자동으로 조절할 수 있다.
- **모니터링 통합**: Hubble 모니터링 시스템과 자동으로 연동되어, 별도 설정 없이 가상머신의 메트릭을 수집하고 대시보드에서 확인할 수 있다[4].

### 3.2.2 HMetal (H-Cloud Bare Metal)

HMetal은 가상화 계층 없이 물리 서버를 직접 할당하는 베어메탈(Bare Metal) 서비스이다[2][3]. 가상화 오버헤드가 없어 최대 성능을 발휘할 수 있으며, 고성능 컴퓨팅(HPC), 대규모 데이터 처리, AI/ML 학습 등 높은 컴퓨팅 성능이 요구되는 워크로드에 적합하다.

베어메탈 서비스의 장점은 다음과 같다:

- **하이퍼바이저 없는 직접 실행**: 가상화 계층이 없어 CPU, 메모리, I/O 성능 손실이 없다.
- **전용 자원 할당**: 물리 서버의 전체 자원을 독점적으로 사용할 수 있어, 다른 테넌트의 영향을 받지 않는다.
- **유연한 OS 선택**: 워크로드에 최적화된 운영체제를 자유롭게 설치할 수 있다.
- **AI/GPU 워크로드 지원**: AI GPU 인프라 투자에 따른 GPU 베어메탈 서비스를 제공하여 AI 학습 및 추론 워크로드를 지원한다[8].

## 3.3 네트워크 서비스

### 3.3.1 HFabric (H-Cloud Network Fabric)

HFabric은 H-Cloud 내부 및 외부 네트워크 연결을 관리하는 네트워크 서비스이다[2][3]. 소프트웨어 정의 네트워킹(SDN, Software Defined Networking) 기술을 기반으로 가상 네트워크를 구성하고 관리한다.

HFabric의 주요 기능은 다음과 같다:

- **가상 네트워크 생성**: 사용자별 독립적인 가상 네트워크를 생성하고 관리할 수 있다.
- **네트워크 격리**: 테넌트 간 네트워크를 격리하여 보안을 강화한다.
- **외부 연결**: 에퀴닉스 Fabric을 통한 퍼블릭 클라우드 및 외부 네트워크와의 상호연결을 지원한다[6].
- **트래픽 관리**: 네트워크 트래픽의 라우팅, 필터링, QoS(Quality of Service) 정책을 소프트웨어적으로 관리한다.

### 3.3.2 HLB (H-Cloud Load Balancer)

HLB는 서버 간 트래픽을 분산 처리하는 서버 로드밸런싱 서비스이다[2][3]. 다수의 서버 인스턴스에 들어오는 요청을 균등하게 분배하여, 특정 서버에 부하가 집중되는 것을 방지하고 전체 시스템의 가용성과 응답 성능을 향상시킨다.

HLB의 주요 기능은 다음과 같다:

- **L4/L7 로드밸런싱**: TCP/UDP 수준(L4)과 HTTP/HTTPS 수준(L7)의 트래픽 분산을 모두 지원한다.
- **헬스체크**: 백엔드 서버의 상태를 주기적으로 확인하여, 장애가 발생한 서버에는 트래픽을 전송하지 않는다.
- **세션 유지(Session Persistence)**: 동일 사용자의 요청이 동일 서버로 전달되도록 세션 어피니티를 지원한다.
- **SSL 터미네이션**: SSL/TLS 암호화 처리를 로드밸런서에서 수행하여 백엔드 서버의 부하를 줄인다.

## 3.4 스토리지 서비스

### 3.4.1 H2O (H-Cloud Object Storage)

H2O는 대용량 비정형 데이터를 저장하고 관리하는 오브젝트 스토리지 서비스이다[2][3]. AWS의 S3에 대응하는 서비스로, 이미지, 동영상, 로그 파일, 백업 데이터 등 대용량 비정형 데이터의 저장 및 관리에 적합하다.

H2O의 주요 특징은 다음과 같다:

- **무제한 확장성**: 저장 용량의 제한 없이 데이터를 저장할 수 있으며, 스케일아웃 구조로 용량 확장이 용이하다.
- **높은 내구성**: 데이터를 여러 노드에 복제하여 저장함으로써 높은 데이터 내구성을 보장한다.
- **RESTful API**: HTTP 기반의 RESTful API를 통해 데이터에 접근할 수 있다.
- **버전 관리**: 오브젝트의 버전을 관리하여 이전 버전으로의 복원이 가능하다.
- **라이프사이클 관리**: 데이터의 보존 기간에 따른 자동 삭제 또는 티어링 정책을 설정할 수 있다.

## 3.5 모니터링 및 헬스체크 서비스

### 3.5.1 Hubble (모니터링 서비스)

Hubble은 H-Cloud 인프라 및 애플리케이션의 상태를 실시간으로 모니터링하는 자체 개발 서비스이다[2][3]. Hubble이라는 명칭은 우주를 관측하는 허블 망원경에서 착안한 것으로, H-Cloud 인프라 전반을 '관측'한다는 의미를 담고 있다.

Hubble의 주요 기능은 다음과 같다[4]:

- **자동 메트릭 수집**: 모든 컨테이너와 가상머신의 메트릭을 자동으로 수집한다. 사용자는 별도의 설정 작업을 수행하지 않고도, 컨테이너와 가상머신을 가동하기만 하면 메트릭이 수집된다.
- **클라우드 대시보드**: 수집된 메트릭 정보를 시각화된 대시보드에서 확인할 수 있다.
- **클러스터 상태 모니터링**: 쿠버네티스 클러스터의 상태를 전용 대시보드에서 실시간으로 모니터링할 수 있다[1].
- **알림(Alerting)**: 사전에 정의된 임계치를 초과하면 자동으로 알림을 발생시킨다.
- **24/7 기술지원**: 24시간 365일 모니터링 및 기술지원 운영 서비스를 제공한다[3].

### 3.5.2 Sonar (헬스체크 서비스)

Sonar는 시스템의 건강 상태를 주기적으로 점검하는 헬스체크 서비스이다[2][3]. Sonar라는 명칭은 수중 탐지에 사용되는 소나(SONAR)에서 착안한 것으로, 시스템의 숨겨진 문제를 탐지한다는 의미를 담고 있다.

Sonar의 주요 기능은 다음과 같다:

- **주기적 상태 점검**: 인프라 구성요소의 상태를 주기적으로 점검하여 이상 여부를 확인한다.
- **엔드포인트 모니터링**: 서비스 엔드포인트의 응답 상태와 응답 시간을 모니터링한다.
- **장애 사전 탐지**: 이상 징후를 조기에 발견하여 장애 발생 전에 선제적 조치를 취할 수 있도록 한다.
- **상태 리포트**: 시스템 건강 상태에 대한 종합 리포트를 생성한다.

## 3.6 컨테이너 서비스

### 3.6.1 HKS (Hyundai Kubernetes Service)

HKS는 H-Cloud에 최적화된 쿠버네티스 기반의 컨테이너 오케스트레이션 서비스이다[4][5]. H-Cloud의 가장 핵심적인 플랫폼 서비스로, SUSE Rancher Prime을 기반으로 자체 커스터마이징하여 구현되었다. HKS에 대한 상세한 기술 분석은 제5장에서 별도로 다룬다.

## 3.7 클라우드 관리 플랫폼(CMP)

현대오토에버는 자체 클라우드 관리 플랫폼(CMP, Cloud Management Platform)을 통해 H-Cloud 자원의 통합 관리를 수행한다[26]. CMP는 다음과 같은 기능을 제공한다:

- **자원 관리**: 컴퓨트, 스토리지, 네트워크 등 클라우드 자원의 프로비저닝, 모니터링, 해제를 통합 관리한다.
- **비용 최적화**: 자원 사용량을 분석하여 비용을 최적화하기 위한 권고사항을 제공한다[3].
- **멀티 클라우드 관리**: 프라이빗 클라우드와 퍼블릭 클라우드의 자원을 단일 콘솔에서 관리할 수 있다.
- **거버넌스**: 자원 사용 정책, 접근 권한, 컴플라이언스 준수를 관리한다.
- **셀프서비스 포털**: 사용자가 필요한 자원을 직접 요청하고 프로비저닝할 수 있는 셀프서비스 기능을 제공한다.

## 3.8 서비스 구성요소 간 관계

H-Cloud의 각 서비스 구성요소는 유기적으로 연결되어 전체 플랫폼을 구성한다. 그 관계는 다음과 같다:

```
                     ┌──────────┐
                     │   CMP    │ (통합 관리)
                     └─────┬────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
    ▼                      ▼                      ▼
┌────────┐          ┌────────────┐          ┌──────────┐
│  HCS   │          │    HKS     │          │  HMetal   │
│(컴퓨트) │          │(쿠버네티스) │          │(베어메탈)  │
└────┬───┘          └─────┬──────┘          └─────┬────┘
     │                    │                       │
     └────────────────────┼───────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │               │               │
          ▼               ▼               ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ HFabric  │   │   H2O    │   │   HLB    │
    │(네트워크) │   │(스토리지) │   │(로드밸런서)│
    └──────────┘   └──────────┘   └──────────┘
          │               │               │
          └───────────────┼───────────────┘
                          │
              ┌───────────┼───────────┐
              │                       │
              ▼                       ▼
        ┌──────────┐           ┌──────────┐
        │  Hubble  │           │  Sonar   │
        │(모니터링) │           │(헬스체크) │
        └──────────┘           └──────────┘
```

모든 컴퓨트 서비스(HCS, HKS, HMetal)는 HFabric 네트워크 서비스를 통해 통신하며, H2O 오브젝트 스토리지에 데이터를 저장한다. HLB 로드밸런서가 트래픽을 분산하고, Hubble과 Sonar가 전체 인프라의 상태를 모니터링한다. 이 모든 서비스는 CMP를 통해 통합 관리된다.
