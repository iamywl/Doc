# 제9장: SDV 및 커넥티드카 서비스

---

## 9.1 개요

H-Cloud는 현대자동차그룹의 소프트웨어 정의 차량(SDV, Software Defined Vehicle) 전략과 커넥티드카(Connected Car) 서비스의 핵심 백엔드 인프라로 기능한다[1][16]. 전세계 50여 개국에서 1,326만 명 이상의 가입자에게 커넥티드카 서비스를 제공하고 있으며[1], 2026년에는 2,000만 명 돌파를 목표로 하고 있다[2].

---

## 9.2 SDV (소프트웨어 정의 차량) 전략

### 9.2.1 SDV의 개념

소프트웨어 정의 차량(SDV)은 하드웨어 중심에서 소프트웨어 중심으로 차량의 설계 철학이 전환되는 패러다임을 의미한다[16][18]. SDV에서는 차량의 기능이 소프트웨어에 의해 정의되며, 출고 이후에도 OTA(Over-The-Air) 업데이트를 통해 차량의 기능을 지속적으로 개선하고 확장할 수 있다[16].

현대자동차그룹은 2022년 'Unlock the Software Age' 행사에서 SDV 개발 전략을 공식 발표하였다[19]. 이 행사에서 SDV 전환을 위한 3대 핵심 요소로 다음을 제시하였다:

1. **통합 제어기 및 차량 운영체제**: ccOS(Connected Car Operating System) 개발[48]
2. **데이터/클라우드 플랫폼**: H-Cloud 기반 데이터 처리 인프라[1]
3. **소프트웨어 인력 및 조직**: 소프트웨어 전문 인력 확충[19]

### 9.2.2 ccOS (Connected Car Operating System)

ccOS는 현대자동차그룹이 독자적으로 개발한 차량용 운영체제이다[48]. 기존에는 안드로이드 OS 기반의 차량 인포테인먼트 시스템을 사용하였으나, 기술적 자율성 확보와 차량 특화 기능 구현을 위해 리눅스 기반의 독자 운영체제를 개발하였다[48].

ccOS는 커넥티드카에서 발생하는 방대한 데이터를 신속하게 가공 및 처리하고, 안정적인 서비스를 제공할 수 있도록 설계된 소프트웨어 플랫폼이다[48]. ccOS는 3대 프레임워크로 구성된다[137]:

1. **차량 연동 프레임워크**: 차량 네트워크 및 제어 기능을 담당한다.
2. **인포테인먼트 프레임워크**: 내비게이션, 멀티미디어, 운전자 맞춤형 UX를 제공한다.
3. **커넥티비티 프레임워크**: 외부 연결 기반 데이터 처리를 담당하며, H-Cloud와의 실시간 통신을 수행한다.

### 9.2.3 HPVC (고성능 차량용 컴퓨터)

HPVC(High Performance Vehicle Computer)는 여러 하드웨어와 도메인 컨트롤러의 기능을 하나의 유닛으로 통합한 차량용 중앙 컴퓨터이다[137]. HPVC의 주요 기능은 다음과 같다:

- 내부 제어기 간 게이트웨이 역할 수행
- 차량 내외부 데이터 저장소 기능
- LTE/5G/BT/WiFi 통신을 통한 OTA 데이터 수신
- 레벨 2~4 자율주행 시스템 실행
- IVI(인포테인먼트) 스택 운영

HPVC와 Zone Controller를 연결하는 고속 네트워크 백본으로 세계 최초 자동차용 기가비트 이더넷 기술이 적용되어 있다[137].

### 9.2.4 SDV와 H-Cloud의 관계

H-Cloud는 SDV 전략에서 다음과 같은 역할을 수행한다:

- **OTA 업데이트 배포**: 소프트웨어 업데이트 패키지를 전세계 차량에 안전하게 배포한다[16].
- **차량 데이터 수집 및 분석**: 차량에서 생성되는 텔레매틱스 데이터를 수집하고 분석한다.
- **서비스 백엔드**: 커넥티드카 서비스(원격 제어, 내비게이션, 인포테인먼트 등)의 백엔드 서버를 운영한다[7][17].
- **AI/ML 학습**: 수집된 차량 데이터를 기반으로 자율주행, 배터리 관리, 예측 정비 등의 AI/ML 모델을 학습한다.

## 9.3 커넥티드카 서비스 아키텍처

### 9.3.1 ccSP (Connected Car Service Platform)

ccSP(커넥티드카 서비스 플랫폼)는 운전자에게 각종 커넥티드카 서비스를 제공하기 위한 플랫폼이다[23]. ccSP는 H-Cloud 위에서 운영되며, 차량으로부터 수집된 데이터를 기반으로 다양한 서비스를 제공한다.

### 9.3.2 서비스 구성

H-Cloud 기반의 커넥티드카 서비스는 다음과 같이 구성된다[7][17][23]:

| 서비스 분류 | 주요 기능 | 설명 |
|-----------|----------|------|
| **원격 차량 제어** | 원격 시동, 에어컨/히터 제어, 문잠금 | 모바일 앱을 통한 차량 원격 제어 |
| **OTA 업데이트** | 차량 소프트웨어 무선 업데이트 | 서비스센터 방문 없이 차량 기능 개선[16] |
| **내비게이션** | 실시간 교통 정보, 경로 안내 | 클라우드 기반 실시간 교통 데이터 처리 |
| **인포테인먼트** | 음악 스트리밍, 음성 인식 | 차량 내 엔터테인먼트 서비스 |
| **차량 진단** | 차량 상태 모니터링, 정비 알림 | 원격 차량 진단 및 예측 정비 |
| **긴급 서비스** | 사고 자동 통보, SOS 호출 | 충돌 감지 시 자동 긴급 구조 요청 |
| **차량 보안** | 도난 추적, 지오펜싱 | 차량 위치 추적 및 이상 감지 |

### 9.3.3 서비스 브랜드

현대자동차그룹의 커넥티드카 서비스는 브랜드별로 다음과 같이 운영된다:

- **현대자동차**: 블루링크(Bluelink)[49]
- **기아**: 기아 커넥트(Kia Connect)[50]
- **제네시스**: 제네시스 커넥티드 서비스[51]

각 브랜드의 서비스는 동일한 H-Cloud 인프라 위에서 운영되며, 브랜드별 차별화된 사용자 경험을 제공한다.

## 9.4 OTA (Over-The-Air) 시스템

### 9.4.1 OTA의 정의 및 중요성

OTA(Over-The-Air)는 무선 통신을 통해 차량의 소프트웨어를 원격으로 업데이트하는 기술이다[16][22]. SDV 시대에서 OTA는 차량의 기능을 출고 이후에도 지속적으로 개선하고 확장할 수 있게 하는 핵심 기술이다.

현대자동차그룹은 2025년까지 글로벌 시장에서 판매되는 모든 차종에 OTA 소프트웨어 업데이트를 기본 적용할 계획을 수립하였다[16].

### 9.4.2 OTA 시스템 아키텍처

H-Cloud 기반의 OTA 시스템은 다음과 같은 구성요소로 이루어진다:

```
┌─────────────────┐         ┌──────────────────┐         ┌─────────────────┐
│                 │         │                  │         │                 │
│  SW 개발 환경    │────────►│   H-Cloud        │────────►│   차량           │
│  (빌드/테스트)   │  업로드  │  OTA 서버         │  배포   │  (ccOS/ECU)     │
│                 │         │                  │         │                 │
└─────────────────┘         └──────────────────┘         └─────────────────┘
                                    │
                            ┌───────┴───────┐
                            │               │
                      ┌─────┴─────┐   ┌─────┴─────┐
                      │ 업데이트   │   │ 캠페인    │
                      │ 패키지 관리│   │ 관리      │
                      └───────────┘   └───────────┘
```

- **업데이트 패키지 관리**: 차량 모델별, 지역별, 버전별 소프트웨어 업데이트 패키지를 관리한다.
- **캠페인 관리**: 특정 차량 그룹에 대한 업데이트 배포 캠페인을 관리한다(대상 차량 선정, 배포 일정, 롤아웃 전략 등).
- **CDN(Content Delivery Network)**: 글로벌 리전의 CDN을 통해 대용량 업데이트 파일을 효율적으로 배포한다.
- **코드 서명**: 업데이트 패키지에 디지털 서명을 적용하여 무결성과 진위성을 보장한다.

### 9.4.3 OTA 핵심 기술

OTA 시스템의 핵심 기술 요소는 다음과 같다[136]:

- **OMA-DM 프로토콜**: OTA 센터 서버와 차량 간 통신에 OMA-DM(Open Mobile Alliance - Device Management) 프로토콜을 사용하여 소프트웨어 버전 체크 및 업데이트 관리를 수행한다.
- **메모리 이중화**: 제어기의 메모리를 이중화하여, 차량이 동작 중인 상태에서도 서비스 중단 없이 업데이트를 수행할 수 있다.
- **압축/차분 업데이트**: 전체 이미지가 아닌 변경된 부분만 전송하는 차분(Differential) 업데이트와 데이터 압축을 통해 전송 시간을 단축하고 통신비용을 절감한다.

### 9.4.4 OTA 보안

OTA 업데이트의 보안은 UN R156 규정에 따라 관리된다[22]. 주요 보안 메커니즘은 다음과 같다:

- **코드 서명(Code Signing)**: 모든 업데이트 패키지에 디지털 서명을 적용한다.
- **암호화 전송**: 업데이트 패키지의 전송 시 TLS 암호화를 적용한다.
- **무결성 검증**: 차량에서 업데이트 패키지 수신 후, 서명 검증과 해시 검증을 통해 무결성을 확인한다.
- **안전한 롤백**: 업데이트 실패 시 이전 버전으로 안전하게 롤백할 수 있는 메커니즘을 갖추고 있다.

## 9.5 엣지-클라우드 통합

### 9.5.1 엣지 클라우드 컴퓨팅

현대오토에버는 중앙 집중식 차량 아키텍처(Centralized Vehicle Architecture)를 기반으로 제어 협력 클라우드(Control Cooperation Cloud)를 운영하고 있다[17]. 이 시스템에서 차량은 엣지(Edge) 디바이스로, H-Cloud는 중앙 클라우드로 역할을 분담한다.

### 9.5.2 차량-클라우드 간 데이터 파이프라인

차량과 H-Cloud 간의 데이터 흐름은 다음과 같은 파이프라인을 통해 이루어진다[17]:

- **데이터 수집**: 차량의 각종 센서, ECU에서 생성되는 데이터를 수집한다.
- **엣지 처리**: 차량 내에서 실시간 처리가 필요한 데이터는 차량 내 엣지 컴퓨팅 환경에서 처리한다.
- **클라우드 전송**: 분석 및 학습이 필요한 데이터는 H-Cloud로 전송한다.
- **데이터 레이크(Data Lake)**: H-Cloud의 데이터 레이크에 차량 데이터를 수집하고 저장한다[17].
- **ML 학습**: 수집된 데이터를 기반으로 머신러닝 모델을 학습한다.
- **모델 배포**: 학습된 모델을 OTA를 통해 차량에 배포한다.

### 9.5.3 실시간 스트리밍

차량과 H-Cloud 간에는 OTA 업데이트뿐만 아니라 실시간 스트리밍 통신이 이루어진다[17]. 실시간 교통 정보, 원격 차량 제어 명령, 텔레매틱스 데이터 등이 실시간으로 양방향 전송된다.

## 9.6 커넥티드카 서비스의 기술적 과제

H-Cloud 기반 커넥티드카 서비스는 다음과 같은 기술적 과제에 대응하고 있다:

### 9.6.1 대규모 동시 접속

전세계 1,326만 대 이상의 차량이 동시에 H-Cloud에 접속할 수 있으며, 특정 시간대(출퇴근 시간) 또는 특정 계절(혹한/혹서기 원격 에어컨/히터 제어)에는 트래픽이 폭발적으로 증가한다[1]. 이에 대응하기 위해 MSA 기반의 비동기 통신과 쿠버네티스 자동 스케일링을 활용하고 있다[4].

### 9.6.2 글로벌 저지연 서비스

전세계 50여 개국에서 서비스를 제공하므로, 지리적으로 분산된 사용자에게 일관된 응답 시간을 제공해야 한다. 이를 위해 글로벌 4개 리전에 인프라를 분산 배치하고, 에퀴닉스 Fabric을 통한 저지연 네트워크를 구축하고 있다[1][6].

### 9.6.3 데이터 주권

EU GDPR 등 지역별 데이터 보호 규정에 따라, 해당 지역에서 생성된 데이터는 해당 지역의 리전에서 처리되어야 한다. 멀티 리전 아키텍처를 통해 이러한 데이터 주권 요구사항을 충족하고 있다.

## 9.7 현대 카 클라우드의 역사

현대자동차그룹의 클라우드 기반 차량 서비스의 역사는 H-Cloud 이전으로 거슬러 올라간다[23]:

| 시기 | 내용 |
|------|------|
| 2013년 | '현대 카 클라우드(Hyundai Car Cloud)' 국내 구축 시작 |
| 2017년 | 중국 구이저우성에 첫 번째 글로벌 빅데이터 센터 구축 |
| 2020년대 초 | H-Cloud 프라이빗 클라우드 플랫폼 개발 착수 |
| 2024년 | H-Cloud 공식 공개, 1,326만 가입자 돌파 |
| 2026년 목표 | 2,000만 가입자 달성 |

2013년 '현대 카 클라우드' 구축을 시작으로, 커넥티드카 서비스를 위한 클라우드 인프라를 지속적으로 발전시켜 왔으며, 그 결정체가 바로 H-Cloud이다[23].
