# 제7장: 글로벌 인프라 구조

---

## 7.1 개요

H-Cloud는 전세계 4개 주요 리전에서 운영되는 글로벌 인프라를 구축하고 있다[1][7]. 각 리전은 에퀴닉스(Equinix) 데이터센터를 중심으로 운영되며, Equinix Fabric을 통해 퍼블릭 클라우드와 연계된 하이브리드 멀티클라우드 아키텍처를 구현하고 있다[6]. 서비스 노드들이 하나의 데이터센터에 집중되지 않고 글로벌로 분산 배치되어 있어, 높은 가용성과 낮은 응답 지연시간을 확보하고 있다[1].

---

## 7.2 리전(Region) 구성

### 7.2.1 한국 리전

| 항목 | 내용 |
|------|------|
| 위치 | 서울, 대한민국 |
| 데이터센터 | 에퀴닉스 서울 데이터센터[6] |
| 역할 | 본사 리전, 국내 커넥티드카 서비스(약 580만 가입자) 지원[1] |
| 비고 | 개발/스테이징 환경 및 글로벌 관제 센터 운영 |

한국 리전은 H-Cloud의 본사 리전으로서, 국내 커넥티드카 서비스의 백엔드 인프라와 현대자동차그룹 내부 업무 시스템을 운영한다[1]. 또한 글로벌 인프라 표준의 정의, 제품 개발, 품질 검증이 이루어지는 중심 리전이다[1].

### 7.2.2 아시아태평양 리전

| 항목 | 내용 |
|------|------|
| 위치 | 싱가포르 |
| 역할 | APAC 지역 서비스 거점[7] |
| 비고 | 호주, 인도네시아 등 APAC 지역 서비스 확장 거점 |

아시아태평양 리전은 싱가포르에 위치하며, 동남아시아, 호주 등 APAC 지역의 커넥티드카 서비스를 지원한다[7].

### 7.2.3 북미 리전

| 항목 | 내용 |
|------|------|
| 위치 | 로스앤젤레스, 미국 |
| 데이터센터 | 에퀴닉스 로스앤젤레스 데이터센터[6] |
| 역할 | 북미 서비스 거점[6][7] |
| 비고 | 현대자동차 및 기아 북미 커넥티드카 서비스 지원 |

북미 리전은 현대자동차와 기아의 북미 시장 커넥티드카 서비스를 지원한다. 에퀴닉스 로스앤젤레스 데이터센터에 인프라가 배치되어 있으며, Equinix Fabric을 통해 AWS 등 퍼블릭 클라우드와 연결되어 있다[6].

### 7.2.4 유럽 리전

| 항목 | 내용 |
|------|------|
| 위치 | 프랑크푸르트, 독일 |
| 데이터센터 | 에퀴닉스 프랑크푸르트 데이터센터[6] |
| 역할 | 유럽 서비스 거점[6][7] |
| 비고 | EU GDPR 데이터 주권 규정 준수, 유럽 커넥티드카 서비스 지원 |

유럽 리전은 EU의 GDPR(General Data Protection Regulation) 등 데이터 주권 규정을 준수하기 위해 프랑크푸르트에 위치하고 있다[6]. 현대자동차와 기아의 유럽 시장 커넥티드카 서비스를 지원한다.

### 7.2.5 국내 자체 데이터센터 (IDC)

에퀴닉스 코로케이션 외에도, 현대오토에버는 국내에 자체 데이터센터(IDC)를 운영하고 있다[130]. 국내 IDC 현황은 다음과 같다:

| 위치 | 용량 | 비고 |
|------|------|------|
| 파주 IDC | 14MW | 주요 운영 데이터센터[130] |
| 의왕 IDC | 6MW | 그룹사 업무 시스템 운영[130] |
| 광주 IDC | 10MW | 재해복구(DR) 및 분산 처리[130] |

이들 국내 IDC는 그룹사 내부 업무 시스템, ERP, 스마트팩토리 등 데이터 주권이 중요한 워크로드를 처리하며, 에퀴닉스 글로벌 리전과 상호 보완적인 역할을 수행한다.

### 7.2.6 확장 계획

H-Cloud는 호주, 인도네시아 등으로 서비스 거점을 지속 확장하고 있다[7]. 커넥티드카 서비스 가입자 수 증가와 신차 출시에 따라 새로운 리전이 추가될 전망이다.

## 7.3 에퀴닉스(Equinix)와의 파트너십

### 7.3.1 에퀴닉스 소개

에퀴닉스(Equinix)는 전세계 72개 이상의 주요 도시에 260개 이상의 데이터센터를 운영하는 세계 최대의 코로케이션 데이터센터 기업이다[36]. 에퀴닉스 데이터센터는 주요 클라우드 서비스 제공업체(CSP), 네트워크 사업자, 엔터프라이즈 고객이 밀집해 있어, 저지연 상호연결이 가능한 것이 핵심 강점이다.

### 7.3.2 코로케이션 서비스

에퀴닉스는 서울, 로스앤젤레스, 프랑크푸르트에 위치한 데이터센터에 H-Cloud 인프라의 물리적 서버를 호스팅한다[6]. 이를 통해 현대오토에버는 자체 데이터센터를 건설하는 대신, 에퀴닉스의 검증된 시설(전력, 냉각, 물리적 보안, 네트워크 연결)을 활용하여 글로벌 인프라를 신속하게 구축할 수 있었다.

### 7.3.3 Equinix Fabric

Equinix Fabric은 에퀴닉스가 제공하는 소프트웨어 정의 상호연결(Software-Defined Interconnection) 서비스이다[36]. H-Cloud에서 Equinix Fabric을 활용하는 주요 목적은 다음과 같다:

- **퍼블릭 클라우드 연결**: H-Cloud 프라이빗 인프라와 AWS 등 퍼블릭 클라우드 서비스를 프라이빗 네트워크로 연결한다[6]. 공용 인터넷을 경유하지 않는 전용 연결을 통해, 높은 보안성과 안정적인 대역폭을 확보한다.
- **저지연 네트워크**: 에퀴닉스 데이터센터가 주요 CSP와 인접해 있어, 밀리초(ms) 단위의 저지연 네트워크 연결이 가능하다[6].
- **글로벌 상호연결**: 에퀴닉스의 글로벌 네트워크를 통해 리전 간 상호연결이 가능하다.

### 7.3.4 파트너십의 효과

에퀴닉스와의 파트너십을 통해 다음과 같은 효과를 달성하였다[6]:

- **서비스 지연 감소**: 퍼블릭 클라우드와의 저지연 프라이빗 연결을 통해 서비스 응답 시간이 개선되었다.
- **안정성 향상**: 전용 네트워크 연결을 통해 공용 인터넷의 불안정성에 영향받지 않는 안정적인 통신이 가능해졌다.
- **높은 확장성**: 에퀴닉스의 글로벌 네트워크를 활용하여, 새로운 리전 확장 시 인프라 구축 시간을 단축할 수 있다.

## 7.4 글로벌 배포 전략

### 7.4.1 표준화된 배포 프로세스

H-Cloud의 글로벌 배포 전략은 다음과 같은 4단계 프로세스를 따른다[1]:

1. **글로벌 인프라 표준 정의**: 모든 리전에서 일관된 서비스를 제공하기 위한 인프라 표준을 정의한다.
2. **자체(In-House) 개발**: 정의된 표준에 맞춰 H-Cloud 제품을 자체 개발한다.
3. **품질 및 성능 검증**: 개발된 제품의 품질과 성능을 한국 리전에서 먼저 검증한다.
4. **글로벌 배포**: 검증이 완료된 제품을 전세계 리전에 배포한다.

이러한 표준화된 프로세스를 통해, 어느 리전에서든 동일한 수준의 서비스 품질을 보장할 수 있다.

### 7.4.2 Rancher Prime 기반 통합 관리

Rancher Prime의 멀티 클러스터 관리 기능을 활용하여, 모든 리전의 쿠버네티스 클러스터를 단일 콘솔에서 관리한다[7]. 이를 통해 다음과 같은 글로벌 운영 효율성을 확보하였다:

- **중앙 집중 관리**: 모든 리전의 클러스터 상태, 배포 상태, 자원 사용률을 단일 대시보드에서 확인할 수 있다.
- **일괄 정책 적용**: 보안 정책, 네트워크 정책 등을 모든 리전에 일괄 적용할 수 있다.
- **소규모 인력 운영**: 중앙 집중 관리를 통해 제한된 인력으로도 글로벌 인프라를 효율적으로 운영할 수 있다[7].

## 7.5 네트워크 아키텍처

### 7.5.1 글로벌 네트워크 토폴로지

H-Cloud의 글로벌 네트워크는 다음과 같은 토폴로지로 구성되어 있다:

```
                    ┌─────────────────┐
                    │   관제 센터      │
                    │   (한국 리전)    │
                    └────────┬────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
   ┌────┴────┐         ┌────┴────┐         ┌────┴────┐
   │  APAC   │         │   북미   │         │  유럽   │
   │(싱가포르)│         │  (LA)   │         │(프랑크) │
   └────┬────┘         └────┬────┘         └────┬────┘
        │                    │                    │
   ┌────┴────┐         ┌────┴────┐         ┌────┴────┐
   │ Equinix │         │ Equinix │         │ Equinix │
   │ Fabric  │         │ Fabric  │         │ Fabric  │
   └────┬────┘         └────┬────┘         └────┬────┘
        │                    │                    │
   ┌────┴────┐         ┌────┴────┐         ┌────┴────┐
   │ AWS 등  │         │ AWS 등  │         │ AWS 등  │
   │ 퍼블릭  │         │ 퍼블릭  │         │ 퍼블릭  │
   └─────────┘         └─────────┘         └─────────┘
```

각 리전은 에퀴닉스 데이터센터를 중심으로 H-Cloud 프라이빗 인프라와 퍼블릭 클라우드가 Equinix Fabric으로 연결되어 있다. 리전 간에는 전용 네트워크를 통해 상호 연결되어 있으며, 한국 리전의 관제 센터에서 글로벌 인프라를 통합 모니터링한다.

### 7.5.2 네트워크 보안

글로벌 네트워크의 보안은 다음과 같은 다계층 방어 전략을 적용하고 있다:

- **물리적 보안**: 에퀴닉스 데이터센터의 물리적 접근 통제(생체인증, 감시 카메라 등)를 활용한다[36].
- **네트워크 보안**: 방화벽, IDS/IPS, DDoS 방어 등의 네트워크 보안 장비를 운영한다.
- **전용 연결**: 퍼블릭 클라우드와의 연결에 Equinix Fabric의 프라이빗 연결을 사용하여, 공용 인터넷을 경유하지 않는다[6].
- **암호화**: 리전 간 통신 및 데이터 전송 시 암호화를 적용한다.

## 7.6 재해 복구(DR) 전략

글로벌 멀티 리전 아키텍처를 통해 다음과 같은 재해 복구 전략을 구현하고 있다:

- **지리적 이중화**: 서비스 노드가 여러 리전에 분산되어 있어, 특정 리전의 장애 시에도 다른 리전에서 서비스를 계속할 수 있다[1].
- **데이터 복제**: 핵심 데이터는 리전 간 복제를 통해 데이터 유실을 방지한다.
- **자동 장애 조치(Failover)**: 장애 발생 시 트래픽을 정상 리전으로 자동 전환하는 장애 조치 메커니즘을 운영한다.
