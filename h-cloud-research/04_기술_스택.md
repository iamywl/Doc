# 제4장: 기술 스택

---

## 4.1 개요

H-Cloud의 기술 스택은 하드웨어 인프라, 운영체제, 가상화/컨테이너, 오케스트레이션, 개발/배포, 모니터링, 보안 등 여러 계층에 걸쳐 구성되어 있다. 전체적으로 오픈소스 기반 기술을 자체 환경에 맞게 커스터마이징하여 사용하고 있으며[1][3], 상용 제품은 SUSE와의 파트너십을 통해 엔터프라이즈급 지원을 확보하고 있다[7][28].

---

## 4.2 하드웨어 인프라

### 4.2.1 서버 하드웨어

H-Cloud의 물리적 서버 인프라는 Dell Technologies의 서버 하드웨어를 기반으로 구성되어 있다[7]. Dell PowerEdge 시리즈 서버를 채택하여, 높은 신뢰성과 성능을 확보하고 있다.

### 4.2.2 프로세서

서버에는 Intel 프로세서가 탑재되어 있다[7]. Intel Xeon Scalable 프로세서 계열을 사용하여, 대규모 가상화 및 컨테이너 워크로드를 처리하기 위한 충분한 컴퓨팅 성능을 제공한다.

### 4.2.3 GPU 인프라

AI/ML 워크로드를 위한 GPU 인프라 투자가 확대되고 있다[8]. AI GPU 인프라 투자와 커넥티드카 서비스 인프라 투자에 힘입어 클라우드 매출이 성장하고 있으며, 2028년까지 5,000억 원 규모로 확대될 전망이다[8].

### 4.2.4 데이터센터

글로벌 데이터센터 인프라는 에퀴닉스(Equinix)의 코로케이션 서비스를 통해 운영된다[6]. 서울, 로스앤젤레스, 프랑크푸르트에 위치한 에퀴닉스 데이터센터에 H-Cloud 인프라가 배치되어 있다[6]. 이와 별도로 국내에 파주(14MW), 의왕(6MW), 광주(10MW) 등 자체 IDC를 운영하여 그룹사 업무 시스템 및 데이터 주권이 필요한 워크로드를 처리하고 있다[130].

### 4.2.5 IaaS 스토리지

H-Cloud의 IaaS 계층에서는 다음과 같은 스토리지 서비스를 제공한다[131]:

- **블록 스토리지**: 10GB 단위로 프로비저닝 가능한 블록 스토리지를 제공하며, VM(가상 머신)에 직접 연결하여 사용한다.
- **NAS 공유 스토리지**: 다수의 인스턴스가 동시에 접근할 수 있는 네트워크 연결 공유 스토리지를 제공한다.
- **카탈로그 기반 프로비저닝**: 사용자가 웹 포털에서 카탈로그 방식으로 컴퓨팅, 스토리지, 네트워크 자원을 선택하여 즉시 프로비저닝할 수 있는 셀프 서비스 환경을 제공한다.

## 4.3 운영체제

### 4.3.1 SUSE Linux Enterprise Server (SLES)

H-Cloud의 기반 운영체제로 SUSE Linux Enterprise Server(SLES)를 사용하고 있다[7][24]. SLES는 엔터프라이즈급 리눅스 운영체제로, 높은 안정성과 보안성을 제공한다.

SLES를 선택한 주요 이유는 다음과 같다:

- **엔터프라이즈 지원**: SUSE로부터 장기 지원(Long-Term Support)을 받을 수 있다[7].
- **SAP 인증**: SLES for SAP Applications는 SAP에 의해 공식 인증된 운영체제로, 현대제철 등 계열사의 SAP ERP 운영에 적합하다[24].
- **보안**: 커널 수준의 보안 패치와 취약점 관리를 SUSE가 적시에 제공한다.
- **Rancher 통합**: 동일한 SUSE 생태계 내에서 Rancher Prime과의 원활한 통합이 가능하다[7][28].

## 4.4 컨테이너 및 오케스트레이션

### 4.4.1 Kubernetes

H-Cloud의 컨테이너 오케스트레이션 플랫폼으로 쿠버네티스(Kubernetes)를 사용한다[4][5]. 쿠버네티스는 Google이 개발하고 CNCF가 관리하는 오픈소스 컨테이너 오케스트레이션 시스템으로, 컨테이너화된 애플리케이션의 배포, 확장, 관리를 자동화한다[27][29].

### 4.4.2 SUSE Rancher Prime

쿠버네티스 클러스터의 통합 관리를 위해 SUSE Rancher Prime을 사용한다[4][5][7]. Rancher Prime은 SUSE가 제공하는 엔터프라이즈급 쿠버네티스 관리 플랫폼으로, 멀티 클러스터 환경에서의 클러스터 생성, 배포, 모니터링, 보안을 통합적으로 관리한다.

현대오토에버는 2022년부터 SUSE와 협업을 시작하여 HKS에 Rancher Prime을 활용하고 있다[4][5]. Rancher Prime을 선택한 주요 이유는 다음과 같다:

- **멀티 클러스터 관리**: 글로벌 4개 리전에 분산된 다수의 클러스터를 단일 인터페이스에서 관리할 수 있다[7].
- **API 통합**: Rancher API를 활용한 커스텀 포털 통합이 가능하여, HKS에 맞춘 사용자 경험을 제공할 수 있다[7].
- **오픈소스 기반**: 오픈소스를 기반으로 하여 내부 환경에 맞는 커스터마이징이 가능하다[1].
- **엔터프라이즈 지원**: SUSE로부터 제품 지원, 컨설팅, 교육, 매니지드 서비스를 제공받을 수 있다[7].

### 4.4.3 컨테이너 런타임

쿠버네티스의 컨테이너 런타임으로는 containerd를 사용하는 것으로 추정된다. 이는 쿠버네티스 1.24 이후 Docker 런타임 지원이 중단(deprecation)됨에 따라, 업계 표준인 containerd로의 전환이 이루어진 것으로 보인다[30].

## 4.5 개발 및 배포 도구

### 4.5.1 GitLab

소스 코드의 버전 관리 및 협업 플랫폼으로 GitLab을 사용한다[4][5]. GitLab은 Git 기반의 소스 코드 관리뿐만 아니라, 이슈 트래킹, 코드 리뷰, CI/CD 파이프라인 등 소프트웨어 개발 라이프사이클 전반을 지원하는 DevOps 플랫폼이다[31].

### 4.5.2 Harbor

컨테이너 이미지의 저장 및 관리를 위해 Harbor를 사용한다[4][5]. Harbor는 CNCF 졸업(Graduated) 프로젝트로, 엔터프라이즈급 컨테이너 레지스트리이다[32]. 프라이빗 클라우드 환경에서 컨테이너 이미지의 보안 스캔, 접근 제어, 이미지 서명 등의 기능을 제공한다.

### 4.5.3 ArgoCD

GitOps 기반의 지속적 배포(Continuous Deployment) 도구로 ArgoCD를 사용한다[4][5]. ArgoCD는 CNCF 졸업 프로젝트로, Git 저장소에 정의된 애플리케이션의 원하는 상태(Desired State)를 쿠버네티스 클러스터에 자동으로 동기화한다[33].

GitOps 패러다임을 채택함으로써 다음과 같은 장점을 확보하였다:

- **선언적 배포**: 애플리케이션의 배포 상태를 Git 저장소에 선언적으로 정의한다.
- **감사 추적(Audit Trail)**: 모든 배포 변경사항이 Git 커밋으로 기록되어, 완전한 감사 추적이 가능하다.
- **자동 동기화**: Git 저장소의 변경사항이 자동으로 클러스터에 반영된다.
- **롤백 용이성**: 이전 Git 커밋으로 되돌림으로써 간단하게 롤백할 수 있다.

## 4.6 코드 품질 및 보안 도구

### 4.6.1 SonarQube

정적 코드 분석 및 코드 품질 관리 도구로 SonarQube를 사용한다[4][5]. SonarQube는 소스 코드의 버그, 코드 스멜(Code Smell), 보안 취약점을 자동으로 탐지하여 코드 품질을 관리한다[34].

CI/CD 파이프라인에 통합되어, 코드가 커밋될 때마다 자동으로 정적 분석이 수행된다. 분석 결과에 따라 품질 게이트(Quality Gate)를 통과하지 못한 코드는 배포가 차단되어, 코드 품질의 최소 기준을 보장한다.

### 4.6.2 Black Duck

오픈소스 구성요소의 라이선스 준수 여부 및 보안 취약점을 점검하기 위해 Black Duck을 사용한다[4][5]. Black Duck은 Synopsys가 제공하는 오픈소스 보안 및 관리 솔루션으로, 소프트웨어 구성 분석(SCA, Software Composition Analysis) 기능을 제공한다[35].

Black Duck의 주요 기능은 다음과 같다:

- **오픈소스 식별**: 프로젝트에 포함된 오픈소스 구성요소를 자동으로 식별한다.
- **취약점 점검**: 식별된 오픈소스 구성요소에 대해 알려진 취약점(CVE)을 점검한다.
- **라이선스 준수**: 오픈소스 라이선스 의무사항 준수 여부를 확인한다.
- **정책 관리**: 조직의 오픈소스 사용 정책에 따른 자동화된 관리가 가능하다.

## 4.7 네트워크 및 상호연결

### 4.7.1 Equinix Fabric

H-Cloud와 퍼블릭 클라우드 서비스 간의 상호연결을 위해 Equinix Fabric을 사용한다[6]. Equinix Fabric은 에퀴닉스가 제공하는 소프트웨어 정의 상호연결(Software-Defined Interconnection) 서비스로, 에퀴닉스 데이터센터 내에서 다양한 클라우드 서비스 제공업체와의 프라이빗 네트워크 연결을 구성할 수 있다[36].

### 4.7.2 SDN (Software Defined Networking)

H-Cloud 내부 네트워크는 SDN(Software Defined Networking) 기술을 기반으로 구성되어 있다. SDN은 네트워크의 제어 평면(Control Plane)과 데이터 평면(Data Plane)을 분리하여, 소프트웨어를 통해 네트워크를 중앙 집중적으로 관리할 수 있는 기술이다[37].

| **IaaS** | 블록 스토리지 | 자체 개발 | 10GB 단위 프로비저닝[131] |
| | NAS 공유 스토리지 | 자체 개발 | 네트워크 연결 스토리지[131] |
| **운영체제** | SUSE Linux Enterprise Server | 상용(오픈소스) | 엔터프라이즈 리눅스[7][24] |
