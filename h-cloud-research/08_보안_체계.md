# 제8장: 보안 체계

---

## 8.1 개요

현대오토에버는 자동차 산업의 사이버보안 요구사항 강화에 대응하여, H-Cloud 및 차량 전반에 걸친 종합적인 보안 체계를 구축하고 있다[15][21]. 기존 정보보안센터를 '사이버시큐리티 사업부'로 격상하여 출범시켰으며, 인력 규모는 200여 명에 달한다[15]. 보안은 DevSecOps 파이프라인, 인프라 보안, 차량 사이버보안의 3개 축으로 운영된다.

---

## 8.2 사이버시큐리티 사업부

### 8.2.1 조직 구성

현대오토에버의 사이버시큐리티 사업부는 다음과 같은 조직으로 구성되어 있다[15]:

- **서비스 보안실**: 모의해킹 훈련 시 공격을 담당하는 레드(Red)팀과 방어를 담당하는 블루(Blue)팀을 한데 모아 시너지를 강화하고 있다.
- **차량 보안 조직**: 차량 사이버보안 관리 체계(CSMS) 운영 및 차량 보안 인증을 담당한다.
- **인프라 보안 조직**: H-Cloud 및 그룹 IT 인프라의 보안을 담당한다.

### 8.2.2 보안 역량

현대오토에버는 다음과 같은 보안 역량을 보유하고 있다[15][21]:

- **200여 명의 전문 보안 인력**을 보유하고 있다[15].
- **Red/Blue 팀 운영**: 공격(Red)팀과 방어(Blue)팀을 모두 운영하여, 실전적인 보안 능력을 갖추고 있다[15].
- **국제 인증 획득**: TUV 라인란드로부터 ISO/SAE 21434 기반 CSMS 레벨3 인증을 국내 최초로 획득하였다[21].

---

## 8.3 차량 사이버보안

### 8.3.1 CSMS (사이버보안 관리 체계)

CSMS(Cyber Security Management System)는 차량의 설계부터 양산, 단종에 이르기까지 전체 라이프 사이클 동안 위험 요소를 지속적으로 관리하기 위한 체계이다[21][22]. 이는 UN R155 규정에 따라 2022년 7월부터 신형 차종에 의무화되었으며, 2024년 7월부터는 모든 차종에 적용되고 있다[46].

현대오토에버는 TUV 라인란드(TUV Rheinland)로부터 ISO/SAE 21434 국제표준에 기반한 CSMS 레벨3 인증을 획득하였다[21]. 레벨3은 최고 수준의 인증으로, 차량 제조사 및 부품 공급업체가 사이버보안을 보장하기 위한 객관적인 프로세스를 갖추고 있음을 증명하는 것이다.

### 8.3.2 차량 보안 기술 상세

현대오토에버는 차량 전자 시스템에 대한 다층 보안 기술을 자체 개발하여 적용하고 있다[132]:

| 보안 기술 | 기능 |
|----------|------|
| **방화벽** | 차량 외부 통신 점검, 비인가 트래픽 차단 |
| **시큐어 액세스** | ECU 접근 사용자 식별 및 인증 |
| **어드밴스드 시드/키** | 군사급 상호 인증 프로토콜 |
| **시큐어 플래시** | 펌웨어 변조 방지 |
| **HSM(Hardware Security Module)** | 부트로더/펌웨어 보호, 보안키 관리 |
| **CAN IDS** | 차량 내 네트워크 감시, 공격 탐지, 보안관제센터 알림[21][47] |
| **화이트박스 암호화(WBC)** | 암호 알고리즘과 키 결합으로 역분석 불가능 |
| **보안 부팅(Secure Boot)** | 부팅 시 소프트웨어 무결성 검증 |
| **차량용 보안 암호 라이브러리** | AES, TDES, SHA, RSA, ECC (PC 대비 메모리 80% 절감) |

특히 CAN IDS(Intrusion Detection System)는 차량 내 제어기 및 네트워크의 활동을 감시하고, 공격자의 침입을 탐지하여 보안관제센터(SOC)로 알림을 제공하는 핵심 시스템이다[21][47]. 또한 MPASS 통합인증플랫폼을 통해 OTP, FIDO 1.0/2.0, 생체인증을 통합 제공하고 있다[132].

### 8.3.3 차량-클라우드 간 보안

H-Cloud와 차량 간의 통신에서는 다음과 같은 보안 메커니즘이 적용된다:

- **TLS 암호화**: 차량과 클라우드 간 모든 통신에 TLS(Transport Layer Security) 암호화를 적용한다.
- **인증 및 인가**: 차량의 고유 식별자를 기반으로 인증(Authentication)과 인가(Authorization)를 수행한다.
- **OTA 보안**: OTA 소프트웨어 업데이트 시 코드 서명(Code Signing)을 통해 업데이트 패키지의 무결성과 진위성을 검증한다.
- **데이터 주권**: EU GDPR 등 지역별 데이터 보호 규정에 따라 데이터가 해당 지역의 리전에서 처리되도록 한다.

### 8.3.4 부품 협력사 보안

현대오토에버는 자동차 부품 협력사의 공장 보안 강화에도 보안 솔루션을 적용하고 있다[25]. 공급망(Supply Chain) 전체의 사이버보안 수준을 향상시키기 위해, 협력사의 보안 역량 강화를 지원한다.

## 8.4 인프라 보안

### 8.4.1 네트워크 보안

H-Cloud 인프라의 네트워크 보안은 다계층 방어(Defense in Depth) 전략을 따른다:

- **방화벽(Firewall)**: 네트워크 경계에서 허용되지 않은 트래픽을 차단한다.
- **IDS/IPS**: 네트워크 트래픽을 실시간으로 분석하여 침입 시도를 탐지하고 차단한다.
- **DDoS 방어**: 분산 서비스 거부(DDoS) 공격에 대한 방어 체계를 갖추고 있다.
- **네트워크 세분화(Microsegmentation)**: 쿠버네티스 네트워크 정책(Network Policy)을 통해 Pod 간 통신을 세밀하게 제어한다.
- **프라이빗 연결**: Equinix Fabric을 통한 프라이빗 네트워크 연결로, 공용 인터넷을 경유하지 않는 보안 통신을 구현한다[6].

### 8.4.2 접근 제어

- **RBAC(Role-Based Access Control)**: 쿠버네티스의 RBAC을 활용하여 역할 기반 접근 제어를 구현한다.
- **최소 권한 원칙(Principle of Least Privilege)**: 사용자와 서비스에게 업무 수행에 필요한 최소한의 권한만 부여한다.
- **다중 인증(MFA)**: 관리 콘솔 접근 시 다중 인증을 요구한다.

### 8.4.3 데이터 보안

- **저장 데이터 암호화(Encryption at Rest)**: 저장된 데이터에 대한 암호화를 적용한다.
- **전송 데이터 암호화(Encryption in Transit)**: 네트워크를 통해 전송되는 데이터에 대한 TLS 암호화를 적용한다.
- **비밀 관리(Secret Management)**: 쿠버네티스 Secret 또는 별도의 비밀 관리 시스템을 통해 민감한 정보(API 키, 비밀번호, 인증서 등)를 안전하게 관리한다.

## 8.5 DevSecOps 파이프라인 보안

CI/CD 파이프라인에 보안을 내재화한 DevSecOps 체계에 대해서는 제6장에서 상세히 다루었다. 핵심 요소를 요약하면 다음과 같다[4][5]:

- **SAST**: SonarQube를 통한 정적 보안 분석[34]
- **SCA**: Black Duck을 통한 오픈소스 구성요소 보안 분석[35]
- **컨테이너 이미지 스캔**: Harbor를 통한 컨테이너 이미지 보안 취약점 스캔[32]
- **Shift-Left**: 보안 검증을 개발 초기 단계로 이동[44]

## 8.6 컴플라이언스 및 인증

현대오토에버가 취득한 주요 보안 관련 인증 및 준수하고 있는 규정은 다음과 같다:

| 인증/규정 | 내용 | 비고 |
|----------|------|------|
| ISO/SAE 21434 CSMS Level 3 | 자동차 사이버보안 관리 체계 | 국내 최초 레벨3, 11개 전 영역 인증[21][133] |
| ASIL-D | 기능안전 최고 등급 인증 | ISO 26262 기반[132] |
| ASPICE CL1 | 자동차 소프트웨어 프로세스 인증 | 개발 프로세스 품질 인증[132] |
| UN R155 | 차량 사이버보안 규정 | 2024년 7월부터 56개 협약국 모든 차종 의무화[46] |
| UN R156 | 소프트웨어 업데이트 관리 규정 | OTA 업데이트 보안[22] |
| EU GDPR | 유럽 개인정보보호 규정 | 유럽 리전 데이터 주권 준수 |
| 개인정보보호법 | 한국 개인정보보호 법률 | 국내 고객 데이터 보호 |
