## 1. 문제 인식 (Problem)

### 1.1 알고리즘 학습의 구조적 어려움

알고리즘과 자료구조는 컴퓨터 과학의 핵심 과목이지만, 추상적 개념을 텍스트만으로 이해하기 어렵다는 근본적 한계가 있다. 한국정보화진흥원 조사에 따르면, CS 전공생의 68%가 알고리즘 과목에서 학습 장벽을 경험하며, 그 주요 원인은 "코드의 동작 과정을 머릿속으로 시뮬레이션하기 어렵다"는 것이다 [1].

특히 재귀(recursion), 동적 프로그래밍(DP), 그래프 탐색 같은 고급 알고리즘일수록 코드와 실행 흐름 사이의 괴리가 커지며, 학생들은 코드를 작성할 수 있어도 "왜 그렇게 동작하는지"를 직관적으로 이해하지 못하는 경우가 많다. 이 문제는 단순히 텍스트 설명을 늘리는 것으로는 해결되지 않으며, 시각적·인터랙티브 도구의 도입이 필수적이다.

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 알고리즘 과목 학습 장벽 경험 비율 | **68%** | 한국정보화진흥원, 2024 [1] |
| 알고리즘 과목 F학점 및 재수강 비율 | **22%** | 한국대학교육협의회, 2024 [2] |
| 코딩테스트 합격률 (대기업 기준) | **15~20%** | 잡코리아, 2025 [5] |
| 시각적 학습 도구 사용 시 이해도 향상 | **40~60%** | Journal of Computer Science Education, 2024 [6] |
| 국내 연간 SW 개발자 채용 규모 | **15만 명** | 과학기술정보통신부, 2025 [7] |
| 코딩 부트캠프 수강생 수 (국내, 연간) | **5만 명** | 한국소프트웨어산업협회, 2025 [8] |
| CS 전공생 알고리즘 과외/학원 이용률 | **35%** | 대학내일, 2025 [9] |
| 알고리즘 학습 시 가장 어려운 주제 (DP) | **47%** | Stack Overflow Survey Korea, 2025 [10] |

#### 학습자 목소리 (사용자 인터뷰 요약)

> "교과서에서 퀵소트 코드를 읽고 이해했다고 생각했는데, 실제로 배열이 어떻게 분할되는지 그림을 그려보면 전혀 다르게 동작해서 혼란스러웠다." — CS 전공 3학년 A씨

> "DP 테이블을 채워가는 과정을 머릿속에서 시뮬레이션하는 것이 너무 어려워서, 결국 풀이를 외워버리게 된다." — 코딩테스트 준비생 B씨

> "시각화 도구가 있으면 좋겠다고 항상 생각했지만, 기존 도구들은 한국어가 안 되거나 제가 짠 코드와 연동이 안 되어서 포기했다." — 부트캠프 수강생 C씨

### 1.2 기존 학습 도구의 한계

현재 시장에 존재하는 알고리즘 학습 도구들은 다음과 같은 한계를 보인다:

- **VisuAlgo**: 시각화는 우수하나 코드 실행과 연동되지 않으며, 한국어 미지원. 학습자가 자신의 코드를 넣어 테스트하는 기능이 없어 능동적 학습이 제한됨
- **Python Tutor**: 코드 실행 시각화는 가능하나 알고리즘 특화 애니메이션 부재. 변수 값 추적 외에 자료구조의 형태 변화를 시각적으로 보여주지 못함
- **LeetCode**: 문제 풀이 중심으로 학습 과정 시각화 기능 부족. 결과(정답/오답)만 피드백하며, 풀이 과정에서의 학습 지원이 미비
- **백준 Online Judge**: 국내 사용자 기반이 크지만, 순수 문제 풀이 환경으로 알고리즘 동작 원리 학습에는 적합하지 않음
- **알고리즘 교재/강의**: 정적인 그림과 의사코드 중심으로, 동적인 알고리즘 동작 과정을 직관적으로 전달하기 어려움

### 1.3 페인포인트 분석

| 페인포인트 | 영향도 | 기존 해결책 | 한계 |
|-----------|--------|-----------|------|
| 알고리즘 동작 과정 이해 어려움 | 매우 높음 | 교재 그림, VisuAlgo | 정적, 코드 비연동 |
| 자신의 코드 디버깅 어려움 | 높음 | 디버거(GDB 등) | 시각화 부재, 학습 곡선 |
| 코딩테스트 실전 감각 부족 | 높음 | LeetCode, 백준 | 풀이 과정 피드백 부족 |
| 한국어 학습 자료 부족 | 중간 | 국내 강의/블로그 | 체계적이지 않음 |
| 학습 진도 관리 어려움 | 중간 | 수기 관리 | 자동화/분석 부재 |

### 2.2 핵심 기능

1. **인터랙티브 알고리즘 애니메이션**: 정렬(버블/퀵/머지 등), 그래프(BFS/DFS/다익스트라), 트리(BST/AVL/힙) 등 50개 이상 알고리즘의 단계별 애니메이션. 속도 조절, 되감기, 특정 단계 점프 기능 제공
2. **코드 실행 동기화 시각화**: 사용자가 작성한 코드의 각 라인 실행과 시각화가 실시간 동기화. 현재 실행 중인 코드 라인이 하이라이트되며, 해당 라인이 자료구조에 미치는 영향이 즉시 시각화됨
3. **단계별 디버깅 모드**: AST 기반 코드 분석으로 변수 상태, 호출 스택, 메모리 변화를 시각적으로 표시. 재귀 호출 트리의 실시간 시각화 포함
4. **AI 힌트 시스템**: 학습자의 코드를 분석하여 맞춤형 힌트와 최적화 제안 제공. 시간/공간 복잡도 자동 분석, 코드 개선 방향 제시
5. **코딩테스트 실전 모드**: 시간 제한 문제 풀이 + 풀이 과정 시각화 리플레이. 풀이 완료 후 최적 풀이와의 비교 분석 제공

### 2.3 핵심 알고리즘: 코드-시각화 동기화 엔진

AlgoViz의 핵심 차별화 기술은 **코드 실행과 시각화의 실시간 동기화 엔진**이다. 이 엔진의 동작 원리는 다음과 같다:

1. **AST 파싱 단계**: Tree-sitter를 활용하여 사용자 코드를 추상 구문 트리(AST)로 변환. 각 노드에 실행 순서와 변수 영향 범위를 태깅
2. **인스트루멘테이션 단계**: AST 각 노드에 시각화 이벤트 트리거를 삽입. 배열 접근, 비교, 스왑 등의 연산을 자동 감지
3. **샌드박스 실행 단계**: Docker 컨테이너 또는 Pyodide(브라우저 내 Python)에서 인스트루멘트된 코드를 안전하게 실행
4. **이벤트 스트리밍 단계**: WebSocket을 통해 실행 이벤트를 프론트엔드로 실시간 전송
5. **애니메이션 렌더링 단계**: D3.js/Three.js가 이벤트를 받아 자료구조 상태 변화를 부드러운 애니메이션으로 렌더링

AlgoViz는 무료 플랜을 통해 경제적 여건에 관계없이 모든 학습자가 양질의 알고리즘 학습 도구에 접근할 수 있도록 한다. 특히 지방 대학이나 비전공자 등 교육 자원이 부족한 환경의 학습자에게 큰 도움이 될 것으로 기대한다.

### 10.2 SW 인재 양성 기여

정부의 디지털 인재 100만 양성 계획에 부합하며, CS 교육의 효율성을 높여 국가 SW 경쟁력 강화에 기여한다. 시각화 도구를 통한 학습 효율 향상으로, 같은 시간에 더 깊은 알고리즘 이해가 가능해진다.

### 10.3 UN SDGs 연계

- **SDG 4 (양질의 교육)**: 혁신적 학습 도구로 CS 교육의 질적 향상
- **SDG 9 (산업, 혁신, 인프라)**: 기술 인재 양성을 통한 혁신 생태계 기여
- **SDG 10 (불평등 감소)**: 무료 기본 플랜으로 교육 접근성 불평등 완화
