## 1. 문제 인식 (Problem)

### 1.1 소프트웨어 보안 취약점의 심각성

소프트웨어 보안 취약점은 현대 디지털 사회에서 가장 심각한 위협 중 하나이다. 한국인터넷진흥원(KISA)에 따르면 2023년 국내 사이버 침해사고 신고 건수는 1,277건으로 전년 대비 약 36% 증가하였으며, 이 중 상당수가 애플리케이션 레벨의 보안 취약점에서 기인하였다 [5].

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| OWASP Top 10 취약점으로 인한 웹 보안 사고 비율 | **75%** | OWASP Foundation, 2024 [1] |
| 오픈소스 프로젝트의 보안 취약점 포함 비율 | **84%** | Synopsys OSSRA Report, 2024 [6] |
| 취약점 발견에서 패치까지 평균 소요 시간 | **205일** | IBM Cost of a Data Breach, 2024 [7] |
| 데이터 유출 1건당 평균 피해 비용 (글로벌) | **$4.88M** | IBM Cost of a Data Breach, 2024 [7] |
| 데이터 유출 1건당 평균 피해 비용 (한국) | **$3.62M** | IBM Cost of a Data Breach, 2024 [7] |
| 개발 후 보안 수정 비용 대비 설계 단계 수정 비용 비율 | **1:30** | NIST, 2024 [8] |
| 국내 사이버 침해사고 신고 건수 (2023) | **1,277건** | KISA, 2024 [5] |
| SW 개발 과정에서 보안 테스트를 수행하는 기업 비율 | **42%** | Veracode State of Software Security, 2024 [9] |

이러한 수치는 **소프트웨어 개발 초기 단계에서의 보안 취약점 탐지**가 비용 효율적이고 필수적임을 보여준다. 특히 NIST의 연구에 따르면 설계 단계에서 취약점을 수정하는 비용 대비 운영 단계에서 수정하는 비용은 최대 30배에 달한다 [8]. 이는 **Shift-Left Security**, 즉 보안을 개발 프로세스의 가장 앞단으로 이동시키는 전략의 당위성을 입증한다.

| 순위 | 취약점 유형 | 발생 비율 | AI 탐지 가능성 |

| A03 | Injection (인젝션) | 19.4% | ★★★★★ |

위 표에서 확인할 수 있듯이, OWASP Top 10 취약점의 대부분은 **AI 기반 자동 분석으로 높은 정확도의 탐지가 가능**하다 [1]. 특히 Injection, 암호화 실패, 보안 설정 오류 등은 정형화된 패턴이 존재하여 AST 분석과 LLM의 조합으로 효과적으로 탐지할 수 있다.

### 1.2 기존 코드 리뷰 프로세스의 한계

현재 소프트웨어 개발 현장에서 코드 리뷰는 크게 세 가지 방식으로 이루어진다.

**1) 수동 코드 리뷰 (Peer Review)**
- 개발자 간 상호 검토 방식으로, 보안 전문 지식 부족으로 취약점 탐지율이 낮음
- Google의 연구에 따르면 수동 코드 리뷰로 발견되는 보안 취약점은 전체의 **15% 미만** [10]
- 리뷰어의 피로도, 시간 압박 등으로 품질 일관성이 부족

**2) 기존 정적 분석 도구 (SAST)**
- SonarQube, Fortify 등 규칙 기반 정적 분석 도구
- **오탐률(False Positive Rate)이 30~70%**에 달하여 개발자의 신뢰도 저하 [9]
- 문맥(Context)을 이해하지 못하여 비즈니스 로직 취약점 탐지 불가
- 새로운 취약점 패턴에 대한 대응이 느림 (규칙 업데이트 의존)

**3) AI 코딩 어시스턴트 (GitHub Copilot 등)**
- 코드 생성에 초점, 보안 취약점 전문 탐지 기능 부족
- 생성된 코드 자체에 취약점이 포함될 가능성이 40% [11]
- 한국어 지원 미흡, 국내 보안 규정(개인정보보호법 등) 미반영

#### 기존 방식의 한계 요약

| 한계 요소 | 수동 리뷰 | 기존 SAST | AI 어시스턴트 |
|-----------|----------|-----------|-------------|
| 보안 취약점 탐지율 | 15% | 45~60% | 20~30% |
| 오탐률 (False Positive) | 낮음 | 30~70% | 중간 |
| 문맥 이해 능력 | 높음 | 낮음 | 중간 |
| 처리 속도 | 느림 (시간~일) | 빠름 (분) | 빠름 (초) |
| 비용 | 높음 (인건비) | 중간 | 낮음 |
| 한국어 지원 | 가능 | 제한적 | 제한적 |
| 자동 패치 제안 | 불가 | 제한적 | 제한적 |
| 교육 기능 | 없음 | 없음 | 없음 |

이러한 한계점들은 **AST 기반 구조 분석과 LLM 기반 문맥 분석을 결합한 새로운 접근법**이 필요함을 시사한다.

## 2. 실현 가능성 (Solution)

### 2.1 핵심 기술: AST + LLM 기반 취약점 분석

CodeGuard의 핵심 기술은 **AST(Abstract Syntax Tree) 기반 구조 분석**과 **LLM(Large Language Model) 기반 문맥 분석**을 결합한 하이브리드 접근법이다.

#### 기술 아키텍처 개념

| 분석 계층 | 기술 | 역할 | 탐지 대상 |
|-----------|------|------|----------|
| **Layer 1: 구조 분석** | AST 파싱 (Tree-sitter) | 코드 구조를 트리로 변환, 패턴 매칭 | SQL Injection, XSS, 버퍼 오버플로우 등 정형 패턴 |
| **Layer 2: 흐름 분석** | Data Flow Analysis | 변수의 데이터 흐름 추적 | Taint Analysis, 미검증 입력값 추적 |
| **Layer 3: 문맥 분석** | LLM (Fine-tuned) | 코드의 의도와 문맥 이해 | 비즈니스 로직 취약점, 설계 결함 |
| **Layer 4: 패치 생성** | LLM + AST Transform | 취약점 수정 코드 자동 생성 | 안전한 대안 코드 제안 |

#### AST + LLM 하이브리드 분석의 장점

| 비교 항목 | 규칙 기반 (SAST) | LLM 단독 | **AST + LLM 하이브리드** |
|-----------|-----------------|----------|------------------------|
| 정형 패턴 탐지 | ★★★★★ | ★★★☆☆ | ★★★★★ |
| 문맥 기반 탐지 | ★☆☆☆☆ | ★★★★☆ | ★★★★★ |
| 오탐률 | 30~70% | 15~25% | **10~15%** |
| 처리 속도 | 빠름 | 느림 | 중간 (최적화 가능) |
| 새로운 패턴 대응 | 느림 (규칙 추가) | 빠름 (학습) | 빠름 |

| 영역 | 기술 | 선정 사유 |
|------|------|----------|
| **Frontend** | Next.js 14 + TypeScript | SSR 지원, React 생태계, 빠른 개발 |

| **AST 파서** | Tree-sitter | 다국어 지원(Python, JS, Java, Go 등), 증분 파싱 |
| **LLM** | OpenAI GPT-4o / Claude 3.5 (API) → 자체 Fine-tuned 모델 (장기) | 초기 비용 최소화, 점진적 자체 모델 전환 |

| **인프라** | AWS (ECS Fargate, S3, RDS) | 서버리스 아키텍처로 비용 최적화 |
| **CI/CD** | GitHub Actions | GitHub 생태계 통합, 무료 사용 가능 |
| **모니터링** | Prometheus + Grafana | 오픈소스, 강력한 메트릭 수집 |
| **VS Code 확장** | VS Code Extension API + TypeScript | 시장 점유율 1위 IDE 대응 |

#### 정성적 기대 효과

1. **보안 인력 양성 기여**: 대학 SW 교육에 실무 보안 역량을 통합하여, 졸업 시점에 보안 기본 역량을 갖춘 개발자 배출에 기여
2. **Shift-Left Security 문화 확산**: 개발 초기 단계에서 보안을 고려하는 문화를 국내 SW 산업에 확산
3. **한국어 보안 교육 자료 축적**: 취약점 분석 과정에서 축적되는 한국어 보안 콘텐츠가 업계 전체의 지식 자산이 됨
4. **CS 전공 학생 창업 롤모델**: 전공 지식을 직접 사업화하는 성공 사례로서 후배 학생들의 기술 창업 동기 부여
