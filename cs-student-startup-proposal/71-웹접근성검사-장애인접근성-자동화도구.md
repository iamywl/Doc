## 1. 문제 인식

### 웹 접근성 현황

| 지표 | 수치 | 출처 |
|------|------|------|
| 국내 웹사이트 WCAG 준수율 | 32% | 한국웹접근성인증평가원 (2025) |
| 장애인 웹 이용 불편 경험률 | 78% | 한국장애인개발원 (2025) |
| 접근성 미준수로 과태료 부과 건수 (연간) | 약 1,200건 | 과학기술정보통신부 (2025) |
| 공공기관 접근성 의무 준수 대상 | 4만개 | 장애인차별금지법 시행령 |
| 접근성 수동 점검 평균 소요 시간 | 40~80시간/사이트 | 업계 평균 (2025) |
| 민간기업 접근성 인식률 | 24% | 한국정보화진흥원 (2025) |
| 장애인 온라인 쇼핑 포기 경험률 | 65% | 한국소비자원 (2025) |
| 접근성 관련 소송 건수 (전년 대비 증가율) | +42% | 법원행정처 (2025) |

2025년부터 장애인차별금지법 강화로 민간 기업까지 접근성 의무가 확대되었으나, 대부분의 기업이 전문 인력과 도구 부족으로 준수에 어려움을 겪고 있다. 수동 검사는 비용과 시간이 과다하게 소요된다.

### 사용자 페인 포인트 분석

**장애인 사용자의 목소리:**
- *"온라인 쇼핑몰에서 상품 이미지에 대체 텍스트가 없어서 스크린리더로 무엇을 파는지 전혀 알 수 없었습니다."* - 시각장애인 A씨 (35세)
- *"은행 웹사이트에서 이체를 하려는데 보안 키패드가 키보드로 작동하지 않아 매번 영업점을 방문해야 합니다."* - 지체장애인 B씨 (42세)
- *"공공기관 민원 사이트의 캡차(CAPTCHA)를 통과할 수 없어 간단한 서류 발급도 다른 사람의 도움이 필요합니다."* - 시각장애인 C씨 (28세)

**개발자/기업의 어려움:**
- WCAG 가이드라인이 복잡하여 어떤 부분을 수정해야 하는지 파악하기 어려움 (개발자 설문 82% 응답)
- 접근성 전문가 채용이 어렵고, 외부 컨설팅 비용이 과다 (건당 300~500만원)
- 기존 코드를 접근성에 맞게 수정하는 작업이 매우 노동집약적
- CI/CD 과정에 접근성 검증이 통합되지 않아 배포 후 이슈 발견이 빈번

### 법적 환경 변화

2025년 장애인차별금지법 개정으로 다음과 같은 변화가 발생했다:
- **민간 기업 확대**: 종전 공공기관 중심에서 일정 규모 이상 민간 기업까지 접근성 의무 확대
- **과태료 상향**: 미준수 시 과태료 최대 3,000만원에서 5,000만원으로 상향
- **인증 의무화**: 공공기관 신규 웹사이트는 접근성 인증 필수
- **손해배상 강화**: 접근성 미준수로 인한 차별에 대한 손해배상 청구 근거 강화

### 핵심 기능 상세

#### 2.1 WCAG 자동 검사 엔진
- **정적 분석**: HTML/CSS/ARIA 마크업 기반 78개 WCAG 2.2 규칙 자동 검사
- **동적 분석**: Puppeteer 기반 Headless Browser로 키보드 내비게이션, 포커스 이동, 동적 콘텐츠 접근성 검사
- **시각 분석**: Vision AI를 활용한 스크린샷 기반 색상 대비, 글씨 크기, 레이아웃 접근성 분석
- **검사 속도**: 페이지당 평균 15초, 100페이지 사이트 약 25분 완료

#### 2.2 접근성 리포트
- **심각도 분류**: Critical / Major / Minor / Info 4단계 분류
- **WCAG 매핑**: 각 이슈별 WCAG 성공 기준 자동 매핑 및 해설
- **점수 산출**: 100점 만점 접근성 점수, 카테고리별 세부 점수
- **트렌드 분석**: 시간별/배포별 접근성 점수 변화 추이
- **한국 인증 대응**: 한국웹접근성인증평가원 기준 준수 여부 별도 표시

#### 2.3 AI 수정 가이드
- **자동 코드 생성**: GPT-4 기반으로 발견된 이슈에 대한 구체적 수정 코드 자동 생성
- **컨텍스트 인식**: 기존 코드 스타일, 프레임워크(React/Vue/Angular)에 맞춤 코드 제안
- **원클릭 적용**: 대시보드에서 바로 수정 코드 적용 및 PR 생성
- **학습 기능**: 수정 이력 기반 반복 이슈 패턴 자동 감지 및 예방 가이드

#### 2.4 CI/CD 연동
- **GitHub Actions**: 공식 액션으로 PR 생성 시 자동 접근성 검사
- **GitLab CI**: .gitlab-ci.yml 설정 한 줄로 파이프라인 연동
- **Jenkins**: 플러그인 설치 후 빌드 단계에 접근성 게이트 추가
- **품질 게이트**: 접근성 점수 기준치 미달 시 배포 자동 차단
- **알림 연동**: Slack, Microsoft Teams, 이메일 자동 알림

### 시장 성장 동인

장애인차별금지법 강화와 ESG 경영 확산으로 웹 접근성 솔루션 시장은 연 28% 이상 고성장이 예상된다. 특히 민간 기업의 접근성 의무 확대가 시장 성장의 핵심 동력이다.

**핵심 성장 동인:**
1. **법적 의무 확대**: 2025년 장애인차별금지법 개정으로 민간기업 접근성 의무 확대
2. **ESG 경영 트렌드**: 기업 ESG 보고서에 웹 접근성 항목 포함 증가
3. **글로벌 규제 강화**: EU European Accessibility Act (EAA) 2025년 시행, ADA 소송 증가
4. **디지털 전환 가속**: 코로나19 이후 온라인 서비스 의존도 증가로 접근성 중요도 상승
5. **개발자 인식 향상**: DevOps/시프트 레프트 트렌드와 맞물린 접근성 자동화 수요
