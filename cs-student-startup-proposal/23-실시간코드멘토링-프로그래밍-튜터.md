### 팀 비전

> "주니어 개발자가 막힐 때 바로 옆에서 도와주는 시니어 멘토를 온라인으로 연결한다."

코드멘토는 실시간 코드 공유, 시니어 개발자 매칭, 코드 리뷰 세션 등을 제공하는 프로그래밍 멘토링 플랫폼을 개발한다. 기존 코딩 Q&A(Stack Overflow 등)가 비동기 텍스트 기반인 반면, 코드멘토는 실시간 화면 공유와 코드 동시 편집으로 "옆에서 같이 코딩하는" 경험을 제공하는 것이 핵심 차별점이다.

코드멘토가 해결하고자 하는 근본적 문제는 "주니어 개발자의 성장 속도"이다. 프로그래밍에서 막히는 순간 즉시 도움을 받을 수 있다면, 평균 3.2시간이 걸리는 디버깅을 30분 이내에 해결할 수 있다. 또한 시니어의 실시간 코드 리뷰를 통해 "잘못된 습관"이 굳어지기 전에 교정받을 수 있어, 주니어에서 미드레벨로의 성장 기간을 대폭 단축할 수 있다. 코드멘토는 이러한 "실시간 시니어 접근성"을 민주화하는 것을 목표로 한다.

## 1. 문제 인식 (Problem)

### 1.1 주니어 개발자의 성장 병목

프로그래밍 학습에서 가장 큰 장벽은 "막힐 때 도움을 받을 수 없는 것"이다. Stack Overflow의 2024 개발자 서베이에 따르면, 주니어 개발자의 73%가 "코딩 중 막힐 때 즉시 도움받을 수 있는 시니어가 없다"고 응답하였다 [1].

이 문제가 심각한 이유는 프로그래밍 학습에서 "막힘"이 단순한 불편이 아니라 **학습 포기의 주요 원인**이기 때문이다. freeCodeCamp의 조사에 따르면 혼자 학습하는 개발자의 프로젝트 완성률은 23%에 불과하며, 그 주된 이유가 "해결되지 않는 버그/에러에 좌절"이었다. 멘토가 있는 환경에서 학습한 경우 프로젝트 완성률은 78%로 크게 높아진다 [5].

또한 코드 리뷰 문화의 부재도 심각한 문제이다. 주니어 개발자의 62%가 "코드 리뷰를 받아본 적이 없다"고 응답했으며 [8], 이로 인해 잘못된 코딩 습관이 고착화되고, 실무 투입 시 코드 품질 이슈로 팀 전체의 생산성이 저하된다.

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 코딩 중 "막힘"으로 진행 중단 경험 비율 | **89%** | Stack Overflow, 2024 [1] |
| 즉시 도움받을 시니어 부재 | **73%** | Stack Overflow, 2024 [1] |
| 버그 디버깅에 소요되는 평균 시간 | **3.2시간/건** | JetBrains, 2024 [2] |
| 혼자 학습 시 프로젝트 완성률 | **23%** | freeCodeCamp, 2024 [5] |
| 코딩 부트캠프 수료 후 취업률 | **55%** | 한국정보화진흥원, 2024 [6] |
| 국내 주니어 개발자 수 (3년 미만) | **약 38만 명** | 과기정통부, 2025 [7] |
| 코드 리뷰를 받아본 적 없는 주니어 비율 | **62%** | 프로그래머스, 2024 [8] |
| 멘토가 있었을 때 프로젝트 완성률 | **78%** | freeCodeCamp, 2024 [5] |
| 스타트업의 시니어 개발자 부족률 | **67%** | 로켓펀치, 2024 |

#### 사용자 Pain Point 심층 분석

**주니어 개발자/대학생:**
- "에러 메시지를 구글링해도 해결이 안 되면 며칠씩 막혀 있다. 옆에 물어볼 사람이 있으면 10분이면 해결될 텐데." (주니어 개발자 A, 경력 1년)
- "코딩 부트캠프를 수료했지만, 실무 프로젝트에서 막히면 어디서 도움을 받아야 할지 모르겠다." (부트캠프 수료생 B)
- "GitHub Copilot은 코드를 생성해주지만, 왜 이렇게 작성하는지 설명을 안 해준다. 개념을 이해하고 싶다." (CS 대학생 C)
- "코드 리뷰 없이 혼자 코딩하니까 나쁜 습관이 고착화되는 것 같다." (비전공 전환 개발자 D)

**시니어 개발자 (멘토 측):**
- "후배들을 도와주고 싶은데, 사내에서는 바쁘니까 따로 시간을 내기 어렵다. 유연하게 시간을 쓸 수 있으면 좋겠다." (시니어 E, 경력 8년)
- "프리랜서인데 빈 시간에 멘토링으로 부수입을 올리고 싶다." (프리랜서 개발자 F)

### 1.2 기존 솔루션의 한계

| 기존 솔루션 | 한계점 | 미충족 니즈 |
|------------|--------|-----------|
| Stack Overflow | 비동기, 질문-답변 시간 지연, 맥락 공유 어려움 | 실시간 대화, 코드 맥락 공유 |
| GitHub Copilot | 코드 생성만, 개념 설명/멘토링 불가 | 개념 이해, 설계 자문 |
| ChatGPT | 실시간 코드 공유 불가, 프로젝트 맥락 이해 한계 | 실제 프로젝트 코드 기반 도움 |
| 코딩 부트캠프 | 고비용 (300~500만 원), 특정 기간만 멘토링 | 상시 접근 가능한 멘토링 |
| 사내 코드 리뷰 | 주니어는 시니어에게 부담감, 스타트업은 시니어 부족 | 부담 없는 외부 멘토링 |
| Codementor.io (해외) | 영어만 지원, 한국 시간대 불편, 시간당 $60+ | 한국어, 합리적 가격 |

### 2.2 핵심 기능 상세

| 기능 | 설명 | 기술적 구현 |

| **즉시 멘토 매칭** | 기술 스택/분야 기반 5분 이내 시니어 멘토 매칭 | Go 기반 매칭 엔진, Redis 대기열, 기술 스택 벡터 유사도 |
| **실시간 코드 동시 편집** | CRDT 기반 양방향 코드 편집, 멘토가 직접 코드 수정/시연 | Monaco Editor + Yjs, WebSocket 기반 CRDT 동기화 |
| **화상 + 화이트보드** | WebRTC 화상 통화 + 개념 설명용 화이트보드 | LiveKit SFU, Excalidraw 기반 화이트보드 |
| **코드 리뷰 세션** | GitHub PR 연동 코드 리뷰, 라인별 댓글, 실시간 토론 | GitHub API v4 (GraphQL), 인라인 코멘트 시스템 |
| **세션 녹화 & 북마크** | 모든 멘토링 세션 자동 녹화, 중요 부분 북마크 | MediaRecorder API, 서버사이드 트랜스코딩 |
| **멘토 평점/리뷰** | 세션 후 양방향 평가, 전문 분야별 멘토 랭킹 | 가중 평점 알고리즘, 기술 스택별 전문성 스코어 |

### 2.3 CRDT 기반 실시간 코드 동시 편집 상세

코드멘토의 핵심 기술인 CRDT(Conflict-free Replicated Data Type) 기반 코드 동시 편집은 다음과 같이 동작한다:

**기술적 구현:**
- **Yjs** 라이브러리를 활용한 CRDT 구현: 분산 환경에서 충돌 없는 동시 편집 보장
- **Monaco Editor** 통합: VS Code와 동일한 에디터 경험 제공 (구문 강조, 자동완성, IntelliSense)
- **WebSocket** 기반 실시간 동기화: 타이핑 지연 50ms 이내
- **커서 공유**: 멘토와 멘티의 커서 위치가 실시간으로 표시되어 "지금 어디를 보고 있는지" 확인 가능

**기존 화면 공유 대비 장점:**
- 멘토가 직접 코드를 수정/추가하여 시연 가능 (화면 공유는 관찰만 가능)
- 양방향 편집으로 "같이 코딩하는" 경험 제공
- 코드 변경 히스토리 자동 기록으로 세션 후 복습 가능
- 네트워크 지연이 있어도 CRDT 특성상 최종 일관성 보장

### 3.2 시장 성장 동인

- **코드 리뷰 문화 확산**: 개발자 커뮤니티에서 코드 리뷰의 중요성이 강조되면서 외부 코드 리뷰 서비스 수요 증가
- **원격 근무 확산**: 온라인 멘토링에 대한 거부감 감소, 시니어 개발자의 유연한 멘토링 참여 가능
- **AI 코딩 도구의 한계 인식**: GitHub Copilot/ChatGPT의 한계를 경험한 주니어들이 "사람 멘토"의 가치를 재인식

| **재무 리스크** | WebRTC 인프라 비용 증가 | 중 | 중 | SFU 최적화, 비활성 시 리소스 해제, 사용량 기반 스케일링 |
