> "두꺼운 전공 서적의 핵심을 AI가 추출하고, 시험 대비 자료까지 자동 생성하는 학습 도우미를 만든다."

스터디다이제스트는 OCR과 LLM을 결합하여 전공 교재를 스캔하면 핵심 개념 추출, 요약 정리, 시험 대비 자료 자동 생성까지 해주는 서비스를 개발한다. 단순 요약이 아닌, 교재 간 개념 연결, 학과별 커리큘럼 맞춤, 기출문제 연계까지 제공하는 것이 핵심 차별점이다.

스터디다이제스트가 궁극적으로 목표하는 것은 "전공 학습의 민주화"이다. 같은 교재를 읽어도 어떤 학생은 핵심을 빠르게 파악하고 효율적으로 학습하는 반면, 어떤 학생은 중요한 부분과 덜 중요한 부분을 구분하지 못해 시간을 낭비한다. 스터디다이제스트는 AI가 전공 교재의 핵심 구조를 파악하고, 시험 출제 가능성이 높은 내용을 자동으로 추출함으로써, 학습 효율의 격차를 줄이고자 한다.

## 1. 문제 인식 (Problem)

### 1.1 대학생 전공 학습의 비효율

대학생들은 매 학기 두꺼운 전공 서적을 여러 권 소화해야 하지만, 실제 완독률은 매우 낮고 시험 준비는 비효율적으로 이루어지고 있다. 대학내일 조사에 따르면 전공 교재 완독률은 18%에 불과하며, 대부분 시험 직전 벼락치기에 의존한다 [1].

이 문제의 핵심 원인은 세 가지이다:

**1) 교재의 접근성 장벽**: 전공 서적은 학술적 문체로 작성되어 있어 학생이 독학으로 핵심을 파악하기 어렵다. 특히 이공계 교재는 수식과 도표가 많아 텍스트만으로는 이해가 부족하고, 인문사회 교재는 방대한 텍스트에서 핵심 논점을 추출하기 어렵다.

**2) 요약 자료 제작의 비효율**: 시험 전 과목당 평균 12시간을 요약 자료 제작에 투자하지만, 5.8과목을 수강하는 학생이라면 총 70시간 가까이를 단순 정리 작업에 소비하게 된다. 이는 실제 학습(이해/암기)에 투자할 시간을 빼앗는다.

**3) 학습 자료의 편차와 신뢰성**: 에브리타임 등에서 공유되는 족보/요약 자료는 품질 편차가 심하고, 교수별 강의 내용과 불일치하는 경우가 많다. 또한 저작권 문제가 상존한다.

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 전공 교재 완독률 | **18%** | 대학내일, 2024 [1] |
| 시험 전 요약 자료 제작 소요 시간 (1과목) | **평균 12시간** | 캠퍼스잡앤조이, 2024 [2] |
| 수업 내용 중 교재에서 직접 출제 비율 | **67%** | 교수학습개발센터 조사, 2024 [5] |
| "전공 서적이 너무 어렵다" 응답 비율 | **74%** | 대학내일, 2024 [1] |
| 타인의 요약 자료 구매/사용 경험 비율 | **52%** | 에브리타임 설문, 2024 [6] |
| 학기당 평균 전공 교재 구입 비용 | **18만 원** | 한국교육개발원, 2024 [7] |
| 대학생 1인당 평균 수강 과목 수 (학기) | **5.8과목** | 교육부, 2024 [8] |
| 시험 직전 벼락치기 비율 | **63%** | 캠퍼스잡앤조이, 2024 [2] |
| "효과적인 요약 정리 방법을 모른다" | **58%** | 대학내일, 2024 [1] |

#### 사용자 Pain Point 심층 분석

**대학생:**
- "700페이지짜리 전공 교재를 어디서부터 읽어야 할지 모르겠다. 핵심만 알려주면 좋겠다." (CS 전공 A)
- "시험 범위가 챕터 5개인데, 요약 노트 만드는 데만 이틀이 걸린다. 실제 공부할 시간이 없다." (경영학 전공 B)
- "에브리타임에서 족보를 구했는데, 교수님이 바뀌어서 출제 범위가 달랐다." (법학 전공 C)
- "수식이 많은 교재는 ChatGPT에 넣을 수도 없고, 사진 찍어서 넣어도 수식을 제대로 인식하지 못한다." (전자공학 전공 D)

**대학원생:**
- "논문 리뷰를 위해 교재 여러 권을 참고해야 하는데, 관련 개념이 어느 챕터에 있는지 찾기가 어렵다." (대학원생 E)

### 2.2 핵심 기능 상세

| 기능 | 설명 | 기술적 구현 |
|------|------|-----------|
| **교재 OCR 인식** | 카메라 촬영/PDF 업로드 → 텍스트+수식+도표 정확 인식 | Tesseract + PaddleOCR + LaTeX-OCR + DETR |
| **AI 핵심 요약** | 챕터별 핵심 개념 3~5줄 요약, 중요도 표시 | GPT-4 API + Fine-tuned Llama 3, 학술 요약 프롬프트 |
| **개념 연결 그래프** | 현재 챕터의 개념이 다른 챕터/과목과 어떻게 연결되는지 시각화 | Neo4j 그래프 DB, D3.js 시각화, 학과별 온톨로지 |
| **시험 대비 자료 생성** | 요약 노트, 플래시카드, 예상 문제, OX퀴즈 자동 생성 | LLM 기반 문제 생성, 난이도 자동 조정 |
| **기출 연계 분석** | 해당 개념이 기출문제에서 어떻게 출제되었는지 매핑 | 기출 DB + 개념 매칭, TF-IDF 유사도 |
| **학과별 커리큘럼 매핑** | 대학별/학과별 수업 순서에 맞춘 학습 가이드 | 대학 커리큘럼 DB, 선수과목 그래프 |

### 2.3 OCR 파이프라인 상세

스터디다이제스트의 OCR 파이프라인은 기존 범용 OCR과 달리 학술 교재에 특화되어 있다:

**1단계 - 레이아웃 분석:** 페이지의 구조(제목, 본문, 수식, 도표, 캡션, 사이드노트)를 자동으로 분류한다. DETR 기반 객체 감지 모델로 95%+ 정확도를 목표로 한다.

**2단계 - 텍스트 OCR:** PaddleOCR을 활용하여 한국어/영어 혼합 텍스트를 인식한다. 학술 용어 사전을 활용한 후처리로 전문 용어 인식 정확도를 높인다.

**3단계 - 수식 인식:** LaTeX-OCR 모델을 활용하여 수식을 LaTeX 코드로 변환한다. 이공계 교재의 핵심인 수식을 정확하게 디지털화하는 것이 기존 서비스 대비 핵심 차별점이다.

**4단계 - 도표 구조 추출:** 표(Table)의 행/열 구조를 자동으로 파악하고, 그래프/차트는 데이터를 추출하여 재생성 가능한 형태로 변환한다.
