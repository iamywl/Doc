## 3. 문제 인식

### 3.1 대학생 통학 현황 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 대학생 통학 소요 시간 (편도 평균) | 약 62분 | 한국교통연구원 (2025) |
| 대학생 월 평균 교통비 | 약 14만 8천 원 | 대학내일20대연구소 (2025) |
| 자차 통학 대학생 비율 | 약 18.4% | 한국교통연구원 (2025) |
| 자차 통학 시 평균 탑승 인원 | 1.1명 (거의 혼자) | 서울연구원 (2025) |
| 대중교통 환승 2회 이상 학생 비율 | 47.3% | 교육부 대학생 실태조사 (2025) |
| 통학 포기 (자퇴/휴학) 연관 비율 | 8.2% | 대학교육연구소 (2025) |
| 통학 버스 미운영 대학 비율 | 63.7% | 한국대학신문 (2025) |
| 대학생 카풀 의향 (조건 충족 시) | 82.6% | 대학내일20대연구소 (2025) |

> "수원에서 서울 학교까지 편도 1시간 40분 걸려요. 같은 방향 친구를 찾고 싶은데 방법이 없습니다."
> — 서울 소재 대학교 3학년 김OO (23세)

> "차가 있어서 통학하는데 주유비가 월 30만 원 넘어요. 카풀하면 나눌 수 있는데, 에브리타임 게시판은 불안해요."
> — 경기도 거주 대학생 운전자 이OO (25세)

> "1학년 때 통학이 너무 힘들어서 자취를 시작했는데, 경제적 부담이 크네요. 카풀이 있었다면..."
> — 충남에서 서울 통학 → 자취 전환 박OO (22세)

> "학교 셔틀이 없어져서 학생들 민원이 많아요. 대안이 필요합니다."
> — 지방 소재 대학교 학생처 직원 최OO (35세)

### 4.3 기술 스택

| 구분 | 기술 | 용도 |
|------|------|------|
| 경로 매칭 | OSRM, Dijkstra 변형 | 출발지-학교 경로 유사도 매칭 |
| 실시간 위치 | WebSocket, Redis Pub/Sub | 차량 위치 실시간 공유 |
| 모바일 | Flutter (Dart) | iOS/Android 크로스 플랫폼 앱 |
| 백엔드 | NestJS (TypeScript) | REST API, 비즈니스 로직 |
| 데이터베이스 | PostgreSQL + PostGIS | 지리 데이터, 사용자 정보 |
| 결제 | 토스페이먼츠 | 인앱 비용 분담 결제 |
| 지도 | 카카오맵 SDK | 경로 시각화, 위치 검색 |
| 인프라 | AWS (ECS, RDS, ElastiCache) | 서비스 배포, DB, 캐시 |
| 안전 | Firebase Crashlytics, Sentry | 앱 안정성, 오류 추적 |
| CI/CD | GitHub Actions, Fastlane | 앱 자동 빌드·배포 |

## 5. 시장 분석

### 5.1 시장 규모 (TAM/SAM/SOM)

| 구분 | 규모 | 산출 근거 |
|------|------|-----------|
| TAM (전체 시장) | 약 1조 8,000억 원 | 국내 카풀/차량공유 시장 전체 |
| SAM (유효 시장) | 약 3,800억 원 | 대학생 통학 교통비 시장 (월 14.8만 x 학생 수) |
| SOM (목표 시장) | 약 120억 원 | 대학생 카풀 서비스 (3년 내 점유) |

### 5.2 경쟁사 분석

| 경쟁사 | 강점 | 약점 | 투게더라이드 차별점 |
|--------|------|------|---------------------|
| 카카오 T 카풀 (종료) | 높은 인지도 | 택시 업계 반발로 서비스 종료 | 대학생 한정 비상업적 비용 분담 (합법) |
| 풀러스 | 출퇴근 카풀 | 대학생 특화 기능 없음, 인증 약함 | 대학 인증 + 시간표 연동 |
| 에브리타임 게시판 | 대학생 접근성 | 체계 없음, 안전 장치 없음, 결제 없음 | 체계적 매칭 + 안전 + 결제 통합 |
| BlaBlaCar (해외) | 글로벌 실적 | 장거리 위주, 한국 미진출 | 한국 대학생 단거리 통학 특화 |
