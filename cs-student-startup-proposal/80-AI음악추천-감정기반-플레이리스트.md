## 1. 문제 인식

### 감정을 무시하는 기존 음악 추천의 한계

| 지표 | 수치 | 출처 |
|------|------|------|
| 국내 음악 스트리밍 이용자 | 3,200만명 (2025년) | 한국콘텐츠진흥원 |
| 추천 음악 불만족 비율 | 71% | 한국갤럽 |
| "기분에 맞는 음악" 수요 | 이용자 86%가 원함 | 음악산업백서 |
| 추천 곡 건너뛰기(Skip) 비율 | 평균 47% | Spotify Korea 보고서 |
| 음악 치료/웰니스 시장 | 국내 3,500억원 | 문화체육관광부 |
| MZ세대 감성 소비 비율 | 73% | 대학내일20대연구소 |
| 멘탈헬스 앱 이용자 | 850만명 | 한국정보화진흥원 |
| 공간 음악(BGM) 시장 | 국내 1,800억원 | 한국음악저작권협회 |
| 웨어러블 기기 보급률 | 성인 38% | 과기정통부 |
| AI 생성 음악 시장 성장률 | 연 45% | PwC |

기존 음악 추천 알고리즘은 사용자의 과거 청취 이력과 다른 사용자의 패턴을 기반으로 하는 협업 필터링에 의존한다. 이 방식은 사용자의 "현재 감정 상태"를 전혀 반영하지 못한다. 슬픈 날에 활기찬 노래가 추천되거나, 집중하고 싶을 때 가사가 많은 노래가 나오는 등 맥락 없는 추천이 반복된다.

### 사용자 페인 포인트 분석

**MZ세대 음악 소비자:**
> *"기분이 우울할 때 공감되는 음악을 듣고 싶은데, 스포티파이가 추천하는 건 항상 비슷한 장르뿐입니다."*

> *"운동할 때, 공부할 때, 잠들 때 다 다른 음악이 필요한데, 일일이 플레이리스트를 만들기 귀찮습니다."*

**카페 운영자:**
> *"오전, 오후, 저녁에 카페 분위기에 맞는 BGM이 달라야 하는데, 자동으로 맞춰주는 서비스가 없어요."*

**명상/웰니스 앱 개발자:**
> *"사용자 상태에 맞는 치유 음악을 실시간으로 제공하고 싶은데, 기존 API로는 감정 기반 추천이 안 됩니다."*

**피트니스 트레이너:**
> *"운동 강도에 따라 BPM이 자동으로 조절되는 음악 기능이 있으면 트레이닝 효과가 높아질 겁니다."*

### 핵심 기능

| 기능 | 설명 | 기술 요소 |
|------|------|-----------|
| 멀티모달 감정 인식 | 음성 톤, 텍스트, 얼굴 표정, 심박수 등 복합 분석 | CNN + Transformer 멀티모달 퓨전 |

### 기술 스택

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 감정 인식 | PyTorch, HuggingFace Transformers | 멀티모달 감정 인식 모델 학습 |
| 음성 분석 | Whisper + Emotion2Vec | 음성 톤에서 감정 추출 |
| 텍스트 감정 | KoBERT Fine-tuned | 한국어 텍스트 감정 분류 |
| 얼굴 표정 | MediaPipe + FER 모델 | 실시간 얼굴 표정 감정 인식 |
| 음악 분석 | Librosa + Essentia | 음악 특성 추출 (BPM, 조성, 에너지) |
| 음악 생성 | MusicGen + AudioLDM2 | 감정 기반 음악 실시간 생성 |
| 백엔드 | FastAPI + Redis | 실시간 추천 API, 캐싱 |
| 모바일 | Flutter | 크로스플랫폼 앱 |
| 인프라 | GCP (Vertex AI + Cloud Run) | AI 모델 서빙, 오토스케일링 |
