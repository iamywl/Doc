## 일반현황

### 팀 구성

| 구분 | 역할 | 전공/학년 | 핵심 역량 |
|:---|:---|:---|:---|
| 팀장 | PM / Backend | 컴퓨터공학 3학년 | Python, Django, AWS 인프라 경험 |
| 팀원A | AI/ML Engineer | 컴퓨터공학 4학년 | 추천시스템, PyTorch, NLP |
| 팀원B | Frontend / Mobile | 컴퓨터공학 3학년 | React Native, TypeScript, UI/UX |
| 팀원C | Data / DevOps | 컴퓨터공학 2학년 | 데이터 파이프라인, Docker, CI/CD |

### 팀 배경

- 전원 컴퓨터공학과 재학생으로 **알고리즘 설계, 소프트웨어 공학, 데이터베이스, 머신러닝** 교과과정 이수
- 학과 캡스톤 프로젝트 및 해커톤 참여 경험 보유
- 교내 자취생 비율 60% 이상인 환경에서 식비 문제를 직접 체감한 당사자 팀

## 창업 아이템 개요

### 한줄 요약

> **"AI가 최적의 공동구매를 설계하고, 같이 밥 먹을 사람을 연결해주는 대학생 전용 플랫폼"**

### 핵심 가치 제안 (Value Proposition)

| 구분 | 기존 방식 | 캠퍼스잇츠 |
|:---|:---|:---|
| **식비 절감** | 배달앱 개별 주문 (배달비 3,500원+) | AI 공동구매로 30~40% 절감 |
| **식품 폐기** | 대용량 구매 후 남은 식재료 폐기 | 적정량 분배 + 나눔 매칭 |
| **사회적 연결** | 혼밥 문화 고착화 | AI 식사 매칭으로 자연스러운 교류 |
| **시간 효율** | 매번 메뉴 고민, 장보기 시간 | AI 추천 + 자동 공동구매 생성 |
| **영양 관리** | 편의점·라면 위주 불균형 식단 | 영양 기반 식단 추천 |

## 1. 문제 인식

### 1.1 대학생 식비 부담과 영양 불균형

#### 식비 부담 현실

대한민국 대학생의 월평균 식비는 약 **30~40만원**으로 [1], 이는 대학생 월평균 생활비(약 70만원)의 **43~57%**에 해당하는 비중이다. 특히 자취생의 경우 외식·배달 의존도가 높아 식비 부담이 더욱 가중된다.

| 항목 | 금액 (월평균) | 비율 | 출처 |
|:---|---:|---:|:---|
| 대학생 월평균 식비 | 35만원 | 100% | 통계청, 2024 [1] |
| 외식/배달 비용 | 18만원 | 51.4% | 한국소비자원, 2024 [2] |
| 편의점 식사 비용 | 8만원 | 22.9% | 한국편의점산업협회, 2024 [3] |
| 직접 조리 비용 | 9만원 | 25.7% | 통계청, 2024 [1] |

#### 배달비 부담 심화

배달앱 시장이 **약 30조원** 규모로 성장하면서 [4], 배달 주문 1건당 평균 배달비는 **3,500원**에 달한다. 대학생이 월 15회 배달을 이용한다고 가정하면 배달비만 월 **52,500원**이 소요되며, 이는 식비의 약 15%에 해당한다.

| 배달 관련 지표 | 수치 | 출처 |
|:---|---:|:---|
| 국내 배달앱 시장 규모 (2025) | 30조원 | 한국외식산업연구원, 2025 [4] |
| 평균 배달비 (1건) | 3,500원 | 배달의민족 내부 데이터, 2024 [5] |
| 대학생 월 평균 배달 횟수 | 15.2회 | 대학내일20대연구소, 2024 [6] |
| 월 배달비 지출 추정 | 53,200원 | 자체 산출 |
| 최소 주문금액 평균 | 15,000원 | 한국소비자원, 2024 [2] |

#### 영양 불균형 실태

자취 대학생의 **73.2%**가 하루 1회 이상 편의점·라면 등 간편식으로 식사를 해결하며 [7], 이로 인한 영양 불균형 문제가 심각하다.

| 영양 지표 | 대학생 평균 | 권장 기준 | 달성률 |
|:---|---:|---:|---:|

### 1.2 1인가구 식품 폐기와 환경 문제

#### 1인가구 식품 폐기 현황

환경부 조사에 따르면 1인가구의 연간 식품 폐기량은 가구당 약 **70kg**에 달하며 [8], 이 중 상당 부분이 **대용량 구매 후 소비하지 못해 버려지는 식재료**이다.

| 식품 폐기 지표 | 수치 | 출처 |
|:---|---:|:---|
| 1인가구 연간 식품 폐기량 | 70kg/가구 | 환경부, 2024 [8] |
| 1인가구 식품 폐기 비용 (연간) | 약 26만원/가구 | 환경부, 2024 [8] |
| 전국 음식물 쓰레기 처리 비용 (연간) | 약 2조원 | 환경부, 2024 [8] |
| 식품 폐기로 인한 탄소 배출 | 약 885만톤 CO₂/연 | 한국환경공단, 2024 [9] |

#### 1인가구 식품 폐기의 주요 원인

| 원인 | 비중 | 캠퍼스잇츠 해결 방안 |
|:---|---:|:---|
| 대용량 구매 후 소비 실패 | 38.2% | AI 적정량 계산 + 공동구매 분배 |
| 유통기한 관리 실패 | 27.5% | 스마트 유통기한 알림 + 나눔 매칭 |
| 조리 계획 부재 | 19.8% | AI 식단 추천 + 레시피 제안 |
| 충동 구매 | 14.5% | AI 기반 필요량 예측 + 공동구매 |

#### 환경 영향 분석

대학생 1인가구(전체 대학생의 약 **45%** [10])가 공동구매를 통해 식품 폐기를 **30%만 줄여도** 연간 약 **18,900톤**의 식품 폐기물을 감소시킬 수 있다.

```
산출 근거:
- 전국 대학생 수: 약 200만명 [11]
- 1인가구 비율: 45% = 90만명 [10]
- 연간 1인당 식품 폐기: 70kg [8]
- 감소 목표: 30%
- 효과: 90만명 × 70kg × 30% = 18,900톤/연
```

### 2.1 AI 추천 엔진 — 공동구매 최적화

#### 알고리즘 개요

캠퍼스잇츠의 AI 추천 엔진은 **3단계 최적화 파이프라인**으로 구성된다.

| 단계 | 기능 | 알고리즘 | 입력 데이터 |
|:---|:---|:---|:---|
| 1단계 | 수요 예측 | LSTM + Attention | 과거 구매 이력, 요일/시간, 날씨, 학사일정 |
| 2단계 | 그룹 최적화 | 유전 알고리즘 + 제약 조건 최적화 | 위치, 시간대, 선호 식재료, 예산 |
| 3단계 | 가격 최적화 | 강화학습 (Q-Learning) | 실시간 가격, 할인 정보, 배송비 구조 |

#### 공동구매 최적화 로직

```
[공동구매 최적화 수학 모델]

목적함수: min Σ(가격_i × 수량_i) + 배달비 / N명
제약조건:
  - Σ(수량_i) ≥ 최소주문금액
  - 각 참여자별 원하는 품목 충족
  - 배송 가능 반경 ≤ 1km
  - 시간 윈도우 일치 (±30분)
  - 총 참여자 N: 2 ≤ N ≤ 8
```

### 2.2 식사 매칭 알고리즘

#### 매칭 스코어 산출 구조

식사 매칭은 **다차원 유사도 기반 매칭 알고리즘**을 사용하며, 아래 가중치로 최종 매칭 스코어를 산출한다.

| 매칭 요소 | 가중치 | 측정 방법 |
|:---|---:|:---|
| 시간대 일치 | 25% | 식사 가능 시간 윈도우 오버랩 |
| 위치 근접도 | 20% | GPS 기반 도보 거리 (≤500m) |
| 음식 취향 유사도 | 20% | 코사인 유사도 (선호 벡터) |
| 예산 범위 일치 | 15% | 가격 구간 오버랩률 |
| 성향 호환성 | 10% | MBTI/식사 스타일 호환 매트릭스 |
| 신뢰도 점수 | 10% | 과거 매칭 이행률, 리뷰 점수 |

#### 매칭 알고리즘 의사코드

```python
def calculate_match_score(user_a, user_b):
    """다차원 매칭 스코어 계산"""
    time_score = time_window_overlap(user_a.available_times, user_b.available_times)
    location_score = 1.0 - (distance(user_a.location, user_b.location) / MAX_DISTANCE)
    taste_score = cosine_similarity(user_a.food_preference_vector, user_b.food_preference_vector)
    budget_score = budget_overlap(user_a.budget_range, user_b.budget_range)
    personality_score = personality_compatibility(user_a.profile, user_b.profile)
    trust_score = (user_a.trust_rating + user_b.trust_rating) / 2.0

    weights = [0.25, 0.20, 0.20, 0.15, 0.10, 0.10]
    scores = [time_score, location_score, taste_score, budget_score, personality_score, trust_score]

    return sum(w * s for w, s in zip(weights, scores))
```

#### 신뢰도 시스템

| 등급 | 점수 범위 | 혜택 | 조건 |
|:---|---:|:---|:---|
| 🌱 새싹 | 0~30 | 기본 매칭 | 가입 시 |
| 🌿 성장 | 31~60 | 우선 매칭, 배지 표시 | 매칭 5회 이상, 노쇼 0회 |
| 🌳 나무 | 61~85 | 프리미엄 매칭, 할인 쿠폰 | 매칭 20회 이상, 평점 4.0+ |
| ⭐ 별 | 86~100 | 앰버서더 자격, 추가 혜택 | 매칭 50회 이상, 평점 4.5+ |

### 2.5 기술 스택

| 계층 | 기술 | 선정 이유 |
|:---|:---|:---|
| **모바일 앱** | React Native + TypeScript | 크로스 플랫폼, 대학생 개발자 풀 풍부 |
| **웹 프론트엔드** | Next.js 14 | SEO 최적화, SSR/SSG 지원 |
| **API 서버** | Django REST Framework | 빠른 프로토타이핑, 풍부한 생태계 |
| **AI 서빙** | FastAPI | 비동기 처리, ML 모델 서빙 최적 |
| **AI/ML** | PyTorch + scikit-learn | 추천 엔진 + 매칭 알고리즘 |
| **NLP** | KoBERT (SKT) | 한국어 리뷰 감성 분석 |
| **데이터베이스** | PostgreSQL + PostGIS | 위치 기반 쿼리 최적화 |
| **캐시** | Redis | 실시간 매칭 상태 관리, 세션 |
| **메시지 큐** | RabbitMQ | 알림, 비동기 작업 처리 |
| **검색** | Elasticsearch | 메뉴/식당 풀텍스트 검색 |
| **인프라** | AWS (EC2, S3, RDS, CloudFront) | 초기 프리티어 활용, 스케일링 용이 |
| **CI/CD** | GitHub Actions + Docker | 대학생 무료 사용, 자동 배포 |
| **모니터링** | Sentry + Grafana | 오류 추적, 성능 대시보드 |
| **결제** | 토스페이먼츠 API | 대학생 간편결제 연동 |
| **지도** | 카카오맵 SDK | 국내 최적화, 무료 쿼터 충분 |
