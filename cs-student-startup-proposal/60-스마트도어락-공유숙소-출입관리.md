## 1. 문제 인식

### 공유 숙소 출입 관리 현황

| 지표 | 수치 | 출처 |
|------|------|------|
| 국내 공유숙박 등록 업소 수 | 38.7만개 | 문화체육관광부 (2025) |
| 키 교환 불편 호소 호스트 비율 | 74% | 에어비앤비코리아 (2025) |
| 비대면 체크인 선호 게스트 비율 | 82% | 야놀자 리서치 (2025) |
| 키분실/복제 관련 보안 사고 | 연 1.2만건 | 경찰청 (2024) |
| 호스트 평균 키 교환 소요 시간 | 건당 43분 | 자사 설문 (2025) |
| 다중 숙소 운영 호스트 비율 | 31.7% | 에어비앤비코리아 (2025) |
| 체크인/아웃 관련 게스트 불만 | 전체 리뷰 불만 중 28% | 야놀자 리서치 (2025) |
| 기존 디지털 도어락 보유율 | 42.3% | 한국스마트홈산업협회 (2025) |

### 출입 관리 문제의 심층 분석

공유 숙소 운영에서 키 교환은 호스트에게 가장 큰 운영 부담이다. 대면 키 교환은 시간 비용이 크고, 키박스는 보안 취약점이 있으며, 일반 디지털 도어락은 비밀번호 관리가 번거롭다. 특히 다수 숙소를 운영하는 호스트에게는 출입 관리의 자동화가 핵심 니즈이다.

**핵심 Pain Point 분석:**

1. **대면 키 교환의 시간 비용**: 호스트가 게스트와 직접 만나 키를 전달하는 데 건당 평균 43분이 소요된다. 하루 2~3팀의 체크인이 있는 경우 2시간 이상을 키 교환에만 소비해야 한다. 야간 체크인이나 늦은 도착의 경우 대기 시간은 더욱 길어진다.

2. **키박스의 보안 취약점**: 외부에 설치되는 키박스는 비밀번호 유출 위험이 높고, 물리적 파손 사례도 빈번하다. 이전 게스트가 비밀번호를 알고 있어 보안 사고의 원인이 된다.

3. **일반 디지털 도어락의 한계**: 기존 디지털 도어락은 OTA(숙박 예약 플랫폼)와 연동이 되지 않아, 매 예약 시마다 수동으로 비밀번호를 설정/변경/전달해야 한다. 다중 숙소 운영 시 이 작업은 매우 번거롭다.

4. **다중 숙소 관리의 복잡성**: 국내 에어비앤비 호스트 중 31.7%가 2개 이상의 숙소를 운영하며, 이 경우 각 숙소의 출입 관리가 개별적으로 이루어져 효율성이 극히 낮다.

5. **보안 사고 위험**: 키 분실이나 복제로 인한 보안 사고가 연간 1.2만 건에 달하며, 이는 호스트의 법적 책임과 재산 피해로 이어질 수 있다.

6. **게스트 경험 저하**: 체크인 과정에서의 불편함은 게스트 리뷰에 직접적으로 영향을 미치며, 전체 부정적 리뷰의 28%가 체크인/아웃 관련 불만이다.

**사용자 인터뷰 주요 발언:**

> "숙소를 3개 운영하는데, 체크인 시간이 겹치면 동시에 세 곳을 대응할 수가 없어요. 하루 종일 키 교환 때문에 뛰어다닙니다." -- 에어비앤비 슈퍼호스트 (38세, 서울 마포구)

> "새벽 1시에 도착했는데 호스트가 키를 맡겨둔 카페가 문을 닫아서 30분 동안 밖에서 기다렸어요." -- 외국인 게스트 (28세, 미국)

> "비밀번호를 변경하지 않아서 이전 게스트가 들어온 적이 있습니다. 그 이후로 보안에 대해 매우 신경 쓰게 되었어요." -- 게스트하우스 운영자 (45세, 서울 종로구)

### 주요 기능 상세

#### 2-1. 앱 기반 출입
게스트가 앱으로 BLE/NFC 기반 도어 개방, 별도 키 교환 불필요하다.

- **BLE 출입**: 게스트 앱에서 버튼 터치로 3m 이내에서 도어 개방. 앱 미설치 게스트는 NFC 태그나 임시 비밀번호 사용
- **NFC 카드**: 호텔 카드키처럼 사용 가능한 NFC 카드 제공 옵션. 외국인 게스트에게 특히 유용
- **다국어 지원**: 한국어, 영어, 중국어, 일본어 4개 국어 앱/안내 지원
- **게스트 온보딩**: 예약 확정 시 게스트에게 자동 메시지 발송 (앱 다운로드 링크 + 출입 안내)

#### 2-2. 시간제한 임시 비밀번호
예약 체크인/체크아웃 시간에 맞춰 자동 생성/만료되는 임시 비밀번호를 제공한다.

- **TOTP 기반**: 시간 기반 일회용 비밀번호 (RFC 6238) 알고리즘으로 보안성 확보
- **자동 생성/만료**: OTA 예약 데이터에서 체크인(15:00)/체크아웃(11:00) 시간을 읽어 자동으로 비밀번호 활성화/만료
- **비밀번호 정책**: 6자리 숫자, 예약 건별 유니크, 이전 비밀번호 자동 무효화
- **비상 비밀번호**: 호스트 전용 마스터 비밀번호, 비상 시 원격 해제 기능

#### 2-3. 실시간 출입 로그
누가, 언제 출입했는지 실시간 기록, 이상 출입 시 즉시 알림한다.

- **전체 출입 이력**: 날짜, 시간, 출입 방식(BLE/NFC/키패드), 사용자 식별 기록
- **이상 감지**: 비정상 시간 출입, 다회 실패, 강제 개방 시도 등 AI 기반 이상 감지
- **Push 알림**: 이상 출입 감지 시 호스트에게 즉시 Push 알림 + SMS 발송
- **감사 로그**: 30일 이상 출입 로그 보관, CSV/PDF 내보내기 지원

#### 2-4. 원격 관리
다수 숙소의 도어락을 하나의 대시보드에서 통합 관리, 원격 잠금/해제한다.

- **통합 대시보드**: 모든 숙소의 도어락 상태, 배터리, 예약 현황을 한 화면에서 확인
- **원격 잠금/해제**: 게스트 요청 시 원격으로 즉시 잠금/해제 가능

### 시장 성장 동인

K-관광 성장과 함께 공유숙박 시장이 확대되고, 비대면 체크인 트렌드가 가속화되면서 스마트 출입 관리 수요가 급증하고 있다. 2027년까지 공유숙박 50% 이상이 스마트 도어락을 도입할 것으로 예상된다.

**핵심 성장 동인:**

1. **K-관광 붐**: 한류 확산으로 방한 외국인 관광객 연 1,800만 명 돌파, 공유숙소 이용 증가
2. **비대면 체크인 트렌드**: 코로나19 이후 비대면 서비스 선호도 정착, 게스트 82%가 비대면 선호
3. **호스트 효율화 니즈**: 다중 숙소 운영 호스트의 증가로 자동화/원격 관리 수요 급증
4. **공유 오피스/코리빙 확장**: 공유 숙소뿐만 아니라 공유 오피스, 코리빙 하우스 등 새로운 세그먼트 성장
5. **보안 의식 강화**: 키 분실/복제 사고 증가로 디지털 보안 솔루션 수요 확대

### 보안 아키텍처 상세

KeyFlow의 보안은 다계층 방어 전략으로 설계되었다:

1. **하드웨어 보안**: 도어락 내부에 보안 칩(Secure Element) 탑재, 암호화 키를 하드웨어 레벨에서 보호
2. **통신 보안**: BLE 통신은 AES-128 CCM으로 암호화, 서버 통신은 TLS 1.3 필수 적용
3. **비밀번호 보안**: TOTP(Time-based One-Time Password) 기반으로 서버와 동기화된 시간 기반 비밀번호 생성. 비밀번호는 절대 평문 전송되지 않음
4. **접근 제어**: 역할 기반 접근 제어(RBAC)로 호스트/게스트/관리자 권한 분리
5. **감사 로그**: 모든 출입 기록 및 설정 변경 이력을 불변 로그로 보관 (30일 이상)
