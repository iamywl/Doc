## 1. 문제 인식 (Problem)

### 1.1 대학 팀프로젝트의 구조적 문제

대학 교육에서 팀프로젝트는 필수 과정이지만, 팀원 구성과 협업 과정에서 다양한 문제가 발생한다. 한국교육개발원에 따르면, 대학생의 78%가 팀프로젝트 팀원 구성에 불만을 경험하며, 이는 학습 효과 저하와 직결된다 [1].

특히 팀프로젝트의 성공 여부는 팀원의 역량 균형에 크게 좌우되는데, 현재 대학에서의 팀 구성 방식은 대부분 무작위 배정이거나 친분 기반 선택으로 이루어진다. 이로 인해 역량 불균형, 무임승차, 갈등 등이 빈번히 발생하며, 이는 학습 효과를 심각하게 저해한다.

무임승차 문제는 단순히 학업 성취에만 영향을 미치는 것이 아니라, 성실한 학생들의 동기 저하와 팀 협업 역량 발달의 장애물로 작용한다. 기업에서 팀워크를 중시하는 현 시대에, 대학에서의 불공정한 팀프로젝트 경험은 졸업 후 직장 적응에까지 부정적 영향을 미칠 수 있다.

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 팀프로젝트 팀원 구성 불만 경험 | **78%** | 한국교육개발원, 2024 [1] |
| 무임승차 경험 비율 | **32%** | 대학내일20대연구소, 2024 [2] |
| 학기당 평균 팀프로젝트 수 | **3.2건** | 한국대학교육협의회, 2025 [3] |
| 팀프로젝트로 인한 갈등 경험 | **64%** | 대학내일, 2024 [4] |
| 역량 불균형으로 인한 학습 효과 저하 | **45%** | 한국교육학술정보원, 2024 [5] |
| 팀 프로젝트 협업 도구 사용 만족도 | **35%** | 대학생 설문 (N=1,200), 2025 [6] |
| 팀프로젝트 때문에 학기 중 스트레스 상승 | **71%** | 한국청소년정책연구원, 2024 [9] |
| 교수의 기여도 측정 어려움 인식 | **82%** | 한국교육개발원, 2025 [10] |

#### 학생 목소리 (사용자 인터뷰 요약)

> "팀프로젝트에서 항상 같은 사람만 일하고, 나머지는 발표 전날에 나타난다. 기여도를 객관적으로 측정할 방법이 없어서 불공정하다고 느낀다." — CS 전공 3학년 A씨

> "팀원을 구하려면 학과 카톡방에 글을 올려야 하는데, 서로 모르는 상태에서 팀을 구성하면 역량 차이가 너무 크다." — 경영학과 2학년 B씨

> "팀프로젝트에서 학생들의 기여도를 정확히 측정할 방법이 없어서, 결국 모든 팀원에게 같은 점수를 줄 수밖에 없다." — 대학 교수 C씨

### 1.2 기존 도구의 한계

- **카카오톡/밴드**: 팀프로젝트 전용 기능 부재, 파일 관리 어려움, 대화 속에서 중요 정보가 묻힘
- **노션/슬랙**: 학습 곡선 존재, 기여도 측정 불가, 대학생 무료 플랜 제한적
- **대학 LMS (블랙보드, 캔버스)**: 팀 구성 기능 제한적, 실시간 협업 미지원, UI/UX가 구세대적
- **구글 드라이브/독스**: 문서 협업은 가능하나, 프로젝트 관리·기여도 분석은 불가능
- **GitHub**: 코드 협업에는 적합하나, 비개발 프로젝트에는 적용 어려움

### 1.3 페인포인트 매트릭스

| 이해관계자 | 페인포인트 | 심각도 | 현재 해결책 | 한계 |
|-----------|-----------|--------|-----------|------|
| **학생** | 역량에 맞는 팀원 찾기 어려움 | 높음 | 친분, 카톡방 | 역량 검증 불가 |
| **학생** | 무임승차 팀원으로 인한 부당함 | 매우 높음 | 교수에게 호소 | 증거 부족, 해결 어려움 |
| **학생** | 프로젝트 관리 도구 파편화 | 중간 | 카톡+노션+구글 | 3개 이상 도구 병행 |
| **교수** | 팀별 기여도 객관적 평가 어려움 | 높음 | 개별 면담, 자기평가서 | 시간 부족, 주관적 |
| **교수** | 팀 구성 최적화 어려움 | 중간 | 무작위 배정 | 역량 데이터 부재 |
| **해커톤 주최** | 참가자 팀 구성 관리 복잡 | 중간 | 수작업 관리 | 확장성 부족 |

### 2.2 핵심 기능

1. **AI 역량 기반 팀원 추천**: 기술 스택, 성격 유형(MBTI/DISC), 시간대, 과거 프로젝트 경험 등을 종합 분석하여 최적의 팀 구성 추천. 역량 다양성과 상호 보완성을 고려한 밸런싱 알고리즘 적용
2. **프로젝트 보드**: 칸반 보드, 간트 차트, 파일 공유, 실시간 문서 편집(CRDT 기반)을 통합 제공. 대학 팀프로젝트에 최적화된 템플릿 제공
3. **기여도 실시간 분석**: Git 커밋, 문서 작성량, 칸반 태스크 완료, 회의 참석률, 채팅 활동 등을 종합 추적하여 객관적 기여도를 수치화 및 시각화
4. **동료 평가 시스템**: 항목별 상호 평가(참여도, 소통, 전문성, 책임감)와 익명 피드백으로 공정한 성과 평가. 교수가 성적 반영에 활용 가능
5. **대학 LMS 연동**: 대학 공식 학습관리시스템(캔버스, 블랙보드 등)과 연동하여 과제 제출, 성적 반영, 출석 관리 자동화

### 2.3 기여도 분석 엔진 상세

TeamUp의 핵심 차별화 기술인 기여도 분석 엔진은 다음과 같은 다차원 지표를 종합한다:

| 기여도 지표 | 가중치 | 측정 방법 | 설명 |
|------------|--------|-----------|------|
| 태스크 완료 | 30% | 칸반 보드 태스크 상태 변경 추적 | 할당된 태스크의 완료율과 적시성 |
| 코드 기여 | 20% | GitHub 커밋, PR, 코드 리뷰 활동 | 코드 기반 프로젝트에서의 개발 기여 |
| 문서 작성 | 15% | 문서 편집 기록 (글자 수, 수정 횟수) | 보고서, 기획서 등 문서 작업 기여 |
| 회의 참석 | 15% | 회의 일정 참석 체크, 회의록 작성 | 팀 회의 참석률과 회의록 기여 |
| 소통 활동 | 10% | 채팅 메시지, 댓글, 피드백 수 | 팀 내 소통 참여도 |
| 동료 평가 | 10% | 팀원 상호 평가 점수 | 주관적 평가의 보정 역할 |
