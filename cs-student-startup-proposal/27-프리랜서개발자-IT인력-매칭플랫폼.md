## 1. 문제 인식 (Problem)

### 1.1 IT 프리랜서 시장의 구조적 비효율

국내 IT 프리랜서 시장은 급성장하고 있으나, 정보 비대칭과 신뢰 부재로 인한 구조적 비효율이 심각하다. 한국소프트웨어산업협회에 따르면, IT 프리랜서 인구는 2025년 기준 약 42만 명으로 매년 15% 이상 증가하고 있다 [1].

프리랜서 개발자 입장에서는 안정적인 프로젝트 확보가 가장 큰 과제이며, 기업 입장에서는 기술 역량이 검증된 개발자를 빠르게 찾는 것이 어렵다. 기존 인력 파견 업체는 30~40%에 달하는 높은 수수료를 부과하면서도, 기술 역량에 대한 객관적 검증 없이 이력서만으로 매칭을 진행하는 경우가 대부분이다.

특히 중소·스타트업 기업의 경우 전문 HR 인프라가 부족하여 적합한 프리랜서를 찾는 데 더 큰 어려움을 겪고 있으며, 이는 프로젝트 지연과 품질 저하로 이어지는 악순환을 만들고 있다.

#### 주요 통계

| 지표 | 수치 | 출처 |
|------|------|------|
| 국내 IT 프리랜서 인구 (2025) | **42만 명** | 한국소프트웨어산업협회, 2025 [1] |
| 프로젝트 수주에 어려움을 겪는 비율 | **72%** | 프리랜서코리아, 2024 [2] |
| 기업의 적합한 개발자 채용 소요 기간 | **평균 45일** | 사람인, 2025 [5] |
| 프리랜서-기업 간 계약 분쟁 경험 비율 | **34%** | 한국노동연구원, 2024 [6] |
| IT 프리랜서 평균 공백 기간 | **2.3개월** | 잡코리아, 2025 [7] |
| 기존 플랫폼 매칭 만족도 | **38%** | 원티드 리서치, 2024 [8] |
| 기업의 외주 개발자 재계약 의향률 | **45%** | 한국SW산업협회, 2025 [9] |
| 프리랜서 단가 하락 경험 비율 | **58%** | 프리랜서코리아, 2024 [10] |

#### 이해관계자 목소리

> "좋은 프로젝트를 찾기 위해 여러 플랫폼에 프로필을 올려놓지만, 실제로 연락이 오는 건 기술 스택이 맞지 않는 프로젝트가 대부분이다." — 프리랜서 백엔드 개발자 A씨 (경력 5년)

> "프리랜서를 뽑을 때 포트폴리오만 보고 판단할 수밖에 없어서, 실제 프로젝트에 투입되면 기대와 다른 경우가 많다." — 스타트업 CTO B씨

> "인력 파견 업체를 통하면 수수료가 30% 넘게 붙어서, 결국 개발자에게 돌아가는 단가가 낮아진다." — 중소기업 PM C씨

### 1.2 기존 매칭 서비스의 한계

- **위시켓/크몽**: 범용 플랫폼으로 IT 전문 매칭 정확도 낮음. 디자인, 마케팅 등 비개발 직군과 혼재되어 IT 특화 필터링 부족
- **원티드/로켓펀치**: 정규직 중심으로 프리랜서 프로젝트 매칭 기능 미흡. 단건 프로젝트 매칭보다는 채용 중심 모델
- **기존 인력 파견 업체**: 높은 수수료(30~40%), 기술 역량 검증 미비. 면접과 레퍼런스 체크만으로 기술력 판단
- **직접 네트워크**: 지인 추천에 의존하여 인맥 없는 신규 프리랜서의 진입 장벽 높음

### 1.3 프리랜서-기업 양측의 페인포인트

| 구분 | 프리랜서 페인포인트 | 기업 페인포인트 |
|------|-------------------|----------------|
| **매칭** | 기술 스택에 맞는 프로젝트 찾기 어려움 | 역량 검증된 개발자 탐색에 평균 45일 |
| **신뢰** | 기업 신뢰도 확인 어려움 (대금 미지급 우려) | 개발자 실력 검증 방법 부재 |
| **가격** | 불투명한 시세, 중개업체에 의한 단가 하락 | 적정 단가 판단 기준 없음 |
| **계약** | 불공정 계약 조건, 법적 보호 미비 | 일정 지연, 품질 이슈 발생 시 대응 어려움 |
| **관리** | 여러 프로젝트 동시 관리 도구 부재 | 원격 프리랜서 업무 현황 파악 어려움 |

### 2.2 핵심 기능

1. **AI 기술 스택 매칭**: 프로젝트 요구사항과 개발자 기술 역량을 분석하여 적합도 점수 산출. 기술 스택 유사도, 프로젝트 경험, 도메인 지식을 종합적으로 고려하는 다차원 매칭 알고리즘 적용
2. **GitHub 포트폴리오 분석**: 커밋 품질, 코드 리뷰 활동, 오픈소스 기여도, 코드 복잡도, 테스트 커버리지 등 15개 이상의 지표로 객관적 역량 평가. 단순 커밋 수가 아닌 코드 품질 중심의 분석
3. **에스크로 결제 시스템**: 마일스톤 기반 안전 결제로 분쟁 최소화. 프로젝트 진행 단계별로 대금을 분할 지급하며, 양측 확인 후 결제 진행
4. **프로젝트 관리 도구**: 칸반 보드, 타임 트래킹, 코드 리뷰, 일일 스탠드업 등 통합 관리 도구 제공. Slack, Jira 연동도 지원
5. **실시간 코딩 역량 테스트**: 기업 맞춤형 기술 평가 시스템으로 프로젝트 투입 전 실력 확인 가능

### 2.3 매칭 알고리즘 상세

DevMatch의 AI 매칭 엔진은 다음 요소를 종합하여 적합도 점수를 산출한다:

| 매칭 요소 | 가중치 | 분석 방법 |
|-----------|--------|-----------|
| 기술 스택 일치도 | 35% | 프로젝트 요구 기술과 개발자 보유 기술의 의미적 유사도 분석 |
| GitHub 활동 분석 | 25% | 커밋 품질, 코드 복잡도, PR 활동, 이슈 해결 패턴 |
| 프로젝트 경험 | 20% | 유사 도메인/규모 프로젝트 수행 이력 |
| 평가 이력 | 15% | 이전 프로젝트 클라이언트 평가 점수 |
| 가용성/일정 | 5% | 현재 프로젝트 상태, 희망 시작일 |
