## 1. 문제 인식

### 대학생 건강 관리 현황

| 지표 | 수치 | 출처 |
|------|------|------|
| 대학생 건강검진 수검률 | 28.4% | 대학보건학회 (2025) |
| 교내 보건센터 이용률 | 15.7% | 한국대학교육협의회 (2025) |
| 건강 관리 필요성 인식 | 82.3% | 대학내일 20대연구소 (2025) |
| 건강검진 미수검 사유 (시간 부족) | 56.8% | 국민건강보험공단 (2024) |
| 대학생 만성피로 호소율 | 71.2% | 대한가정의학회 (2025) |
| 대학생 정신건강 위험군 비율 | 24.7% | 교육부 (2025) |
| 보건센터 존재 인지율 | 63.2% | 한국대학교육협의회 (2025) |
| 건강기록 관리 경험 (앱/웹) | 12.1% | 한국건강증진개발원 (2024) |

대학생은 건강 관리의 필요성을 인식하지만 시간 부족, 정보 부재, 번거로운 예약 절차로 건강검진을 미루는 경우가 많다. 대학 보건센터와 외부 의료기관의 연계가 부족하여 건강 데이터가 분산되고, 지속적인 건강 관리가 어렵다.

### 대학생 건강 관리 Pain Point 분석

| Pain Point | 비율 | 상세 설명 |
|------------|------|-----------|
| 건강검진 예약이 번거롭다 | 56.8% | 검진 기관 검색 → 비교 → 예약 과정이 복잡, 대학생 맞춤 패키지 부재 |
| 어디서 검진받아야 하는지 모르겠다 | 43.2% | 대학 근처 검진 기관 정보 부족, 가격/항목 비교 어려움 |
| 건강 기록이 분산되어 있다 | 67.5% | 보건센터, 외부 병원, 건강보험공단 기록이 각각 별도 관리 |
| 보건센터를 어떻게 이용하는지 모르겠다 | 36.8% | 이용 절차, 진료 가능 항목, 운영 시간 등 정보 부족 |
| 검진 결과를 어떻게 해석해야 하는지 모르겠다 | 52.1% | 수치 의미, 정상 범위, 생활 습관 개선 방법 이해 부족 |
| 건강 관리를 지속할 동기가 없다 | 38.4% | 일회성 검진 후 후속 관리 부재, 또래 비교/동기 부여 기능 없음 |

### 사용자 목소리

> "건강검진을 받아야 한다는 건 알지만, 수업 사이에 시간 내기도 어렵고 어디서 받아야 하는지도 모르겠어요." - 22세 대학생 (공학계열)

> "보건센터가 어디 있는지도 몰랐고, 가봤자 감기약만 줄 줄 알았어요. 실제로 무료 검진도 해준다는 걸 나중에 알았어요." - 21세 대학생 (인문계열)

> "학생들 건강검진 수검률이 너무 낮아서 고민인데, 예약부터 결과 관리까지 원스톱으로 해주는 시스템이 있으면 좋겠습니다." - 대학 보건센터장

### 핵심 기능 상세

- **건강검진 예약**: 대학 근처 건강검진 기관 비교, 원클릭 예약, 검진 항목 맞춤 추천. 대학생 맞춤 검진 패키지(기본 혈액검사, 시력, 청력, 체성분, 스트레스 검사 등)를 저렴한 가격에 제공. 수업 시간표와 연동하여 빈 시간에 예약 가능한 슬롯 자동 추천.

- **건강기록 통합**: 건강보험공단, 보건센터, 외부 병원 검진 기록 FHIR 표준 통합 관리. HL7 FHIR R4 표준 기반으로 다양한 의료기관의 건강 데이터를 단일 타임라인으로 통합. 대학 입학 전 건강검진부터 졸업 후까지 평생 건강 기록의 시작점 제공.

- **교내 진료 연동**: 보건센터 실시간 대기 현황, 온라인 예약, 처방 기록 관리. 보건센터 방문 전 증상 사전 입력으로 대기 시간 단축, 처방 이력 기반 재방문 시 신속한 진료 지원.

- **건강 인사이트**: 검진 결과 기반 건강 위험도 분석, 생활 습관 개선 코칭. XGBoost 모델이 검진 수치 + 생활 습관 설문 데이터를 분석하여 "당신의 건강 위험도" 점수 산출, 또래 평균 대비 비교, 맞춤 생활 습관 개선 리포트 제공.

### FHIR 건강기록 통합 상세

CampusHealth의 핵심 기술적 차별점은 HL7 FHIR R4 표준 기반의 건강기록 통합이다.

**통합 대상 데이터**:
| 데이터 소스 | 연동 방식 | 포함 데이터 |
|-------------|-----------|-------------|
| 국민건강보험공단 | 공공 API | 건강검진 결과, 진료 이력, 처방 이력 |
| 대학 보건센터 | EMR 연동 API | 교내 진료 기록, 예방접종, 상담 기록 |
| 외부 검진기관 | FHIR API | 건강검진 상세 결과 (혈액, 영상 등) |
| 사용자 입력 | 앱 직접 입력 | 생활 습관 설문, 증상 기록, 약 복용 |
| 웨어러블 기기 | HealthKit/Health Connect | 걸음 수, 수면, 심박수 |

**FHIR 리소스 매핑**:
- Patient → 학생 기본 정보
- Observation → 검진 수치 (혈압, 혈당, 콜레스테롤 등)
- DiagnosticReport → 검진 결과 리포트
- Encounter → 진료/상담 이력
- MedicationRequest → 처방 기록

### CampusHealth의 핵심 경쟁 우위

1. **대학생 특화 유일한 솔루션**: 기존 의료 예약 앱은 범용 서비스로, 대학생의 시간표, 학사 일정, 보건센터 연동을 고려하지 않음.
2. **FHIR 표준 건강기록**: 국내 대학생 대상 서비스 중 유일하게 국제 표준(HL7 FHIR R4)으로 건강기록을 통합, 졸업 후에도 활용 가능한 평생 건강 기록.
3. **보건센터 SaaS**: 보건센터에 EMR + 통계 대시보드를 제공하여, 대학 측에 실질적 운영 효율 개선 가치 전달.
4. **또래 비교 인사이트**: "같은 나이/성별 학생 대비 나의 건강 수준" 비교를 통한 동기부여 (gamification).

### 팀 역량 상세

**조은서 (대표/PM)**: 의료 IT 기업에서 FHIR 기반 EMR 시스템 개발 프로젝트에 인턴으로 참여, 헬스케어 데이터 표준화 경험. 대학 헬스케어 해커톤에서 "캠퍼스 건강 관리 플랫폼" 기획으로 1위 수상. 보건센터 실무자 인터뷰 20건 수행.

**신현준 (백엔드 개발)**: Spring Boot/Kotlin 3년 경력, HAPI FHIR 서버 구축 및 운영 경험. 의료 데이터 암호화, PIPA 준수 시스템 설계. 마이크로서비스 아키텍처 설계 전문.

**류성민 (데이터 분석)**: 보건학 석사 과정에서 대학생 건강 검진 데이터 분석 연구 수행, XGBoost 기반 건강 위험도 예측 모델 개발 경험. 논문 2편 발표.
