# 가상화, 컨테이너 & 서버리스 - 논문 요약 및 링크

## 1. 가상화 (Virtualization)

### [Xen and the Art of Virtualization](https://dl.acm.org/doi/10.1145/945445.945462)
- **저자**: Paul Barham, Boris Dragovic, Keir Fraser, Steven Hand, Tim Harris, Alex Ho, Rolf Neugebauer, Ian Pratt, Andrew Warfield
- **연도/학회**: 2003 / ACM SOSP
- **핵심 요약**: x86에서 준가상화(paravirtualization) 기법을 통해 베어메탈 대비 수 퍼센트 이내의 오버헤드로 최대 100개 VM을 동시 실행할 수 있는 Xen VMM을 제안하였다. Amazon EC2 등 주요 클라우드 플랫폼의 기반 기술이 되었다.

### [kvm: the Linux Virtual Machine Monitor](https://www.kernel.org/doc/ols/2007/ols2007v1-pages-225-230.pdf)
- **저자**: Avi Kivity, Yaniv Kamay, Dor Laor, Uri Lublin, Anthony Liguori
- **연도/학회**: 2007 / Linux Symposium, Ottawa
- **핵심 요약**: 하드웨어 가상화 확장(Intel VT-x, AMD-V)을 활용하여 Linux 커널에 하이퍼바이저 기능을 추가하는 KVM 서브시스템을 소개한다. Linux 커널에 직접 통합된 최초의 주류 하이퍼바이저이다.

### [A Full GPU Virtualization Solution with Mediated Pass-Through (gVirt)](https://www.usenix.org/conference/atc14/technical-sessions/presentation/tian)
- **저자**: Kun Tian, Yaozu Dong, David Cowperthwaite
- **연도/학회**: 2014 / USENIX ATC
- **핵심 요약**: Intel 내장 GPU를 위한 Mediated Pass-Through 기법으로 네이티브 대비 최대 95% 성능을 달성하는 전체 GPU 가상화 솔루션 gVirt를 제안하였다. 현대 vGPU 및 SR-IOV 기반 GPU 공유 기술의 학술적 기반이다.

## 2. VM 라이브 마이그레이션 (VM Live Migration)

### [Post-copy Live Migration of Virtual Machines](https://dl.acm.org/doi/10.1145/1618525.1618528)
- **저자**: Michael R. Hines, Umesh Deshpande, Kartik Gopalan
- **연도/학회**: 2009 / ACM SIGOPS Operating Systems Review (VEE 2009 Best Paper)
- **핵심 요약**: VM의 프로세서 상태를 먼저 전송하고 메모리를 나중에 가져오는 post-copy 라이브 마이그레이션을 제안하여, pre-copy 방식 대비 전송 페이지 수, 총 마이그레이션 시간, 네트워크 오버헤드에서 우수한 결과를 보였다.

## 3. 컨테이너 기초 기술 (Container Fundamentals)

### [Docker: Lightweight Linux Containers for Consistent Development and Deployment](https://www.linuxjournal.com/content/docker-lightweight-linux-containers-consistent-development-and-deployment)
- **저자**: Dirk Merkel
- **연도/학회**: 2014 / Linux Journal
- **핵심 요약**: Linux 컨테이너(LXC) 기반으로 애플리케이션과 의존성을 경량 컨테이너에 패키징하는 Docker 시스템을 소개하며, cgroups와 namespaces 기반 격리의 실용적 활용을 대중화하였다.

### [Cntr: Lightweight OS Containers](https://www.usenix.org/conference/atc18/presentation/thalheim)
- **저자**: Jorg Thalheim, Pramod Bhatotia, Pedro Fonseca, Baris Kasikci
- **연도/학회**: 2018 / USENIX ATC
- **핵심 요약**: 컨테이너 이미지를 "slim"(주 애플리케이션)과 "fat"(개발 도구)으로 분리하여 런타임에 동적 결합하는 CNTR 시스템을 제안하여, Docker Hub Top-50 이미지 크기를 평균 66.6% 절감하였다.

- **연도/학회**: 2016 / USENIX FAST
- **핵심 요약**: 컨테이너 시작 시간의 76%를 차지하는 패키지 풀링에서 실제 읽히는 데이터가 6.4%에 불과함을 밝히고, 지연 로딩 기반 Slacker 드라이버로 개발 사이클을 20배, 배포 사이클을 5배 가속하였다.

## 5. 컨테이너 오케스트레이션 (Container Orchestration)

### [Large-scale Cluster Management at Google with Borg](https://dl.acm.org/doi/10.1145/2741948.2741964)
- **저자**: Abhishek Verma, Luis Pedrosa, Madhukar R. Korupolu, David Oppenheimer, Eric Tune, John Wilkes
- **연도/학회**: 2015 / ACM EuroSys
- **핵심 요약**: Google의 클러스터 관리 시스템 Borg를 소개하며, 수만 대 머신에서 수십만 개 작업을 실행하는 설계를 다룬다. Kubernetes의 직접적 전신으로 현대 컨테이너 오케스트레이션의 핵심 개념을 정립하였다.

### [Omega: Flexible, Scalable Schedulers for Large Compute Clusters](https://dl.acm.org/doi/10.1145/2465351.2465386)
- **저자**: Malte Schwarzkopf, Andy Konwinski, Michael Abd-El-Malek, John Wilkes
- **연도/학회**: 2013 / ACM EuroSys
- **핵심 요약**: 공유 상태 기반 병렬 스케줄링과 lock-free 낙관적 동시성 제어를 활용하여 모놀리식 스케줄러의 한계를 극복하는 Omega 시스템을 제안하였다.

### [Mesos: A Platform for Fine-Grained Resource Sharing in the Data Center](https://www.usenix.org/conference/nsdi11/mesos-platform-fine-grained-resource-sharing-data-center)
- **저자**: Benjamin Hindman, Andy Konwinski, Matei Zaharia, Ali Ghodsi, Anthony D. Joseph, Randy Katz, Scott Shenker, Ion Stoica
- **연도/학회**: 2011 / USENIX NSDI
- **핵심 요약**: Hadoop, MPI 등 다양한 프레임워크가 하나의 클러스터를 세밀하게 공유할 수 있는 2단계 스케줄링(resource offers) 기반 Mesos 플랫폼을 제안하였다.

### [Borg, Omega, and Kubernetes](https://dl.acm.org/doi/10.1145/2890784)
- **저자**: Brendan Burns, Brian Grant, David Oppenheimer, Eric Brewer, John Wilkes
- **연도/학회**: 2016 / ACM Queue / Communications of the ACM
- **핵심 요약**: Google의 Borg, Omega, Kubernetes 세 시스템의 설계 철학과 10년 이상의 컨테이너 관리 경험에서 얻은 교훈을 정리한 클라우드 네이티브 컴퓨팅 필수 문헌이다.

## 8. 컨테이너 보안 및 런타임 격리 (Container Security & Runtime Isolation)

### [The True Cost of Containing: A gVisor Case Study](https://www.usenix.org/conference/hotcloud19/presentation/young)
- **저자**: Eric Young 등
- **연도/학회**: 2019 / USENIX HotCloud
- **핵심 요약**: gVisor의 성능 비용을 정량적으로 분석하여, 단순 시스템 콜이 2.2배 이상, 외부 tmpfs 파일 열기/닫기가 216배 느린 것을 밝혔다.

### [Performance and Isolation Analysis of RunC, gVisor and Kata Containers Runtimes](https://link.springer.com/article/10.1007/s10586-021-03517-8)
- **저자**: Wang, Du 등
- **연도/학회**: 2022 / Cluster Computing (Springer)
- **핵심 요약**: RunC, gVisor, Kata Containers 세 런타임의 성능과 격리 수준을 비교 분석하여, gVisor는 격리 최우수이나 I/O 성능 저하가 크고 Kata는 VM 수준 격리와 상대적 성능 우위를 보임을 확인하였다.

### [Performance Evaluation of Container Runtimes](https://www.scitepress.org/Papers/2020/93404/93404.pdf)
- **저자**: Lennart Espe, Anshul Jindal, Vladimir Podolskiy, Michael Gerndt
- **연도/학회**: 2020 / CLOSER (SciTePress)
- **핵심 요약**: containerd와 CRI-O를 runc 및 gVisor에서 평가하여, containerd가 CPU/메모리 레이턴시/확장성에서 우수하고 CRI-O가 파일시스템 연산에서 효율적임을 보였다.

## 9. 마이크로VM (MicroVM)

### [Firecracker: Lightweight Virtualization for Serverless Applications](https://www.usenix.org/conference/nsdi20/presentation/agache)
- **저자**: Alexandru Agache, Marc Brooker, Andreea Florescu, Alexandra Iordache, Anthony Liguori, Rolf Neugebauer, Phil Piwonka, Diana-Maria Popa
- **연도/학회**: 2020 / USENIX NSDI
- **핵심 요약**: AWS Lambda와 Fargate를 구동하는 Rust 기반 경량 VMM으로, 컨테이너당 5MB 미만 메모리 오버헤드, 125ms 이내 부팅, 초당 150개 MicroVM 생성이 가능하다. 컨테이너의 속도와 VM의 격리를 결합한 새로운 패러다임을 확립하였다.

### [Blending Containers and Virtual Machines: A Study of Firecracker and gVisor](https://dl.acm.org/doi/10.1145/3381052.3381315)
- **저자**: Anjali, Tyler Caraza-Harter, Michael M. Swift
- **연도/학회**: 2020 / ACM VEE
- **핵심 요약**: Docker 컨테이너, gVisor, Firecracker microVM이 Linux 커널 서비스를 어떻게 다르게 사용하는지 세밀한 코드 커버리지 분석을 통해 비교한 최초의 심층 연구이다.

### [My VM is Lighter (and Safer) than your Container (LightVM)](https://dl.acm.org/doi/10.1145/3132747.3132763)
- **저자**: Filipe Manco 등
- **연도/학회**: 2017 / ACM SOSP
- **핵심 요약**: Xen 기반 LightVM으로 2.3ms 만에 VM을 부팅할 수 있으며, 이는 Docker보다 두 자릿수 빠르다. VM이 컨테이너보다 빠르고 안전할 수 있음을 실증적으로 보였다.

## 10. 유니커널 (Unikernel)

### [Unikernels: Rise of the Virtual Library Operating System (MirageOS)](https://queue.acm.org/detail.cfm?id=2566628)
- **저자**: Anil Madhavapeddy, David J. Scott
- **연도/학회**: 2013 / ACM Queue
- **핵심 요약**: "유니커널" 용어를 최초로 정의하고, OCaml 기반 MirageOS를 통해 타입 안전한 단일 목적 VM 이미지를 생성하는 라이브러리 운영체제 개념을 제안하였다.

### [Unikraft: Fast, Specialized Unikernels the Easy Way](https://dl.acm.org/doi/10.1145/3447786.3456248)
- **저자**: Simon Kuenzer, Vlad-Andrei Badoiu, Hugo Lefeuvre, Sharan Santhanam, Alexander Jung 등
- **연도/학회**: 2021 / ACM EuroSys (Best Paper Award)
- **핵심 요약**: OS 프리미티브를 완전히 모듈화한 마이크로 라이브러리 OS로, nginx/SQLite/Redis를 Linux 대비 1.7-2.7배 성능 향상, 이미지 ~1MB, 부팅 ~1ms를 달성하였다.

### [OSv: Optimizing the Operating System for Virtual Machines](https://www.usenix.org/conference/atc14/technical-sessions/presentation/kivity)
- **저자**: Avi Kivity, Dor Laor, Glauber Costa, Pekka Enberg, Nadav Har'El, Don Marti, Vlad Zolotarov
- **연도/학회**: 2014 / USENIX ATC
- **핵심 요약**: 바이너리 호환성을 제공하여 JVM, Python 등 기존 애플리케이션을 수정 없이 실행할 수 있는 클라우드 최적화 OS로, Firecracker에서 4-5ms 부팅이 가능하다.

## 12. 서버리스 콜드 스타트 (Serverless Cold Start)

### [Cold Start Latency in Serverless Computing: A Systematic Review, Taxonomy, and Future Directions](https://arxiv.org/abs/2310.08437)
- **저자**: Muhammed Golec, Guneet Kaur Walia, Mohit Kumar, Felix Cuadrado, Sukhpal Singh Gill, Steve Uhlig
- **연도/학회**: 2024 / ACM Computing Surveys (arXiv: 2310.08437)
- **핵심 요약**: 서버리스 콜드 스타트 레이턴시 완화 메커니즘을 애플리케이션/체크포인트/호출 시간 예측/캐시 기반 4가지로 분류하고, AI/ML 기반 솔루션이 새로운 연구 방향으로 부상하고 있음을 제시하였다.

### [FaaSLight: General Application-level Cold-start Latency Optimization for FaaS](https://dl.acm.org/doi/10.1145/3585007)
- **저자**: (다수 저자)
- **연도/학회**: 2023 / ACM Transactions on Software Engineering and Methodology
- **핵심 요약**: FaaS 애플리케이션에서 필수 코드만 로딩하여 코드 로딩 레이턴시를 최대 78.95%, 전체 응답 레이턴시를 최대 42.05% 감소시키는 범용 도구를 제안하였다.

### [SOCK: Rapid Task Provisioning with Serverless-Optimized Containers](https://www.usenix.org/conference/atc18/presentation/oakes)
- **저자**: Edward Oakes, Leon Yang, Dennis Zhou, Kevin Houck, Tyler Harter, Andrea C. Arpaci-Dusseau, Remzi H. Arpaci-Dusseau
- **연도/학회**: 2018 / USENIX ATC
- **핵심 요약**: Linux 컨테이너 프리미티브의 확장성 병목을 분석하고, Docker 대비 18배 빠른 서버리스 최적화 컨테이너 SOCK을 구현하여 AWS Lambda 대비 2.8배 오버헤드를 감소시켰다.

### [Catalyzer: Sub-millisecond Startup for Serverless Computing with Initialization-less Booting](https://dl.acm.org/doi/10.1145/3373376.3378512)
- **저자**: Dong Du, Tianyi Yu, Yubin Xia, Binyu Zang 등 (SJTU IPADS)
- **연도/학회**: 2020 / ACM ASPLOS
- **핵심 요약**: 체크포인트 이미지에서 함수 인스턴스를 복원하는 init-less 부팅과 sfork(sandbox fork) OS 프리미티브로 gVisor 기반 1ms 미만 서버리스 시작 레이턴시를 달성하였다.

## 13. FaaS (Function-as-a-Service)

### [Serverless in the Wild: Characterizing and Optimizing the Serverless Workload at a Large Cloud Provider](https://www.usenix.org/conference/atc20/presentation/shahrad)
- **저자**: Mohammad Shahrad, Rodrigo Fonseca, Inigo Goiri 등
- **연도/학회**: 2020 / USENIX ATC (Community Award)
- **핵심 요약**: Azure Functions의 전체 프로덕션 FaaS 워크로드를 분석하여 호출 빈도가 8자릿수에 걸쳐 있고, 50%의 함수가 1초 이내 실행됨을 밝히며, 적응형 keep-alive 정책을 제안하였다.

### [Architectural Implications of Function-as-a-Service Computing](https://dl.acm.org/doi/10.1145/3352460.3358296)
- **저자**: Mohammad Shahrad, Jonathan Balkind, David Wentzlaff
- **연도/학회**: 2019 / IEEE/ACM MICRO
