## 1. 클라우드 보안 일반

### 1-1. Cloud Native Security Report 2024

- **제목**: 2024 Cloud Native Security Report
- **저자**: Linux Foundation / CNCF
- **연도**: 2024
- **출처**: Linux Foundation Research
- **요약**: 클라우드 네이티브 환경의 보안 현황을 종합적으로 조사한 보고서이다. 멀티 클라우드 도입율이 78%로 전년 대비 7%p 증가했으며, 보안 및 컴플라이언스가 최대 과제(61%)로 부각되었다. AI 기반 보안 위협 탐지에 대한 관심도 급격히 증가하고 있음을 보여준다.
- **중요성**: 클라우드 네이티브 보안의 산업 전반적 동향과 실제 현장의 과제를 파악할 수 있는 핵심 자료이다.

### 1-2. IDC Cloud Security Research 2024

- **제목**: New IDC Research Highlights a Major Cloud Security Shift
- **저자**: IDC / Microsoft Security
- **연도**: 2024-2025
- **출처**: Microsoft Security Blog / IDC
- **요약**: 조직들이 2024년에 평균 9건의 클라우드 보안 인시던트를 경험했으며, 89%가 전년 대비 증가했다고 보고했다. 통합 플랫폼 기반 보안으로의 패러다임 전환이 가속화되고 있음을 강조한다.
- **중요성**: 기업 현장에서의 클라우드 보안 인시던트 빈도와 대응 전략 변화를 정량적으로 파악할 수 있다.

## 2. 기밀 컴퓨팅 / TEE

### 2-1. Survey of Research on Confidential Computing

- **제목**: Survey of Research on Confidential Computing
- **저자**: Feng et al.
- **연도**: 2024
- **출처**: IET Communications (Wiley Online Library)
- **요약**: 프로세스 수준 TEE(Intel SGX)와 시스템 수준 TEE(Intel TDX, AMD SEV, Arm CCA)를 비교 분석한 서베이 논문이다. SGX는 TCB가 작지만 사이드 채널 공격 표면이 넓고, TDX/SEV는 TCB가 크지만 호환성이 우수하다는 트레이드오프를 분석한다.
- **중요성**: 기밀 컴퓨팅 기술의 전체 지형을 조감할 수 있는 포괄적인 서베이로, TEE 기술 선택 시 참고할 핵심 자료이다.

### 2-2. An Experimental Evaluation of TEE Technology

- **제목**: An Experimental Evaluation of TEE Technology Evolution: Benchmarking Transparent Approaches based on SGX, SEV, and TDX
- **저자**: (ScienceDirect 게재)
- **연도**: 2025
- **출처**: Computers & Security (ScienceDirect) / arXiv:2408.00443
- **요약**: TDX, SEV, Gramine-SGX, Occlum-SGX의 성능 오버헤드와 자원 사용량을 다양한 운영 시나리오에서 실험적으로 비교 평가한 논문이다. 투명한(transparent) TEE 접근 방식의 실용성을 정량적으로 검증한다.
- **중요성**: 기밀 컴퓨팅 도입 시 발생하는 성능 오버헤드를 정량적으로 이해하는 데 필수적인 벤치마크 자료이다.

### 2-3. WeSee: Using Malicious #VC Interrupts to Break AMD SEV-SNP

- **제목**: WeSee: Using Malicious #VC Interrupts to Break AMD SEV-SNP
- **저자**: (IEEE S&P 2024 발표)
- **연도**: 2024
- **출처**: IEEE Symposium on Security and Privacy (S&P 2024)
- **요약**: AMD SEV-SNP의 #VC 인터럽트 메커니즘을 악용하여 기밀 VM의 보안을 침해하는 공격 기법을 제시한 논문이다. 악의적인 하이퍼바이저가 인터럽트를 조작하여 게스트 VM 내부 데이터에 접근할 수 있음을 증명한다.
- **중요성**: 최신 TEE 기술의 실질적 보안 한계를 보여주는 중요한 공격 연구로, 기밀 컴퓨팅의 보안 모델 개선에 핵심적인 시사점을 제공한다.

### 2-4. HECKLER: Breaking Confidential VMs with Malicious Interrupts

- **제목**: HECKLER: Breaking Confidential VMs with Malicious Interrupts
- **저자**: (USENIX Security 2024 발표)
- **연도**: 2024
- **출처**: 33rd USENIX Security Symposium
- **요약**: 악의적인 인터럽트를 통해 기밀 VM을 공격하는 새로운 방법론을 제시한다. AMD SEV-SNP 및 Intel TDX 환경에서 하이퍼바이저가 인터럽트를 이용하여 기밀 VM의 실행 흐름을 조작할 수 있음을 시연한다.
- **중요성**: TEE 기반 기밀 컴퓨팅의 인터럽트 처리 취약점을 체계적으로 분석한 논문으로, 방어 메커니즘 설계에 직접적인 영향을 준다.

### 2-5. COIN Attacks: On Insecurity of Enclave Untrusted Interfaces in SGX

- **제목**: COIN Attacks: On Insecurity of Enclave Untrusted Interfaces in SGX
- **저자**: (ASPLOS '20 발표)
- **연도**: 2020
- **출처**: ACM ASPLOS '20
- **요약**: Intel SGX 엔클레이브의 비신뢰 인터페이스(untrusted interface)를 체계적으로 분석하여 보안 취약점을 도출한 논문이다. 엔클레이브와 외부 환경 간의 경계면에서 발생하는 다양한 공격 벡터를 분류하고 실증한다.
- **중요성**: SGX 기반 애플리케이션 개발 시 반드시 고려해야 할 인터페이스 보안 문제를 체계적으로 정리한 핵심 참고 자료이다.

## 3. 사이드 채널 공격

### 3-1. Spectre Attacks: Exploiting Speculative Execution

- **제목**: Spectre Attacks: Exploiting Speculative Execution
- **저자**: Paul Kocher, Jann Horn, Anders Fogh, Daniel Genkin, Daniel Gruss, Werner Haas, Mike Hamburg, Moritz Lipp, Stefan Mangard, Thomas Prescher, Michael Schwarz, Yuval Yarom
- **연도**: 2019 (원 발표 2018)
- **출처**: IEEE S&P 2019 / spectreattack.com
- **요약**: CPU의 투기적 실행(speculative execution) 메커니즘을 악용하여 메모리 격리 경계를 위반하고 민감한 데이터를 유출하는 공격 기법이다. 마이크로아키텍처 코버트 채널을 통해 데이터를 추출하며, 아키텍처 수준의 문제이므로 단일 패치로 해결이 불가능하다.
- **중요성**: 현대 CPU 보안의 근본적 한계를 드러낸 역사적 논문으로, 클라우드 환경에서 테넌트 간 격리에 심각한 위협을 제기한다.

### 3-2. Meltdown: Reading Kernel Memory from User Space

- **제목**: Meltdown: Reading Kernel Memory from User Space
- **저자**: Moritz Lipp, Michael Schwarz, Daniel Gruss, Thomas Prescher, Werner Haas, Anders Fogh, Jann Horn, Stefan Mangard, Paul Kocher, Daniel Genkin, Yuval Yarom, Mike Hamburg
- **연도**: 2018
- **출처**: USENIX Security 2018 / meltdownattack.com
- **요약**: 비순차 실행(out-of-order execution)을 악용하여 사용자 공간에서 커널 메모리를 읽을 수 있는 공격이다. 권한이나 특권 없이 임의의 물리 메모리를 읽을 수 있으며, 노트북, 데스크탑, 모바일 폰, 퍼블릭 클라우드 서버에서 모두 작동한다.
- **중요성**: KPTI(Kernel Page Table Isolation) 등 근본적인 커널 보안 아키텍처 변화를 촉발한 논문으로, 클라우드 보안 아키텍처에 지대한 영향을 미쳤다.

### 3-3. Modern Hardware Security: A Review of Attacks and Countermeasures

- **제목**: Modern Hardware Security: A Review of Attacks and Countermeasures
- **저자**: (arXiv:2501.04394)
- **연도**: 2025
- **출처**: arXiv
- **요약**: Spectre, Meltdown 이후 발견된 마이크로아키텍처 공격과 대응책을 종합적으로 리뷰한 논문이다. 캐시 기반 사이드 채널, 투기적 실행 공격, 그리고 하드웨어/소프트웨어 수준의 방어 기법을 포괄적으로 분석한다.
- **중요성**: 최신 하드웨어 보안 위협의 전체 지형을 파악하고 향후 방어 전략을 수립하는 데 유용한 종합 자료이다.

## 4. 제로 트러스트 아키텍처

### 4-1. NIST SP 800-207: Zero Trust Architecture

- **제목**: NIST Special Publication 800-207: Zero Trust Architecture
- **저자**: Scott Rose, Oliver Borchert, Stu Mitchell, Sean Connelly (NIST)
- **연도**: 2020
- **출처**: NIST (National Institute of Standards and Technology)
- **요약**: 제로 트러스트 보안 패러다임의 공식적 정의와 참조 아키텍처를 제시하는 NIST 표준 문서이다. 정적 네트워크 경계 방어에서 사용자, 자산, 리소스 중심의 보안 모델로의 전환을 체계적으로 기술한다.
- **중요성**: 제로 트러스트 아키텍처의 사실상 표준(de facto standard) 문서로, 정부 및 기업의 보안 정책 수립에 가장 널리 참조되는 프레임워크이다.

### 4-2. BeyondCorp: A New Approach to Enterprise Security

- **제목**: BeyondCorp: A New Approach to Enterprise Security
- **저자**: Rory Ward, Betsy Beyer (Google)
- **연도**: 2014
- **출처**: ;login: (USENIX Magazine)
- **요약**: 2009년 중국 APT 공격(Operation Aurora) 이후 Google이 구현한 제로 트러스트 아키텍처이다. 네트워크 경계에서 개별 사용자로 접근 제어를 이동시켜 VPN 없이도 어디서나 안전하게 업무를 수행할 수 있도록 한 혁신적 보안 모델이다.
- **중요성**: 제로 트러스트의 실제 대규모 구현 사례로, NIST SP 800-207의 이론적 토대를 제공한 산업계 최초의 본격적 ZTA 구현이다.

### 4-3. Zero Trust Architecture (ZTA): A Comprehensive Survey

- **제목**: Zero Trust Architecture (ZTA): A Comprehensive Survey
- **저자**: IEEE Xplore 게재
- **연도**: 2022
- **출처**: IEEE Access (IEEE Xplore, DOI: 10.1109/ACCESS.2022.3174679)
- **요약**: 제로 트러스트 보안 패러다임의 기본 원칙을 기술적 관점에서 제시하고, 성공적 구현을 위한 다양한 옵션을 리뷰한 서베이이다. 인증, 접근 제어, 암호화, 마이크로 세그멘테이션, 보안 자동화 등 핵심 구성 요소를 심층 분석한다.
- **중요성**: ZTA의 핵심 기술 요소를 학술적 관점에서 체계적으로 정리한 포괄적 서베이 논문이다.

### 4-4. Zero Trust Architecture: A Systematic Literature Review (2025)

- **제목**: Zero Trust Architecture: A Systematic Literature Review
- **저자**: Gambo, Almulhem et al.
- **연도**: 2025
- **출처**: arXiv:2503.11659 / TechRxiv
- **요약**: 2016-2025년 사이 출판된 74편의 피어 리뷰 논문을 분석한 체계적 문헌 조사이다. 클라우드 컴퓨팅(24편), IoT(11편), 헬스케어(7편), 엔터프라이즈(6편) 등 도메인별 ZTA 적용 현황을 종합적으로 분석한다.
- **중요성**: ZTA의 최신 연구 동향과 도메인별 적용 현황을 한눈에 파악할 수 있는 최신 종합 자료이다.

## 5. 컨테이너 보안

### 5-1. Borg, Omega, and Kubernetes

- **제목**: Borg, Omega, and Kubernetes: Lessons Learned from Three Container-Management Systems over a Decade
- **저자**: Brendan Burns, Brian Grant, David Oppenheimer, Eric Brewer, John Wilkes (Google)
- **연도**: 2016
- **출처**: ACM Queue, Vol. 14, No. 1 / Communications of the ACM
- **요약**: Google에서 10년간 운영한 세 가지 컨테이너 관리 시스템(Borg, Omega, Kubernetes)의 교훈을 정리한 논문이다. Borg의 중앙 집중식 관리, Omega의 Paxos 기반 낙관적 동시성 제어, Kubernetes의 오픈소스 설계 철학을 비교 분석한다.
- **중요성**: 현대 컨테이너 오케스트레이션의 기원과 설계 원칙을 이해하는 데 필수적인 논문으로, Kubernetes 보안 아키텍처 이해의 출발점이다.

### 5-2. Towards Improving Container Security by Preventing Runtime Escapes

- **제목**: Towards Improving Container Security by Preventing Runtime Escapes
- **저자**: Berkay Celik et al.
- **연도**: 2021
- **출처**: IEEE SecDev 2021
- **요약**: 11개 컨테이너 런타임과 59개 취약점에 대한 보안 연구로, 28개 CVE에 대한 7가지 분류 체계를 제시한다. 컨테이너 탈출(escape)의 주요 원인이 호스트 컴포넌트가 컨테이너 내부로 누출되는 것임을 밝히고, 사용자 네임스페이스 방어가 9개 탈출 익스플로잇 중 7개를 차단할 수 있음을 보여준다.
- **중요성**: 컨테이너 탈출 공격의 체계적 분류와 방어 전략을 제시한 실용적 보안 연구이다.

### 5-3. An Overview of Container Security in a Kubernetes Cluster

- **제목**: An Overview of Container Security in a Kubernetes Cluster
- **저자**: IEEE Conference Publication
- **연도**: 2023
- **출처**: IEEE Xplore (DOI: 10.1109/10158865)
- **요약**: Kubernetes 클러스터에서의 컨테이너 보안을 종합적으로 개관한 논문이다. 공격자가 애플리케이션 컨테이너를 통해 데이터를 획득하거나 기업 인프라로의 진입점으로 활용할 수 있는 위협을 분석한다.
- **중요성**: Kubernetes 환경의 보안 위협과 대응 방안을 실무적 관점에서 정리한 참고 자료이다.

## 6. 소프트웨어 공급망 보안

### 6-1. in-toto: Providing Farm-to-Table Guarantees for Bits and Bytes

- **제목**: in-toto: Providing Farm-to-Table Guarantees for Bits and Bytes
- **저자**: Santiago Torres-Arias, Hammad Afzali, Trishank Karthik Kuppusamy, Reza Curtmola, Justin Cappos (NYU)
- **연도**: 2019
- **출처**: 28th USENIX Security Symposium
- **요약**: 소프트웨어 공급망의 무결성을 암호학적으로 보장하는 프레임워크를 제시한 논문이다. 30건 이상의 실제 공급망 침해 사례를 재현한 실험에서 83% 이상의 공격을 효과적으로 방지할 수 있음을 증명했다.

- **출처**: IEEE Software / arXiv:1702.05843
- **요약**: 카오스 엔지니어링의 원칙과 방법론을 체계적으로 정의한 논문이다. Netflix의 실제 경험을 바탕으로, 프로덕션 또는 사전 프로덕션 환경에 의도적으로 결함을 주입하여 시스템 행동에 미치는 영향을 관찰하는 복원력 테스트 방법론을 제시한다.
- **중요성**: 카오스 엔지니어링을 학술적으로 정의하고 체계화한 기초 논문으로, 이 분야의 출발점이다.

### 12-2. Chaos Engineering in the Wild: Findings from GitHub

- **제목**: Chaos Engineering in the Wild: Findings from GitHub
- **저자**: (arXiv:2505.13654)
- **연도**: 2025
- **출처**: arXiv
- **요약**: GitHub에서의 카오스 엔지니어링 도구 채택 현황을 실증적으로 분석한 논문이다. 14개 도구(Chaos Mesh, LitmusChaos, Chaos Monkey 등)를 조사하여 Toxiproxy, Chaos Mesh, Chaos Monkey가 분석된 저장소의 64.57%를 차지하며 가장 널리 채택되었음을 발견했다.
- **중요성**: 카오스 엔지니어링의 실제 산업 채택 현황을 데이터 기반으로 분석한 최신 실증 연구이다.

### 12-3. Lineage-driven Fault Injection

- **제목**: Lineage-driven Fault Injection
- **저자**: Peter Alvaro, Joshua Rosen, Joseph M. Hellerstein (UC Berkeley)
- **연도**: 2015
- **출처**: ACM SIGMOD 2015
- **요약**: 정확한 시스템 결과로부터 역방향으로 추론하여 어떤 실행 실패가 결과를 방지할 수 있었는지를 결정하는 계보 기반 결함 주입 기법이다. 프로토타입 MOLLY는 데이터베이스 계보 기법과 최신 충족성 검사(SAT solving)를 결합하여, 랜덤 결함 주입 대비 10배 적은 실행으로 버그를 발견한다.
