## 1. 가상화 (Virtualization)

### 1.1 Xen and the Art of Virtualization
- **저자:** Paul Barham, Boris Dragovic, Keir Fraser, Steven Hand, Tim Harris, Alex Ho, Rolf Neugebauer, Ian Pratt, Andrew Warfield
- **연도:** 2003
- **학회:** ACM SOSP (Symposium on Operating Systems Principles)
- **요약:** x86 아키텍처에서 다수의 운영체제를 동시에 안전하게 실행할 수 있는 가상 머신 모니터 Xen을 제안한 논문이다. 준가상화(paravirtualization) 기법을 통해 베어메탈 대비 수 퍼센트 이내의 성능 오버헤드만으로 최대 100개의 VM을 동시 호스팅할 수 있음을 보였다. Amazon EC2 등 주요 클라우드 플랫폼의 기반 기술이 되었다.
- **중요성:** 현대 클라우드 컴퓨팅의 기반을 마련한 가상화 분야의 가장 영향력 있는 논문 중 하나로, 준가상화 개념을 대중화하였다.

### 1.2 kvm: the Linux Virtual Machine Monitor
- **저자:** Avi Kivity, Yaniv Kamay, Dor Laor, Uri Lublin, Anthony Liguori
- **연도:** 2007
- **학회:** Linux Symposium, Ottawa
- **요약:** Linux 커널에 하이퍼바이저 기능을 추가하는 KVM(Kernel-based Virtual Machine) 서브시스템을 소개한다. 하드웨어 가상화 확장(Intel VT-x, AMD-V)을 활용하여 Linux 프로세스처럼 동작하는 가상 머신을 생성하고 실행할 수 있다. I/O 가상화는 사용자 공간의 디바이스 모델을 통해 수행된다.
- **중요성:** Linux 커널에 직접 통합된 최초의 주류 하이퍼바이저로, 현재 대부분의 Linux 기반 가상화 인프라의 핵심 기술이다.

### 1.3 A Full GPU Virtualization Solution with Mediated Pass-Through (gVirt)
- **저자:** Kun Tian, Yaozu Dong, David Cowperthwaite
- **연도:** 2014
- **학회:** USENIX ATC (Annual Technical Conference)
- **요약:** Intel 내장 GPU를 위한 전체 GPU 가상화 솔루션 gVirt를 제안한다. Mediated Pass-Through 기법으로 게스트가 성능에 민감한 리소스에 직접 접근하되, 특권 연산은 트랩하여 에뮬레이션한다. GPU 집약적 워크로드에서 네이티브 대비 최대 95% 성능을 달성하고 7개 VM까지 확장 가능하다.
- **중요성:** GPU 가상화의 실용적 구현을 최초로 제시한 논문으로, 현대 vGPU와 SR-IOV 기반 GPU 공유 기술의 학술적 기반이 되었다.

## 2. VM 라이브 마이그레이션 (VM Live Migration)

### 2.1 Post-copy Live Migration of Virtual Machines
- **저자:** Michael R. Hines, Umesh Deshpande, Kartik Gopalan
- **연도:** 2009
- **학회:** ACM SIGOPS Operating Systems Review (VEE 2009 Best Paper 선정)
- **요약:** 기존 pre-copy 방식과 달리, VM의 프로세서 상태를 먼저 전송한 후 메모리를 나중에 가져오는 post-copy 라이브 마이그레이션을 제안한다. Xen 하이퍼바이저에서 실험한 결과, 전송 페이지 수, 총 마이그레이션 시간, 네트워크 오버헤드 등 여러 지표에서 pre-copy보다 우수한 결과를 보였다.
- **중요성:** Post-copy 기법은 메모리 집약적 워크로드에서 pre-copy의 수렴 문제를 해결하며, 현대 클라우드의 VM 마이그레이션 기술의 핵심 대안이다.

## 3. 컨테이너 기초 기술 (Container Fundamentals)

### 3.1 Docker: Lightweight Linux Containers for Consistent Development and Deployment
- **저자:** Dirk Merkel
- **연도:** 2014
- **학회:** Linux Journal
- **요약:** Linux 컨테이너(LXC) 기반으로 애플리케이션과 의존성을 경량 컨테이너에 패키징하는 Docker 시스템을 소개한다. Control Groups(cgroups)로 리소스를 제한하고, AuFS 계층형 파일시스템으로 데이터 중복을 방지한다. 다양한 Linux 배포판 간 이식성을 제공한다.
- **중요성:** 컨테이너 기술을 대중화한 핵심 문헌으로, namespaces와 cgroups 기반 격리의 실용적 활용을 Linux 커뮤니티에 널리 알렸다.

### 3.2 Cntr: Lightweight OS Containers
- **저자:** Jorg Thalheim, Pramod Bhatotia, Pedro Fonseca, Baris Kasikci
- **연도:** 2018
- **학회:** USENIX ATC
- **요약:** 컨테이너 이미지를 "slim" 이미지(주 애플리케이션)와 "fat" 이미지(개발 도구)로 분리하여, 런타임에 필요시 동적으로 결합하는 CNTR 시스템을 제안한다. Docker, rkt, LXC 등 모든 컨테이너 구현체와 호환되며, Docker Hub Top-50 이미지의 크기를 평균 66.6% 줄였다.
- **중요성:** 컨테이너 이미지의 비대화 문제를 근본적으로 해결하는 접근법으로, 보안과 이미지 크기 최적화를 동시에 달성한다.

## 4. 컨테이너 이미지 최적화 (Container Image Optimization)

### 4.1 Slacker: Fast Distribution with Lazy Docker Containers
- **저자:** Tyler Harter, Brandon Salmon, Rose Liu, Andrea C. Arpaci-Dusseau, Remzi H. Arpaci-Dusseau
- **연도:** 2016
- **학회:** USENIX FAST (Conference on File and Storage Technologies)
- **요약:** 57개 컨테이너 애플리케이션의 시작 시간을 분석한 HelloBench를 개발하여, 패키지 풀링이 컨테이너 시작 시간의 76%를 차지하지만 실제 읽히는 데이터는 6.4%에 불과함을 밝혔다. 중앙 집중 저장소에서 필요한 데이터만 지연 로딩하는 Slacker 드라이버를 구현하여 개발 사이클을 20배, 배포 사이클을 5배 가속하였다.
- **중요성:** 컨테이너 이미지 배포의 근본적 비효율성을 정량적으로 분석하고 지연 로딩 해법을 제시한 선구적 연구이다.

## 5. 컨테이너 오케스트레이션 (Container Orchestration)

### 5.1 Large-scale Cluster Management at Google with Borg
- **저자:** Abhishek Verma, Luis Pedrosa, Madhukar R. Korupolu, David Oppenheimer, Eric Tune, John Wilkes
- **연도:** 2015
- **학회:** ACM EuroSys
- **요약:** Google의 클러스터 관리 시스템 Borg를 소개한다. 수만 대의 머신으로 구성된 여러 클러스터에서 수천 개 애플리케이션의 수십만 개 작업을 실행한다. 고가용성, 효율적 리소스 활용, 사용자 편의성을 동시에 달성하는 설계를 다룬다.
- **중요성:** Kubernetes의 직접적 전신으로, 현대 컨테이너 오케스트레이션의 핵심 개념(선언적 배포, bin-packing 등)을 정립하였다.

### 5.2 Omega: Flexible, Scalable Schedulers for Large Compute Clusters
- **저자:** Malte Schwarzkopf, Andy Konwinski, Michael Abd-El-Malek, John Wilkes
- **연도:** 2013
- **학회:** ACM EuroSys
- **요약:** Google의 차세대 클러스터 관리 시스템 Omega를 제안한다. 기존 모놀리식 스케줄러의 한계를 극복하기 위해, 공유 상태 기반 병렬 스케줄링과 lock-free 낙관적 동시성 제어를 활용한다. 실제 Google 프로덕션 워크로드로 스케줄러 간 간섭을 평가하였다.
- **중요성:** 대규모 클러스터 스케줄링의 아키텍처적 대안을 제시하여, Borg에서 Kubernetes로 이어지는 발전의 중간 단계를 학술적으로 정립하였다.

### 5.3 Mesos: A Platform for Fine-Grained Resource Sharing in the Data Center
- **저자:** Benjamin Hindman, Andy Konwinski, Matei Zaharia, Ali Ghodsi, Anthony D. Joseph, Randy Katz, Scott Shenker, Ion Stoica
- **연도:** 2011
- **학회:** USENIX NSDI
- **요약:** Hadoop, MPI 등 다양한 클러스터 컴퓨팅 프레임워크가 하나의 클러스터를 세밀하게 공유할 수 있는 Mesos 플랫폼을 제안한다. 2단계 스케줄링(resource offers) 메커니즘을 통해 프레임워크별 정교한 스케줄러를 지원하면서 데이터 지역성을 달성한다.
- **중요성:** 현대 클러스터 관리 및 스케줄링 시스템에 관한 최초의 학술 논문 중 하나로, 리소스 공유 추상화의 기초를 확립하였다.

### 5.4 Borg, Omega, and Kubernetes
- **저자:** Brendan Burns, Brian Grant, David Oppenheimer, Eric Brewer, John Wilkes
- **연도:** 2016
- **학회:** ACM Queue / Communications of the ACM
- **요약:** Google의 Borg, Omega, Kubernetes 세 시스템의 설계 철학과 교훈을 정리한다. 10년 이상의 컨테이너 관리 경험에서 얻은 통찰을 바탕으로, 현대 클러스터 관리 시스템의 설계 원칙과 과제를 논의한다.
- **중요성:** 클라우드 네이티브 컴퓨팅의 발전 역사와 핵심 설계 결정을 이해하는 데 필수적인 문헌이다.

## 7. 컨테이너 네트워킹 (Container Networking)

### 7.1 Performance Analysis of CNI (Container Networking Interface) based Container Network
- **저자:** Park, Yang 등
- **연도:** 2018
- **학회:** IEEE Conference
- **요약:** OpenStack 클라우드 플랫폼과 Kubernetes 환경에서 다양한 CNI 기술의 네트워크 성능을 측정하였다. Flannel, Calico, Weave 등 주요 CNI 플러그인의 처리량, CPU 및 메모리 사용량을 비교 분석한다.
- **중요성:** 컨테이너 네트워킹 인터페이스의 성능을 체계적으로 비교한 최초의 연구 중 하나이다.

### 7.2 A Comprehensive Performance Evaluation of Kubernetes CNI Plugins
- **저자:** Gokhale 등 (Vanderbilt University)
- **연도:** 2021
- **학회:** IEEE IC2E
- **요약:** 9개 주요 CNI 플러그인(Flannel, Calico, Cilium, Antrea, Kube-router 등)의 TCP/HTTP 프로토콜 성능을 VMware ESXi 가상화 환경과 물리 데이터센터에서 평가하였다. Kube-Router가 TCP 전송에서 명목 대역폭의 90% 이상을 달성하여 최고 처리량을 보였다.
- **중요성:** 실제 운영 환경에서 CNI 선택 시 참고할 수 있는 종합적 벤치마크 결과를 제공한다.

## 8. 컨테이너 보안 및 런타임 격리 (Container Security & Runtime Isolation)

### 8.1 The True Cost of Containing: A gVisor Case Study
- **저자:** Eric Young 등
- **연도:** 2019
- **학회:** USENIX HotCloud
- **요약:** Google의 컨테이너 런타임 샌드박스 gVisor의 성능 비용을 분석한다. 단순 시스템 콜이 기존 컨테이너 대비 2.2배 이상 느리고, 외부 tmpfs에서 파일 열기/닫기가 216배 느린 것으로 나타났다. 대용량 파일 다운로드는 2.8배 느렸다.
- **중요성:** 보안 강화형 컨테이너 런타임의 성능 비용을 정량적으로 분석하여 보안-성능 트레이드오프를 이해하는 데 기여한다.

### 8.2 Performance and Isolation Analysis of RunC, gVisor and Kata Containers Runtimes
- **저자:** Wang, Du 등
- **연도:** 2022
- **학회:** Cluster Computing (Springer)
- **요약:** RunC, gVisor, Kata Containers 세 가지 컨테이너 런타임의 성능과 격리 수준을 비교 분석한다. gVisor는 I/O와 시스템 콜에서 상당한 성능 저하를 보이지만 격리 수준이 가장 우수하고, Kata Containers는 VM 수준 격리를 제공하면서 상대적으로 나은 성능을 보인다.
- **중요성:** 세 가지 주요 컨테이너 런타임의 보안-성능 스펙트럼을 종합적으로 비교한 중요한 참조 자료이다.

### 8.3 Performance Evaluation of Container Runtimes
- **저자:** Lennart Espe, Anshul Jindal
- **연도:** 2020
- **학회:** SciTePress
- **요약:** containerd(산업 표준)와 CRI-O(CRI 참조 구현)를 runc와 gVisor 두 가지 OCI 런타임에서 평가하였다. containerd가 CPU 사용량, 메모리 레이턴시, 확장성에서 우수하고, CRI-O는 파일시스템 연산에서 더 효율적이었다.
- **중요성:** Kubernetes 환경에서 고수준 컨테이너 런타임 선택 시 성능 기준을 제공한다.

## 9. 마이크로VM (MicroVM)

### 9.1 Firecracker: Lightweight Virtualization for Serverless Applications
- **저자:** Alexandru Agache, Marc Brooker, Andreea Florescu, Alexandra Iordache, Anthony Liguori, Rolf Neugebauer, Phil Piwonka, Diana-Maria Popa
- **연도:** 2020
- **학회:** USENIX NSDI
- **요약:** AWS Lambda와 Fargate를 구동하는 가상 머신 모니터 Firecracker를 소개한다. Rust로 작성된 약 50kloc의 경량 VMM으로, Linux KVM 위에서 동작한다. 컨테이너당 5MB 미만의 메모리 오버헤드, 125ms 이내 부팅, 호스트당 초당 150개 MicroVM 생성이 가능하다.
- **중요성:** 서버리스 컴퓨팅을 위한 경량 가상화의 산업 표준을 제시한 논문으로, 컨테이너의 속도와 VM의 격리를 결합하는 새로운 패러다임을 확립하였다.

### 9.2 Blending Containers and Virtual Machines: A Study of Firecracker and gVisor
- **저자:** Anjali, Caraza-Harter 등
- **연도:** 2020
- **학회:** ACM VEE (Virtual Execution Environments)
- **요약:** Docker 컨테이너(LXC), gVisor 보안 컨테이너, Firecracker microVM이 Linux 커널 서비스를 어떻게 다르게 사용하는지 비교 연구한다. Firecracker와 gVisor 모두 네이티브 Linux보다 상당히 더 많은 커널 코드를 실행하며, 워크로드에 따라 성능 우위가 달라진다.
- **중요성:** 컨테이너-VM 하이브리드 기술의 커널 수준 동작을 최초로 심층 분석한 논문이다.

### 9.3 My VM is Lighter (and Safer) than your Container (LightVM)
- **저자:** Filipe Manco 등
- **연도:** 2017
- **학회:** ACM SOSP
- **요약:** Xen 기반의 최적화된 가상화 솔루션 LightVM을 제안한다. XenStore를 대체하는 경량 드라이버 noxs와 VM 셸 사전 준비 기법으로, 2.3ms 만에 VM을 부팅할 수 있으며 이는 Docker보다 두 자릿수 빠르다. 유니커널과 경량화된 Linux(Tinyx)를 활용한다.
- **중요성:** VM이 컨테이너보다 빠르고 안전할 수 있음을 실증적으로 보여, 가상화 vs. 컨테이너 이분법에 근본적 문제를 제기하였다.

## 10. 유니커널 (Unikernel)

### 10.1 Unikernels: Rise of the Virtual Library Operating System (MirageOS)
- **저자:** Anil Madhavapeddy 등
- **연도:** 2013
- **학회:** ACM ASPLOS (일부), ACM Queue
- **요약:** "유니커널" 용어를 최초로 정의하고, OCaml 기반의 MirageOS를 통해 라이브러리 운영체제 개념을 제안한다. 타입 안전성(type-safety)을 중시하며, 애플리케이션에 특화된 단일 목적 VM 이미지를 생성한다. Solo5에서 1-2ms 부팅이 가능하다.
- **중요성:** 유니커널 개념을 학계에 도입한 선구적 연구로, 이후 모든 유니커널 연구의 기반이 되었다.

### 10.2 Unikraft: Fast, Specialized Unikernels the Easy Way
- **저자:** Simon Kuenzer, Vlad-Andrei Badoiu, Hugo Lefeuvre, Sharan Santhanam, Alexander Jung 등
- **연도:** 2021
- **학회:** ACM EuroSys (Best Paper Award)
- **요약:** OS 프리미티브를 완전히 모듈화하여 쉽게 커스터마이징할 수 있는 마이크로 라이브러리 OS Unikraft를 제안한다. nginx, SQLite, Redis를 Linux 게스트 대비 1.7-2.7배 성능 향상으로 실행하며, 이미지 크기 ~1MB, RAM 10MB 미만, 부팅 시간 ~1ms를 달성하였다.
- **중요성:** 유니커널의 실용화를 크게 앞당긴 연구로, Linux Foundation 프로젝트로 발전하며 Kubernetes와의 통합도 지원한다.

### 10.3 OSv: Optimizing the Operating System for Virtual Machines
- **저자:** Avi Kivity, Dor Laor, Glauber Costa, Pekka Enberg, Nadav Har'El, Don Marti, Vlad Zolotarov
- **연도:** 2014
- **학회:** USENIX ATC
- **요약:** 클라우드 컴퓨팅에 최적화된 OS로, 바이너리 호환성을 제공하여 다양한 프로그래밍 언어와 플랫폼을 지원한다. 모놀리식 커널 구조로 JVM, Python 등 기존 애플리케이션을 수정 없이 실행할 수 있다. Firecracker에서 읽기 전용 파일시스템으로 4-5ms 부팅이 가능하다.
- **중요성:** 기존 애플리케이션과의 호환성을 유지하면서 유니커널의 이점을 제공하는 실용적 접근법을 제시하였다.

- **중요성:** 서버리스 연구의 최신 동향을 가장 광범위하게 분석한 체계적 문헌 고찰이다.

## 12. 서버리스 콜드 스타트 (Serverless Cold Start)

### 12.1 Cold Start Latency in Serverless Computing: A Systematic Review, Taxonomy, and Future Directions
- **저자:** (다수 저자)
- **연도:** 2024
- **학회:** ACM Computing Surveys (arXiv: 2310.08437)
- **요약:** 서버리스 콜드 스타트 레이턴시 완화 메커니즘을 애플리케이션 기반, 체크포인트 기반, 호출 시간 예측 기반, 캐시 기반의 4가지로 분류한다. OpenWhisk와 AWS Lambda가 가장 빈번히 연구된 플랫폼이며, AI/ML 기반 솔루션이 새로운 연구 방향으로 부상하고 있다.
- **중요성:** 서버리스 콜드 스타트 문제의 전체 연구 지형을 체계적으로 정리한 최신 분류 체계를 제공한다.

### 12.2 FaaSLight: General Application-level Cold-start Latency Optimization for FaaS
- **저자:** (다수 저자)
- **연도:** 2023
- **학회:** ACM Transactions on Software Engineering and Methodology
- **요약:** FaaS 애플리케이션에서 필수 코드만 로딩하여 콜드 스타트를 최적화하는 FaaSLight 도구를 제안한다. 코드 로딩 레이턴시를 최대 78.95%(평균 28.78%) 줄이고, 전체 응답 레이턴시를 최대 42.05%(평균 19.21%) 감소시켰다.
- **중요성:** 애플리케이션 수준에서 콜드 스타트를 해결하는 실용적이고 범용적인 도구를 제시하였다.

### 12.3 SOCK: Rapid Task Provisioning with Serverless-Optimized Containers
- **저자:** Edward Oakes, Leon Yang, Dennis Zhou, Kevin Houck, Tyler Harter, Andrea C. Arpaci-Dusseau, Remzi H. Arpaci-Dusseau
- **연도:** 2018
- **학회:** USENIX ATC
- **요약:** Linux 컨테이너 프리미티브의 확장성 병목(스토리지, 네트워크 격리)을 분석하고, 서버리스에 최적화된 SOCK 컨테이너 시스템을 구현한다. Docker 대비 18배, Zygote 기반 캐싱으로 추가 3배 속도 향상을 달성하였다. AWS Lambda 대비 2.8배, OpenWhisk 대비 5.3배 오버헤드 감소를 보였다.
- **중요성:** 서버리스 워크로드 특성에 맞춘 컨테이너 최적화의 선구적 연구이다.

### 12.4 Catalyzer: Sub-millisecond Startup for Serverless Computing with Initialization-less Booting
- **저자:** Du, Yu 등 (SJTU IPADS)
- **연도:** 2020
- **학회:** ACM ASPLOS
- **요약:** 체크포인트 이미지에서 함수 인스턴스를 복원하여 초기화를 생략하는 init-less 부팅 기법과, 실행 중인 샌드박스 상태를 직접 재사용하는 sfork(sandbox fork) OS 프리미티브를 제안한다. gVisor 기반 구현에서 1ms 미만의 시작 레이턴시를 달성하였다.
- **중요성:** 스냅샷/복원 기반 콜드 스타트 해결의 핵심 연구로, 밀리초 이하의 서버리스 시작을 실현하였다.

## 13. FaaS (Function-as-a-Service)

### 13.1 Serverless in the Wild: Characterizing and Optimizing the Serverless Workload at a Large Cloud Provider
- **저자:** Mohammad Shahrad, Rodrigo Fonseca, Inigo Goiri 등
- **연도:** 2020
- **학회:** USENIX ATC (Community Award)
- **요약:** Azure Functions의 전체 프로덕션 FaaS 워크로드를 분석하였다. 대부분의 함수가 매우 드물게 호출되며 호출 빈도가 8자릿수 범위에 걸쳐 있다. 메모리 사용량 4배 차이, 50%의 함수가 1초 이내 실행. AWS/Azure의 고정 keep-alive 정책 대신 사용자별 적응형 keep-alive와 사전 워밍 정책을 제안하였다.
- **중요성:** 실제 대규모 서버리스 워크로드의 특성을 최초로 공개적으로 분석한 핵심 논문이다.

### 13.2 Architectural Implications of Function-as-a-Service Computing
- **저자:** Mohammad Shahrad 등
- **연도:** 2019
- **학회:** IEEE/ACM MICRO
- **요약:** FaaS 모델의 서버 수준 아키텍처 분석을 최초로 수행하였다. 바텀업 접근법으로 FaaS 워크로드의 하드웨어 활용 패턴, 마이크로아키텍처 특성, 리소스 병목을 분석한다.
- **중요성:** FaaS의 하드웨어/아키텍처적 함의를 분석한 최초의 연구로, 서버리스 인프라 설계에 중요한 통찰을 제공한다.

### 13.3 Function-as-a-Service Performance Evaluation: A Multivocal Literature Review
- **저자:** (다수 저자)
- **연도:** 2020
- **학회:** arXiv (2004.03276)
- **요약:** 학술 및 회색 문헌의 112개 연구를 분석하여 FaaS 성능 평가의 현황을 정리한다. 기존 연구가 주로 AWS Lambda를 대상으로 단순 함수의 마이크로벤치마크(CPU 속도, 콜드 스타트 오버헤드)에 집중되어 있음을 지적하였다.
- **중요성:** FaaS 성능 연구의 현황과 한계를 체계적으로 파악할 수 있는 포괄적 문헌 리뷰이다.
