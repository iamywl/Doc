# 분산 파일 시스템 및 오브젝트 스토리지 주요 논문 정리

---

## 1. 분산 파일 시스템 (Distributed File Systems)

### 1.1 The Google File System
- **제목:** The Google File System
- **저자:** Sanjay Ghemawat, Howard Gobioff, Shun-Tak Leung
- **연도:** 2003
- **학회:** SOSP '03 (19th ACM Symposium on Operating Systems Principles)

**요약:** 대규모 분산 데이터 집약적 애플리케이션을 위한 확장 가능한 분산 파일 시스템의 설계와 구현을 제시한다. 저가 상용 하드웨어에서 실행되면서 장애 허용을 제공하고, 다수의 클라이언트에게 높은 집계 성능을 전달한다. 파일을 고정 크기 청크로 분할하고, 단일 마스터 서버가 메타데이터를 관리하는 구조를 채택했다.

**중요성:** 현대 분산 스토리지 시스템의 설계 철학을 정립한 가장 영향력 있는 논문으로, HDFS, Colossus 등 후속 시스템의 직접적인 기반이 되었다. 6,300회 이상 인용된 분산 시스템 분야의 필독 논문이다.

---

### 1.2 The Hadoop Distributed File System (HDFS)
- **제목:** The Hadoop Distributed File System
- **저자:** Konstantin Shvachko, Hairong Kuang, Sanjay Radia, Robert Chansler
- **연도:** 2010
- **학회:** IEEE 26th Symposium on Mass Storage Systems and Technologies (MSST 2010)

**요약:** 매우 큰 데이터 세트를 안정적으로 저장하고 높은 대역폭으로 사용자 애플리케이션에 스트리밍하도록 설계된 HDFS의 아키텍처를 기술한다. 수천 대의 서버가 직접 연결된 스토리지를 호스팅하고 사용자 애플리케이션 작업을 실행하며, Yahoo!에서 25페타바이트의 엔터프라이즈 데이터를 관리한 경험을 보고한다.

**중요성:** 오픈소스 빅데이터 생태계의 핵심 저장 계층으로, MapReduce, Spark 등과 결합하여 빅데이터 혁명을 이끈 시스템이다. 5,000회 이상 인용되었으며 산업계에서 가장 널리 사용된 분산 파일 시스템이다.

---

### 1.3 Ceph: A Scalable, High-Performance Distributed File System
- **제목:** Ceph: A Scalable, High-Performance Distributed File System
- **저자:** Sage A. Weil, Scott A. Brandt, Ethan L. Miller, Darrell D. E. Long, Carlos Maltzahn
- **연도:** 2006
- **학회:** OSDI '06 (7th USENIX Symposium on Operating Systems Design and Implementation)

**요약:** 데이터와 메타데이터 관리를 최대한 분리하여 확장성을 달성하는 분산 파일 시스템을 제시한다. 할당 테이블 대신 CRUSH라는 의사 난수 데이터 분배 함수를 사용하여 이기종 및 동적 클러스터에서 객체의 위치를 계산(lookup 불필요)할 수 있게 했다. 초당 250,000건 이상의 메타데이터 연산을 지원하는 우수한 성능을 보여준다.

**중요성:** 오픈소스 통합 스토리지 플랫폼(블록/오브젝트/파일)의 대표 주자로, OpenStack, Kubernetes 등 클라우드 인프라의 핵심 스토리지로 자리잡았다. CRUSH 알고리즘은 분산 데이터 배치의 혁신적 접근법으로 평가받는다.

---

### 1.4 RADOS: A Scalable, Reliable Storage Service for Petabyte-scale Storage Clusters
- **제목:** RADOS: A Scalable, Reliable Storage Service for Petabyte-scale Storage Clusters
- **저자:** Sage A. Weil, Andrew Leung, Scott A. Brandt, Carlos Maltzahn
- **연도:** 2007
- **학회:** PDSW '07 (2nd International Petascale Data Storage Workshop, co-located with SC '07)

**요약:** 개별 스토리지 노드의 지능을 활용하여 수천 대의 장치로 확장할 수 있는 신뢰성 있는 객체 스토리지 서비스를 제시한다. 작은 클러스터 맵을 사용하여 노드가 반자율적으로 복제, 장애 감지, 장애 복구를 자체 관리하면서도 일관된 데이터 접근과 강력한 안전 시멘틱을 보존한다.

**중요성:** Ceph의 핵심 저장 계층으로, 분산 객체 스토어의 자율적 관리 개념을 정립했다. 현대 소프트웨어 정의 스토리지(SDS)의 기반 아키텍처로 널리 참조된다.

---

### 1.5 File Systems Unfit as Distributed Storage Backends: Lessons from 10 Years of Ceph Evolution
- **제목:** File Systems Unfit as Distributed Storage Backends: Lessons from 10 Years of Ceph Evolution
- **저자:** Abutalib Aghayev, Sage Weil, Michael Kozuch, Gregory R. Ganger 외
- **연도:** 2019
- **학회:** SOSP '19 (27th ACM Symposium on Operating Systems Principles)

**요약:** 10년간 Ceph가 로컬 파일 시스템 위에 스토리지 백엔드를 구축하면서 겪은 문제점을 분석하고, 이를 해결하기 위해 로우 스토리지 디바이스 위에서 직접 동작하는 BlueStore를 개발한 과정을 기술한다. BlueStore는 출시 2년 만에 기존 백엔드를 성능에서 능가했으며, 사용자의 70%가 프로덕션에서 채택했다.

**중요성:** 분산 스토리지 백엔드 설계에서 로컬 파일 시스템의 한계를 실증적으로 입증한 중요한 연구이다. 사용자 공간 스토리지 엔진 설계의 정당성과 방향성을 제시하여 후속 시스템 설계에 큰 영향을 미쳤다.

---

### 1.6 Facebook's Tectonic Filesystem: Efficiency from Exascale
- **제목:** Facebook's Tectonic Filesystem: Efficiency from Exascale
- **저자:** Satadru Pan, Theano Stavrinos, Yunqiao Zhang 외 (Facebook, Princeton University, Columbia University)
- **연도:** 2021
- **학회:** FAST '21 (19th USENIX Conference on File and Storage Technologies)

**요약:** 페이스북의 엑사바이트 규모 분산 파일 시스템인 Tectonic을 소개한다. 이전에 서비스별로 사용하던 전용 시스템들을 범용 멀티테넌트 파일 시스템 인스턴스로 통합하여, 전문화된 시스템에 필적하는 성능을 달성했다. 테넌트가 다양한 워크로드에 최적화할 수 있도록 연산 특수화를 지원한다.

**중요성:** 엑사바이트 규모에서의 스토리지 통합 및 멀티테넌시 설계의 실전 사례를 제공한다. 대규모 인터넷 기업의 스토리지 인프라 진화 방향을 보여주는 최신 연구이다.

---

### 1.7 CFS: A Distributed File System for Large Scale Container Platforms
- **제목:** CFS: A Distributed File System for Large Scale Container Platforms
- **저자:** Haifeng Liu, Wei Ding, Yuan Chen, Weilong Guo, Shuoran Liu, Tianpeng Li, Mofei Zhang 외 (JD.com)
- **연도:** 2019
- **학회:** SIGMOD '19 (ACM International Conference on Management of Data)

**요약:** 대규모 컨테이너 플랫폼을 위한 분산 파일 시스템으로, 순차 및 랜덤 파일 접근을 모두 지원하고 대용량/소용량 파일 모두에 최적화된 스토리지를 제공한다. 메타데이터 배치 전략으로 용량 확장 시 데이터 리밸런싱을 회피하며, 일반적인 7가지 메타데이터 연산에서 평균 3배의 성능 향상을 달성했다.

**중요성:** 컨테이너/클라우드 네이티브 환경에 특화된 분산 파일 시스템 설계를 제시했다. Kubernetes 기반 대규모 컨테이너 환경의 스토리지 요구사항을 해결하는 실용적 접근법을 보여준다.

---

### 1.8 Lustre: Building a File System for 1,000-node Clusters
- **제목:** Lustre: Building a File System for 1,000-node Clusters
- **저자:** Philip Schwan (Cluster File Systems, Inc.)
- **연도:** 2003
- **학회:** Ottawa Linux Symposium 2003

**요약:** 1,000노드 클러스터를 위한 GPL 라이선스 병렬 파일 시스템 Lustre의 설계와 구현을 기술한다. 당시 세계 최대 규모의 Linux 슈퍼컴퓨터 3대에서 테스트되었으며, 메타데이터와 파일 데이터를 분리하는 객체 기반 접근법과 파일 스트라이핑을 통한 병렬성을 활용한다.

**중요성:** HPC(고성능 컴퓨팅) 스토리지의 사실상 표준으로, Top 500 슈퍼컴퓨터의 80% 이상에서 사용되고 있다. 병렬 I/O 최적화의 핵심 참고 자료이다.

## 2. 오브젝트 스토리지 및 Blob 스토리지 (Object & Blob Storage)

### 2.1 Windows Azure Storage: A Highly Available Cloud Storage Service with Strong Consistency
- **제목:** Windows Azure Storage: A Highly Available Cloud Storage Service with Strong Consistency
- **저자:** Brad Calder, Ju Wang, Aaron Ogus, Niranjan Nilakantan, Arild Skjolsvold 외 (Microsoft)
- **연도:** 2011
- **학회:** SOSP '11 (23rd ACM Symposium on Operating Systems Principles)

**요약:** 고객이 거의 무한한 양의 데이터를 임의의 기간 동안 저장할 수 있는 클라우드 스토리지 시스템을 기술한다. Blob(파일), Table(구조적 저장), Queue(메시지 전달) 형태의 스토리지를 제공하며, 스토리지 스탬프를 통한 프로비저닝과 스케일아웃, 프론트엔드/파티션/스트림의 3계층 아키텍처를 상세히 설명한다.

**중요성:** 대규모 클라우드 스토리지 서비스의 내부 아키텍처를 최초로 공개한 논문 중 하나로, 강한 일관성과 높은 가용성을 동시에 달성하는 설계 패턴을 제시했다. Azure Storage의 기반이 되는 핵심 기술 문서이다.

---

### 2.2 Finding a Needle in Haystack: Facebook's Photo Storage
- **제목:** Finding a Needle in Haystack: Facebook's Photo Storage
- **저자:** Doug Beaver, Sanjeev Kumar, Harry C. Li, Jason Sobel, Peter Vajgel
- **연도:** 2010
- **학회:** OSDI '10 (9th USENIX Symposium on Operating Systems Design and Implementation)

**요약:** 2,600억 개 이상의 이미지(20PB 이상)를 저장하고 주당 10억 장의 새 사진이 업로드되며 피크 시 초당 100만 장 이상의 이미지를 서빙하는 Facebook의 사진 스토리지 시스템을 기술한다. 전통적 설계의 메타데이터 조회로 인한 과도한 디스크 연산 문제를 해결하기 위해, 사진당 메타데이터를 최소화하여 모든 메타데이터 조회를 메인 메모리에서 수행하도록 했다.

**중요성:** 대규모 비정형 데이터(사진/동영상) 저장에 특화된 Blob 스토리지 설계의 선구적 사례이다. 메타데이터 오버헤드 최소화를 통한 성능 최적화 전략은 후속 시스템 설계에 큰 영향을 미쳤다.

---

### 2.3 f4: Facebook's Warm BLOB Storage System
- **제목:** f4: Facebook's Warm BLOB Storage System
- **저자:** Subramanian Muralidhar, Wyatt Lloyd, Sabyasachi Roy, Cory Hill, Ernest Lin 외 (Facebook, Princeton University)
- **연도:** 2014
- **학회:** OSDI '14 (11th USENIX Symposium on Operating Systems Design and Implementation)

**요약:** BLOB의 접근 패턴을 분석하여 Hot/Warm 온도 구역을 식별하고, Warm BLOB에 특화된 스토리지 시스템 f4를 제시한다. 65PB 이상의 논리적 BLOB을 저장하면서 유효 복제 팩터를 3.6에서 2.8 또는 2.1로 감소시켰다. 디스크, 호스트, 랙, 데이터센터 장애에 대한 복원력을 유지하면서 낮은 지연시간을 제공한다.

**중요성:** 데이터 온도(Hot/Warm/Cold) 기반 계층화 스토리지 전략의 실전 적용 사례로, 스토리지 비용 절감과 내결함성 사이의 트레이드오프를 정량적으로 분석했다. 이레이저 코딩의 대규모 실전 적용 사례이기도 하다.

---

### 2.4 Ambry: LinkedIn's Scalable Geo-Distributed Object Store
- **제목:** Ambry: LinkedIn's Scalable Geo-Distributed Object Store
- **저자:** Shadi A. Noghabi, Subramanian Subramanian, Priyesh Narayanan 외 (LinkedIn, University of Illinois Urbana-Champaign)
- **연도:** 2016
- **학회:** SIGMOD '16 (ACM International Conference on Management of Data, Industrial Track)

**요약:** 사진, 동영상, 오디오 클립 등 수십억 개의 가변 크기 미디어 객체를 저장하고 서빙하기 위한 분산 객체 스토어를 제시한다. 논리적 blob 그룹핑, 비동기 복제, 리밸런싱 메커니즘, 무비용 장애 감지, OS 캐싱 등의 기법을 활용한다. 네트워크 대역폭의 88%까지 도달하는 처리량과 1MB blob을 50ms 이내에 서빙하는 낮은 지연시간을 달성했다.

**중요성:** 글로벌 소셜 네트워크의 미디어 스토리지 요구사항을 해결하는 실전 시스템으로, 지리적 분산 환경에서의 객체 스토어 설계 패턴을 제시했다. 디스크 간 불균형을 8~10배 개선하는 리밸런싱 기법이 주목할 만하다.

---

### 2.5 Dynamo: Amazon's Highly Available Key-value Store
- **제목:** Dynamo: Amazon's Highly Available Key-value Store
- **저자:** Giuseppe DeCandia, Deniz Hastorun, Madan Jampani, Gunavardhan Kakulapati, Avinash Lakshman, Alex Pilchin, Swaminathan Sivasubramanian, Peter Vosshall, Werner Vogels
- **연도:** 2007
- **학회:** SOSP '07 (21st ACM SIGOPS Symposium on Operating Systems Principles)

**요약:** 매우 높은 신뢰성 요구사항을 가진 서비스의 상태를 관리하기 위해 구축된 고가용성, 확장 가능한 분산 데이터 스토어를 제시한다. 일관된 해싱을 이용한 데이터 파티셔닝과 복제, 객체 버전관리를 통한 일관성 지원, 쿼럼 기반 기법, 가십 기반 장애 감지 프로토콜 등의 기법을 결합했다.

**중요성:** 분산 키-밸류 스토어 설계의 교과서적 논문으로, Cassandra, Riak, DynamoDB 등 수많은 후속 NoSQL 시스템에 직접적 영향을 미쳤다. 가용성과 일관성 사이의 트레이드오프를 실용적으로 해결한 이정표적 연구이다.

## 3. 이레이저 코딩 (Erasure Coding)

### 3.1 Erasure Coding in Windows Azure Storage
- **제목:** Erasure Coding in Windows Azure Storage
- **저자:** Cheng Huang, Huseyin Simitci, Yikang Xu, Aaron Ogus, Brad Calder, Parikshit Gopalan, Jin Li, Sergey Yekhanin
- **연도:** 2012
- **학회:** USENIX ATC '12 (USENIX Annual Technical Conference) - **Best Paper Award**

**요약:** Windows Azure Storage에서 데이터 내구성을 제공하면서 스토리지 비용을 낮추기 위한 이레이저 코딩의 실전 적용을 기술한다. 핵심 기여는 Local Reconstruction Codes(LRC)라는 새로운 코드 체계로, 오프라인 데이터 프래그먼트 복구 시 읽어야 할 프래그먼트 수를 줄이면서도 스토리지 오버헤드를 낮게 유지한다.

**중요성:** USENIX ATC Best Paper를 수상했으며, LRC는 Azure Storage를 시작으로 Microsoft의 모든 스토리지 프로덕션 라인(클라우드, 엔터프라이즈, 데스크톱)에 채택되었다. 이레이저 코딩의 실전 적용에서 가장 영향력 있는 논문 중 하나이다.

---

### 3.2 Erasure Coding for Distributed Storage: An Overview
- **제목:** Erasure Coding for Distributed Storage: An Overview
- **저자:** Alexandros G. Dimakis, Kannan Ramchandran 외
- **연도:** 2018 (arXiv), Survey
- **학회:** arXiv:1806.04437 (Survey Paper)

- **제목:** IPFS - Content Addressed, Versioned, P2P File System

**요약:** 엑사바이트 규모의 콜드 데이터 저장을 위한 기본 구성 단위인 랙 규모 하드디스크 기반 스토리지 유닛 Pelican을 제시한다. 콜드 데이터의 낮은 접근 빈도를 활용하여 에너지와 비용을 절감하며, 디스크 스핀업과 마운트 지연시간이 새로운 탐색 지연시간으로 작용한다.

**중요성:** 데이터 센터의 콜드 데이터 저장 비용 절감을 위한 혁신적 하드웨어-소프트웨어 공동 설계를 제시했다. 엑사바이트 규모의 아카이브 스토리지 경제학에 대한 통찰을 제공한다.
