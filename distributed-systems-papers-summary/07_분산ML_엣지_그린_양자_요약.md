## 1. 분산 머신러닝 학습 (Distributed ML Training)

### 1.1 서베이 논문

### [A Survey on Distributed Machine Learning](https://dl.acm.org/doi/10.1145/3377454)
- **저자**: Joost Verbraeken, Matthijs Baez, Wannes Meert, Cees Snoek, Max Welling, Jan van Rijn
- **연도/학회**: 2020, ACM Computing Surveys
- **핵심 요약**: 분산 머신러닝의 전반적인 기술 현황을 데이터 병렬화, 모델 병렬화, 통신 전략 등 핵심 개념을 중심으로 포괄적으로 정리한 서베이 논문.

### [Communication-Efficient Distributed Deep Learning: A Comprehensive Survey](https://arxiv.org/abs/2003.06307)
- **저자**: Zhenheng Tang, Shaohuai Shi, Xiaowen Chu, Wei Wang, Bo Li
- **연도/학회**: 2020, arXiv (2003.06307)
- **핵심 요약**: 분산 딥러닝의 통신 효율성 향상 알고리즘을 통신 동기화, 시스템 아키텍처, 압축 기술, 통신-연산 병렬화의 4개 차원으로 분류하여 종합 정리.

### [Distributed Training of Large Language Models: A Survey](https://www.sciencedirect.com/science/article/pii/S2949719125000500)
- **저자**: Various authors
- **연도/학회**: 2025, ScienceDirect
- **핵심 요약**: LLM 분산 학습의 병렬 전략, 프레임워크, 최적화 기법을 데이터/모델/파이프라인 병렬화 중심으로 심층 분석한 최신 서베이.

### 1.2 Parameter Server

### [Scaling Distributed Machine Learning with the Parameter Server](https://www.usenix.org/conference/osdi14/technical-sessions/presentation/li_mu)
- **저자**: Mu Li, David G. Andersen, Jun Woo Park, Alexander J. Smola, Amr Ahmed, Vanja Josifovski, James Long, Eugene J. Shekita, Bor-Yiing Su
- **연도/학회**: 2014, OSDI (USENIX)
- **핵심 요약**: 워커-서버 구조로 전역 파라미터를 관리하며 비동기 통신과 유연한 일관성 모델을 지원하는 파라미터 서버 프레임워크의 기념비적 논문.

### [Communication Efficient Distributed Machine Learning with the Parameter Server](https://papers.nips.cc/paper/5597-communication-efficient-distributed-machine-learning-with-the-parameter-server)
- **저자**: Mu Li, David G. Andersen, Alexander Smola, Kai Yu
- **연도/학회**: 2014, NeurIPS
- **핵심 요약**: 3세대 파라미터 서버 프레임워크를 통해 분산 ML의 통신 효율성을 크게 개선하고 대규모 워크로드를 효율적으로 처리하는 시스템적 최적화를 제안.

### 1.3 데이터 병렬화 및 AllReduce

### [Horovod: Fast and Easy Distributed Deep Learning in TensorFlow](https://arxiv.org/abs/1802.05799)
- **저자**: Alexander Sergeev, Mike Del Balso
- **연도/학회**: 2018, arXiv
- **핵심 요약**: Uber에서 개발한 Ring-AllReduce 기반 분산 딥러닝 프레임워크로, 최소한의 코드 수정으로 GPU 간 효율적 그래디언트 평균화를 수행.

### [BytePS: A Unified Architecture for Accelerating Distributed DNN Training in Heterogeneous GPU/CPU Clusters](https://www.usenix.org/conference/osdi20/presentation/jiang)
- **저자**: Yimin Jiang, Yibo Zhu, Chang Lan, Bairen Yi, Yong Cui, Chuanxiong Guo
- **연도/학회**: 2020, OSDI (USENIX)
- **핵심 요약**: AllReduce와 PS를 통합한 이종 GPU/CPU 클러스터용 분산 학습 아키텍처로, 256 GPU 환경에서 기존 대비 최대 84~245% 성능 향상 달성.

### 1.4 모델 병렬화 및 파이프라인 병렬화

### [Megatron-LM: Training Multi-Billion Parameter Language Models Using Model Parallelism](https://arxiv.org/abs/1909.08053)
- **저자**: Mohammad Shoeybi, Mostofa Patwary, Raul Puri, Patrick LeGresley, Jared Casper, Bryan Catanzaro
- **연도/학회**: 2019, arXiv (NVIDIA Research)
- **핵심 요약**: 텐서 병렬화를 통해 512개 GPU로 83억 파라미터 트랜스포머 모델을 76% 스케일링 효율로 학습하는 기법을 제안한 사실상 표준 논문.

### [Efficient Large-Scale Language Model Training on GPU Clusters Using Megatron-LM](https://dl.acm.org/doi/10.1145/3458817.3476209)
- **저자**: Deepak Narayanan, Mohammad Shoeybi, Jared Casper, Patrick LeGresley, Mostofa Patwary 외
- **연도/학회**: 2021, SC (Supercomputing Conference)
- **핵심 요약**: 텐서, 파이프라인, 데이터 병렬화를 결합한 3D 병렬화로 1조 파라미터 규모 모델 학습을 가능하게 한 실질적 구현 가이드.

### [GPipe: Efficient Training of Giant Neural Networks using Pipeline Parallelism](https://arxiv.org/abs/1811.06965)
- **저자**: Yanping Huang, Youlong Cheng, Ankur Bapna, Orhan Firat 외
- **연도/학회**: 2019, NeurIPS (Google)
- **핵심 요약**: 마이크로배치 분할 기반 파이프라인 병렬화 라이브러리로, 동기식 그래디언트 업데이트를 유지하면서 디바이스 수에 거의 선형적인 속도 향상을 달성.

### [PipeDream: Generalized Pipeline Parallelism for DNN Training](https://dl.acm.org/doi/10.1145/3341301.3359646)
- **저자**: Deepak Narayanan, Aaron Harlap, Amar Phanishayee, Vivek Seshadri, Nikhil R. Devanur, Gregory R. Ganger, Phillip B. Gibbons, Matei Zaharia
- **연도/학회**: 2019, SOSP
- **핵심 요약**: 1F1B 스케줄링으로 인터배치 파이프라이닝과 인트라배치 병렬화를 결합하여 GPipe 대비 1.7배 빠른 학습 처리량을 달성한 비동기 파이프라인 병렬화 논문.

---

### 1.5 메모리 최적화

### [ZeRO: Memory Optimizations Toward Training Trillion Parameter Models](https://arxiv.org/abs/1910.02054)
- **저자**: Samyam Rajbhandari, Jeff Rasley, Olatunji Ruwase, Yuxiong He
- **연도/학회**: 2020, SC (Supercomputing Conference) / Microsoft Research
- **핵심 요약**: 가중치, 그래디언트, 옵티마이저 상태를 디바이스 간 분할하여 메모리 중복을 제거하는 ZeRO를 제안, DeepSpeed에 통합되어 1조 파라미터 모델 학습을 가능하게 함.

## 2. Mixture of Experts (MoE) 분산 학습

### [GShard: Scaling Giant Models with Conditional Computation and Automatic Sharding](https://arxiv.org/abs/2006.16668)
- **저자**: Dmitry Lepikhin, HyoukJoong Lee, Yuanzhong Xu, Dehao Chen, Orhan Firat, Yanping Huang, Maxim Krikun, Noam Shazeer, Zhifeng Chen
- **연도/학회**: 2020, ICLR (Google)
- **핵심 요약**: 조건부 계산 기반 희소 MoE 트랜스포머를 6000억 파라미터까지 확장하고, 2048개 TPU v3에서 자동 샤딩 API를 활용한 대규모 분산 학습을 실증.

### [Switch Transformers: Scaling to Trillion Parameter Models with Simple and Efficient Sparsity](https://jmlr.org/papers/v23/21-0998.html)
- **저자**: William Fedus, Barret Zoph, Noam Shazeer
- **연도/학회**: 2022, JMLR (Google)
- **핵심 요약**: Top-1 라우팅으로 MoE를 단순화하고 bfloat16 학습을 가능하게 하여 1조 파라미터 규모 희소 모델의 효율적 학습 방법을 제시.

## 3. 분산 추론 및 서빙 (Distributed Inference & Serving)

### [Orca: A Distributed Serving System for Transformer-Based Generative Models](https://www.usenix.org/conference/osdi22/presentation/yu)
- **저자**: Gyeong-In Yu, Joo Seong Jeong, Geon-Woo Kim, Soojeong Kim, Byung-Gon Chun
- **연도/학회**: 2022, OSDI (USENIX)
- **핵심 요약**: 반복 수준 스케줄링과 선택적 배칭으로 GPT-3 175B 모델에서 FasterTransformer 대비 36.9배 처리량 향상을 달성한 continuous batching의 원조 논문.

### [Efficient Memory Management for Large Language Model Serving with PagedAttention (vLLM)](https://dl.acm.org/doi/10.1145/3600006.3613165)
- **저자**: Woosuk Kwon, Zhuohan Li, Siyuan Zhuang, Ying Sheng, Lianmin Zheng, Cody Hao Yu, Joseph E. Gonzalez, Hao Zhang, Ion Stoica
- **연도/학회**: 2023, SOSP
- **핵심 요약**: OS 가상 메모리에서 영감받은 PagedAttention으로 KV 캐시 메모리 낭비를 거의 0으로 줄여 기존 대비 2~4배 처리량을 향상한 LLM 서빙의 사실상 표준.

### [Parallax: Efficient Distributed LLM Inference on Heterogeneous Hardware](https://gradient.network/parallax.pdf)
- **저자**: Gradient Network Research Team
- **연도/학회**: 2024, Technical Report
- **핵심 요약**: 지리적으로 분산된 이종 하드웨어에서 파이프라인 병렬화와 동적 KV 캐시 관리를 통해 end-to-end 지연 3.1배, 토큰 간 지연 5.3배 개선을 달성.
