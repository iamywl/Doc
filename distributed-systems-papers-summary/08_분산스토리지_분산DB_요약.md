# 분산 스토리지 & 분산 DB - 논문 요약 및 링크

## 1. 분산 파일 시스템 & 기반 인프라

### [The Google File System](https://dl.acm.org/doi/10.1145/945445.945450)
- **저자**: Sanjay Ghemawat, Howard Gobioff, Shun-Tak Leung
- **연도/학회**: 2003 / SOSP '03
- **핵심 요약**: 대규모 데이터 집약적 애플리케이션을 위한 확장 가능한 분산 파일 시스템으로, 저가 상용 하드웨어에서 내결함성과 높은 집계 성능을 제공하며 HDFS를 비롯한 현대 분산 파일 시스템의 설계 원형이 되었다.

### [MapReduce: Simplified Data Processing on Large Clusters](https://www.usenix.org/conference/osdi-04/mapreduce-simplified-data-processing-large-clusters)
- **저자**: Jeffrey Dean, Sanjay Ghemawat
- **연도/학회**: 2004 / OSDI '04
- **핵심 요약**: Map과 Reduce 두 함수만 정의하면 시스템이 자동으로 분산, 병렬화, 내결함성을 처리하는 프로그래밍 모델을 제시하여, Hadoop 에코시스템의 기초이자 대규모 데이터 처리 패러다임을 근본적으로 변화시킨 논문이다.

## 2. 합의 알고리즘 (Consensus)

### [Paxos Made Simple](https://lamport.azurewebsites.net/pubs/paxos-simple.pdf)
- **저자**: Leslie Lamport
- **연도/학회**: 2001 / ACM SIGACT News
- **핵심 요약**: 분산 시스템에서 합의를 달성하기 위한 Paxos 알고리즘을 간결하게 설명한 논문으로, Google Spanner, Apache ZooKeeper, Chubby 등 수많은 분산 시스템의 이론적 기반이 되었다.

### [In Search of an Understandable Consensus Algorithm (Raft)](https://www.usenix.org/conference/atc14/technical-sessions/presentation/ongaro)
- **저자**: Diego Ongaro, John Ousterhout
- **연도/학회**: 2014 / USENIX ATC '14 (Best Paper Award)
- **핵심 요약**: Paxos와 동등한 결과를 내면서도 이해하기 쉬운 합의 알고리즘 Raft를 제안하였으며, etcd, CockroachDB, TiKV, Consul 등 현대 분산 시스템에서 가장 널리 사용되는 합의 알고리즘이다.

## 3. 분산 키-값 저장소 (Distributed Key-Value Store)

### [Dynamo: Amazon's Highly Available Key-value Store](https://dl.acm.org/doi/10.1145/1294261.1294281)
- **저자**: Giuseppe DeCandia, Deniz Hastorun, Madan Jampani 외
- **연도/학회**: 2007 / SOSP '07
- **핵심 요약**: Consistent Hashing, 벡터 클럭, Sloppy Quorum 등을 조합하여 "always-on" 고가용성 키-값 저장소를 구현하였으며, 최종 일관성 개념을 대중화하고 Cassandra, Riak 등 수많은 NoSQL 시스템에 직접적 영향을 미쳤다.

### [Amazon DynamoDB: A Scalable, Predictably Performant, and Fully Managed NoSQL Database Service](https://www.usenix.org/conference/atc22/presentation/elhemali)
- **저자**: Mostafa Elhemali, Niall Gallagher, Nick Gordon 외
- **연도/학회**: 2022 / USENIX ATC '22
- **핵심 요약**: DynamoDB가 10년간 진화하여 2021년 Prime Day에서 초당 8,920만 요청을 처리한 완전 관리형 서비스가 된 과정을 설명하는, 클라우드 NoSQL 서비스 운영 경험의 산업계 필독 논문이다.

### [FoundationDB: A Distributed Unbundled Transactional Key Value Store](https://dl.acm.org/doi/10.1145/3448016.3457559)
- **저자**: Jingyu Zhou, Meng Xu, Alexander Shraer 외
- **연도/학회**: 2021 / SIGMOD '21
- **핵심 요약**: 인메모리 트랜잭션 관리, 분산 스토리지, 설정 시스템을 분리(unbundle)하는 아키텍처로 전체 키스페이스에 걸친 직렬화 가능 트랜잭션을 지원하며, Apple iCloud 등 핵심 서비스의 기반으로 사용된다.

## 4. NoSQL 데이터베이스

### [Bigtable: A Distributed Storage System for Structured Data](https://www.usenix.org/conference/osdi-06/bigtable-distributed-storage-system-structured-data)
- **저자**: Fay Chang, Jeffrey Dean, Sanjay Ghemawat 외
- **연도/학회**: 2006 / OSDI '06
- **핵심 요약**: 수천 대의 서버에서 페타바이트급 구조화 데이터를 관리하는 컬럼 패밀리 데이터 모델과 SSTable 기반 저장 구조를 제시하여, HBase, Cassandra, LevelDB 등 NoSQL 운동의 촉발제가 된 논문이다.

### [Cassandra: A Decentralized Structured Storage System](https://dl.acm.org/doi/10.1145/1773912.1773922)
- **저자**: Avinash Lakshman, Prashant Malik
- **연도/학회**: 2010 / ACM SIGOPS Operating Systems Review
- **핵심 요약**: Dynamo의 파티셔닝/복제 전략과 BigTable의 데이터 모델을 결합한 하이브리드 분산 스토리지 시스템으로, Apache Cassandra로 오픈소스화되어 Netflix, Apple 등에서 널리 사용된다.

---

## 5. NewSQL / 분산 SQL 데이터베이스

### [Spanner: Google's Globally-Distributed Database](https://www.usenix.org/conference/osdi12/technical-sessions/presentation/corbett)
- **저자**: James C. Corbett, Jeffrey Dean, Michael Epstein 외
- **연도/학회**: 2012 / OSDI '12 (Jay Lepreau Best Paper Award)
- **핵심 요약**: TrueTime API를 통해 전 세계 규모에서 외부 일관성을 보장하는 분산 트랜잭션을 최초로 지원한 시스템으로, CockroachDB, YugabyteDB 등 후속 분산 SQL 시스템의 직접적 기반이 되었다.

### [F1: A Distributed SQL Database That Scales](https://dl.acm.org/doi/10.14778/2536222.2536232)
- **저자**: Jeff Shute, Radek Vingralek, Bart Samwel 외
- **연도/학회**: 2013 / VLDB '13
- **핵심 요약**: Google AdWords를 위해 Spanner 위에 구축된 분산 관계형 데이터베이스로, NoSQL의 확장성과 전통 SQL의 일관성을 결합하고 무중단 스키마 변경 기법을 제시하였다.

### [Calvin: Fast Distributed Transactions for Partitioned Database Systems](https://dl.acm.org/doi/10.1145/2213836.2213838)
- **저자**: Alexander Thomson, Thaddeus Diamond, Shu-Chun Weng, Kun Ren, Philip Shao, Daniel J. Abadi
- **연도/학회**: 2012 / SIGMOD '12
- **핵심 요약**: 결정론적 순서 보장을 이용하여 분산 커밋 프로토콜(2PC) 없이 분산 트랜잭션의 경합 비용을 대폭 줄이는 접근법을 제시하여, Spanner와 대비되는 대안적 패러다임을 열었다.

### [CockroachDB: The Resilient Geo-Distributed SQL Database](https://dl.acm.org/doi/10.1145/3318464.3386134)
- **저자**: Rebecca Taft, Irfan Sharif, Andrei Matei 외
- **연도/학회**: 2020 / SIGMOD '20
- **핵심 요약**: 상용 하드웨어에서 일관된 지리적 분산 트랜잭션을 지원하는 오픈소스 분산 SQL 데이터베이스로, Spanner의 오픈소스 대안으로서 가장 성공적인 분산 SQL 구현 사례 중 하나이다.

### [Megastore: Providing Scalable, Highly Available Storage for Interactive Services](https://research.google/pubs/megastore-providing-scalable-highly-available-storage-for-interactive-services/)
- **저자**: Jason Baker, Chris Bond, James C. Corbett 외
- **연도/학회**: 2011 / CIDR '11
- **핵심 요약**: 세밀한 데이터 파티션 내에서 ACID 시맨틱스를 제공하면서 WAN 동기식 복제와 데이터센터 간 페일오버를 지원하는, BigTable과 Spanner 사이의 진화 단계를 보여주는 시스템이다.

## 6. 클라우드 네이티브 데이터베이스

### [Amazon Aurora: Design Considerations for High Throughput Cloud-Native Relational Databases](https://dl.acm.org/doi/10.1145/3035918.3056101)

### [Socrates: The New SQL Server in the Cloud](https://dl.acm.org/doi/10.1145/3299869.3314047)
- **저자**: Panagiotis Antonopoulos, Alex Budovski, Cristian Diaconu 외
- **연도/학회**: 2019 / SIGMOD '19
- **핵심 요약**: Azure SQL Database Hyperscale의 기반으로, 컴퓨팅, 로그, 스토리지, 백업을 완전히 분리하여 각 계층을 독립적으로 확장 가능하게 한 Microsoft의 클라우드 네이티브 SQL Server 재설계 사례이다.

### [PolarFS: An Ultra-low Latency and Failure Resilient Distributed File System for Shared Storage Cloud Database](https://dl.acm.org/doi/10.14778/3229863.3229872)
- **저자**: Wei Cao, Zhenjun Liu, Peng Wang 외
- **연도/학회**: 2018 / VLDB '18

### [Cloud-Native Database Systems at Alibaba: Opportunities and Challenges](https://dl.acm.org/doi/10.14778/3352063.3352141)
- **저자**: Feifei Li 외
- **연도/학회**: 2019 / VLDB '19
- **핵심 요약**: Alibaba의 PolarDB, AnalyticDB, X-Engine 등 클라우드 네이티브 DB 시스템의 아키텍처와 설계 철학을 종합적으로 소개하며, 클라우드 네이티브 DB의 기회와 도전 과제를 체계적으로 정리하였다.

## 7. LSM-Tree & 스토리지 엔진

### [The Log-Structured Merge-Tree (LSM-Tree)](https://link.springer.com/article/10.1007/s002360050048)
- **저자**: Patrick O'Neil, Edward Cheng, Dieter Gawlick, Elizabeth O'Neil
- **연도/학회**: 1996 / Acta Informatica
- **핵심 요약**: 메모리 상주 트리와 디스크 상주 트리를 병합 프로세스로 결합하여 쓰기 성능을 극대화하는 자료구조로, LevelDB, RocksDB, Cassandra, HBase 등 거의 모든 쓰기 최적화 시스템의 기반이다.

### [WiscKey: Separating Keys from Values in SSD-Conscious Storage](https://www.usenix.org/conference/fast16/technical-sessions/presentation/lu)
- **저자**: Lanyue Lu, Thanumalayan Sankaranarayana Pillai, Andrea C. Arpaci-Dusseau, Remzi H. Arpaci-Dusseau
- **연도/학회**: 2016 / FAST '16
- **핵심 요약**: 키와 값을 분리 저장하여 LSM-tree의 쓰기 증폭 문제를 해결하는 아이디어를 제안하였으며, LevelDB 대비 로딩 시 최대 111배, 랜덤 조회 시 최대 14배 빠른 성능을 달성하였다.

### [MyRocks: LSM-Tree Database Storage Engine Serving Facebook's Social Graph](https://dl.acm.org/doi/10.14778/3415478.3415546)
- **저자**: Yoshinori Matsunobu, Siying Dong, Herman Lee
- **연도/학회**: 2020 / VLDB '20
- **핵심 요약**: Facebook의 수십 페타바이트 규모 User Database에 RocksDB 기반 MySQL 스토리지 엔진을 적용하여 InnoDB 대비 인스턴스 크기를 62.3% 줄이고 서버 수를 절반 이하로 감소시킨 산업계 사례이다.

### [LSM-based Storage Techniques: A Survey](https://arxiv.org/abs/1812.07527)
- **저자**: Chen Luo, Michael J. Carey
- **연도/학회**: 2020 / The VLDB Journal
- **핵심 요약**: LSM-tree 기반 스토리지 기술에 대한 포괄적 서베이로, 체계적 분류 체계를 제공하고 LevelDB, RocksDB, Cassandra, HBase 등 대표적 오픈소스 NoSQL 시스템을 비교 분석하였다.

## 9. HTAP (Hybrid Transactional/Analytical Processing)

### [TiDB: A Raft-based HTAP Database](https://dl.acm.org/doi/10.14778/3415478.3415535)
- **저자**: Dongxu Huang, Qi Liu, Qiu Cui, Zhuhe Fang 외 (PingCAP)
- **연도/학회**: 2020 / VLDB '20
- **핵심 요약**: 행 저장소(TiKV)와 컬럼 저장소(TiFlash)로 구성된 멀티-Raft 스토리지에서 Raft 로그를 비동기 복제하여 실시간 HTAP를 구현한, Raft 기반 HTAP 데이터베이스의 최초 학술 발표 논문이다.

### [F1 Lightning: HTAP as a Service](https://dl.acm.org/doi/10.14778/3415478.3415553)
- **저자**: Jiacheng Yang, Ian Rae, Jun Xu, Jeff Shute 외 (Google)
- **연도/학회**: 2020 / VLDB '20
- **핵심 요약**: Spanner 기반 OLTP 엔진에 CDC를 통한 데이터 복제 서비스를 통합하여 행 형식을 컬럼 형식으로 변환하는 HTAP 기능을 서비스 형태로 제공하는 Google 내부 운영 사례이다.

### [PolarDB-IMCI: A Cloud-Native HTAP Database System at Alibaba](https://dl.acm.org/doi/10.1145/3589785)
- **저자**: Alibaba Cloud 팀
- **연도/학회**: 2023 / SIGMOD '23
- **핵심 요약**: PolarDB에 인메모리 컬럼 인덱스(IMCI)를 통합하여 OLTP에 5% 미만의 영향으로 분석 쿼리를 TPC-H 기준 최대 149배 가속하는 클라우드 네이티브 HTAP 시스템이다.

### [HTAP Databases: A Survey](https://arxiv.org/abs/2404.15670)
- **저자**: Chao Zhang, Guoliang Li, Jintao Zhang, Xinning Zhang, Jianhua Feng (청화대학교)
- **연도/학회**: 2024 / arXiv / VLDB Journal
- **핵심 요약**: HTAP 데이터베이스를 스토리지 아키텍처에 따라 체계적으로 분류하고 각 접근법의 장단점을 비교한, HTAP 분야의 가장 최신이고 포괄적인 서베이 논문이다.
